# Persistent Sparse Autoencoders: Learning Feature Timescales in Language Models

Abstract 

Sparse autoencoders (SAEs) decompose lan-guage model activations into sparse features, but standard SAEs encode each token indepen-dently and do not expose information that per-sists across a sequence. We introduce Persistent Sparse Autoencoders (Persistent SAEs), which extend standard SAEs by learning a persistence coefficient for each feature, allowing the model to learn which features should persist and for how long. Our experiments show that they re-tain competitive reconstruction quality while learning a spectrum of feature timescales: fast features behave as locally interpretable detec-tors, whereas slow features concentrate topic-level information in a persistent state. More-over, as shown in a prompt-injection monitor-ing case study, slow features preserve detection signals and remain causally effective over long contexts. These results suggest that Persistent SAEs open up new opportunities for interpret-ing and monitoring language models through persistent semantic representations. 

1 Introduction 

Understanding why a large language model (LLM) produces a particular output requires identifying the information represented in its hidden states and how that information evolves as the model pro-cesses a sequence (Luo and Specia, 2024; Sharkey et al., 2025). Sparse autoencoders (SAEs) (Huben et al., 2024; Lan et al., 2024) have become a lead-ing approach to this problem. An SAE recon-structs an LLM’s dense hidden state using a small number of learned features that may align with human-interpretable descriptions (Huben et al., 2024; Bricken et al., 2023; Templeton et al., 2024; Gao et al., 2025). This decomposition has enabled researchers to inspect concepts represented by lan-guage models and intervene on selected features to

Although causal transformer hidden states integrate the preceding context, standard SAEs encode each token independently (Huben et al., 2024; Bussmann et al., 2024, 2025). This atemporal formulation favours token-specific features but obscures the semantic state maintained by the model, that is, the topic-level context it carries across a sequence (e.g., topic, style, or task state). Recent methods incorporate temporal structure into SAEs through predefined partitions and decompositions of temporal roles (Bhalla et al., 2026; Lubana et al., 2026). However, they specify in advance which subset should carry temporal information before reconstruction, so the resulting temporal roles are predefined without clear evidence. It therefore remains unclear whether the temporal role of features can emerge directly from the data and reconstruction training. 1arXiv:2607.17117v1 [cs.LG] 19 Jul 2026 We explore this question starting from the observation that temporal persistence associated with SAE features is already present in the hidden states. As shown in Figure 2, after a feature activates, later hidden states remain aligned with its decoder direction, and its past activations contain information that helps reconstruct subsequent hidden states. Yet, standard SAEs encode each position independently and therefore do not represent this persistence explicitly. To fill this gap, we introduce Persistent Sparse Autoencoders ,which learn a persistence coefficient λj for every feature. Persistent SAEs separate persistent state 

from sparse evidence : each feature maintains a decaying state variable , updated by token-specific sparse evidence , with λj controlling its timescale and how long past evidence is retained. If carrying a feature’s state forward helps reconstruct later hidden states, there is an incentive to learn a higher 

λj during optimisation; if it does not, the objective favours a smaller λj and faster decay. Our experiments show that Persistent SAEs re-tain competitive reconstruction quality while learn-ing a broad range of feature timescales. Most are low-λj fast features that behave as locally inter-pretable detectors like standard SAEs, while high-

λj slow features form compact semantic states that retain topic-level information in the hidden states. In a prompt-injection monitoring case study (Debenedetti et al., 2024), these slow fea-tures preserve detection signals and remain causally effective on the monitor’s judgement consistently after the injection. These results show that SAE reconstruction can organise sparse features by timescale, separating local evidence from a com-pact persistent state that tracks semantic context and can steer a monitor’s judgement over long con-texts. Therefore, our main contributions can be summarised as: • We introduce Persistent Sparse Autoencoders ,which learn a persistence coefficient for each sparse feature through reconstruction alone. Persistent SAEs retain competitive reconstruc-tion quality while separating low-λj fast fea-tures that act as locally interpretable detectors from high-λj slow features that concentrate topic-level information in a persistent state. • Through a prompt-injection monitoring case study, we demonstrate the potential of slow features for monitoring and for steering mod-els over long contexts, opening up new oppor-tunities for interpreting and controlling LLMs through compact semantic states.

Temporal structure in LLM representations. 

Although an autoregressive LLM builds xt from the preceding context, a standard SAE computes each ft independently. It therefore does not ex-plicitly expose information carried across tokens. Traditional slow feature analysis shows that slowly varying latent factors can reveal higher-level struc-ture in neural networks (Wiskott and Sejnowski, 2002). 5 Experiments 

We first test whether adding persistent state preserves good reconstruction quality and locally interpretable sparse evidence. We then ask what the learned timescales organise, testing whether slow features jointly form a compact representation of semantic context. Finally, Section 6 studies whether this persistent state remains useful in a long-context monitoring case study. 

Models and hyperparameters. We use the same models and dataset as Section 3. We compare with BatchTopK SAE (Bussmann et al., 2024) and Matryoshka SAE (Bussmann et al., 2025) as strong standard baselines, and Temporal SAE (Bhalla et al., 2026) as the closest temporally structured baseline. All models are trained on the same activations with m = 16 ,384 features and a BatchTopK budget of k = 20 . Matryoshka and Temporal SAEs both use the same predefined 20%/80% feature partition. Appendix B gives the full training configuration. We show that training stabilises for Persistent SAEs without continual growth in the active state variable. 

5.1 Reconstruction Quality 

Table 1 reports fraction of variance explained (FVE), cosine similarity between the hidden state and its reconstruction, fraction of active fea-tures, and the temporal smoothness metric adopted from Bhalla et al. (2026), which measures the degree to which features change smoothly over time. The Persistent SAE remains competitive with the strongest token-specific baselines. On Pythia-160M, it reaches 0.93 FVE and 0.95 cosine simi-larity. We show that the high-λj slow features have smoothness 0.05 on Pythia and 0.04 on Gemma, whereas low-λj features score 0.20 on both mod-els. The Persistent SAE therefore preserves fast features for token-specific detail while concentrat-ing temporal continuity in a smaller slow subset without predefined feature partitions. Figure 4 (a) shows that the learned timescales of Persistent SAEs span a broad range. Most features are fast, with a smaller tail of highly persistent features. These slow features also coactivate more strongly and with richer structure than randomly selected features (Figure 4 (b)). This suggests that persistent context may be represented jointly by a structured group of slow features, which we investigate in Section 5.3. 

5.2 Fast Features Are Locally Interpretable

(2023): a judge model explains a feature from its highest-activation contexts, then scores how well that explanation predicts activa-tion. For the Persistent SAE, we evaluate the sparse evidence at. This asks whether local evidence re-mains understandable before recurrence is applied (see Appendix E for autointerpretation details). Across both models, low-λj fast features are eas-ier to explain locally than high-λj slow features (Table 1). Fast features match or exceed the best baseline score on both models. 5.3 Slow Features Track Semantic States Slow features form subject-specific clusters. 

The automated interpretability results suggest that slow features should be evaluated as a group and over a sequence. We first examine their geometry using MMLU examples from five subjects. For each sample, we collect the representations of its final 20 tokens and visualise them in two dimen-sions using t-SNE (see Appendix F for details). We select the highest 1% slow features for visual-isation. 6 Case Study: Persistent Monitoring of Prompt Injection 

The preceding experiments identify two comple-mentary roles: fast features detect interpretable patterns at the current token, while slow features in-tegrate local evidence into a compact semantic state. Indirect prompt injection (Debenedetti et al., 2024) provides a practical test of this separation. An agent may encounter a malicious instruction in an email, webpage, or tool response, continue reasoning for hundreds of tokens, and act only later (Debenedetti et al., 2025). A useful agentic monitor must there-fore do more than detect the injected text when it ap-pears; it must preserve a security-relevant signal un-til the agent acts (Zhan et al., 2024; Ye et al., 2026). 

Setting. We use AgentDojo trajectories from the banking, travel, and workspace do-mains (Debenedetti et al., 2024), generated by Llama-3.3-70B-Instruct (Dubey et al., 2024). Each trajectory contains a user request, an untrusted span with either benign content or a prompt injection, and the agent’s subsequent tool interactions. We consider only failed attacks, in which an injection is present but the agent does not complete the attacker’s target action. This prevents the monitor from relying on an overtly malicious action to identify an attack. We use Gemma-2-2B as an external monitor: we replay each trajectory through the monitor model and record the layer-12 residual stream. We train a linear probe at the final token of the untrusted span ( d = 0 ), where the injection is directly visible, and evaluate the same probe up to 

350 tokens later without retraining (Figure 7 (a)). We compare baseline SAE activations, fast features (approximated with evidence at), and the highest 

1% slow features. This evaluation measures how long each representation preserves the injection signal (see Appendix H for evaluation details). 

Fast features detect injections locally; slow fea-tures track them over longer contexts. At d =0, Persistent SAE evidence and the token-specific SAE baselines reach 94.0–96.7% accuracy (Fig-ure 7 (b), top). Thus, fast features are equally strong locally interpretable detectors while the in-jection text is present. As the evaluation moves beyond the untrusted span, evidence and the token-specific representations fall towards chance. In con-trast, the slow features remain informative across longer distances. Fast and slow features therefore play complementary roles: fast features detect the injection locally, while slow features carry its sig-nal through the subsequent context. 

Slow features causally influence the monitor’s judgement. Probe accuracy shows that a repre-sentation contains injection information, but not whether the corresponding direction influences the monitor’s own judgement. To test this, we truncate each trajectory at a checkpoint, append a query asking whether it should be flagged for prompt in-jection, and intervene at the final query token. 7 Discussion and Conclusion 

Discussion. Standard SAEs encode each token independently, even though hidden states also rep-resent information accumulated over earlier tokens. Persistent SAEs address this mismatch by learning a feature-specific persistence coefficient through reconstruction alone, separating fast, locally inter-pretable features from a slow features that retains information across tokens. This slow subset forms topic-specific clusters, supports topic probing nearly as well as supervised baseline, and preserves prompt-injection signals after their local evidence has passed. These properties make slow features a promising compact semantic state for unsupervised topic tracking and for monitoring or intervening on safety-relevant signals over long contexts. This mechanism has several limitations in its current form. Its diagonal recurrence gives each feature a single, fixed persistence coefficient and cannot model feature interactions or explicit re-sets. The prompt-injection study uses an external replay monitor over agent trajectories: interven-tions shift the monitor’s judgements, but we do not establish whether they affect the original agent’s actions. Future work can incorporate richer recur-rent dynamics and test whether interventions affect end-to-end agent behaviour. 

Conclusion. Persistent SAEs learn, from recon-struction alone, which sparse features should retain information across tokens and for how long. They preserve competitive reconstruction quality while separating fast, locally interpretable features from slow features that concentrate topic information and preserve monitoring and causal signals over long contexts. Persistent SAEs provide a potential tool for interpreting and monitoring the long-range information maintained by LLMs. 8References

C Dataset Details 

We use three datasets for distinct parts of the evalu-ation. Pile documents provide the language-model hidden states used to train and evaluate the SAEs, MMLU questions provide subject-labelled text for analysing semantic state, and AgentDojo trajecto-ries support the prompt-injection monitoring case study. We describe each dataset and our use of it below. 

C.1 Pile Corpus 

The original Pile is an 825 GiB English-language corpus assembled from 22 sources, including aca-demic papers, web pages, books, code, and online discussions (Gao et al., 2020). We use the Pile Un-copyrighted release (monology, 2023), a derived version that removes the Books3, BookCorpus2, OpenSubtitles, YouTube Subtitles, and OpenWeb-Text2 components. Documents are accessed in streaming mode and shuffled with a buffer of size 

10 ,000 using a fixed seed. A document is accepted if, after tokenisation with the model tokeniser and truncation to the context cap, it contains at least the required minimum number of tokens. 12 For the natural corpus, each accepted docu-ment is passed through the model and the chosen residual-stream hidden states are retained. If a se-quence has retained length L, temporal statistics are computed on token positions {16 , . . . , L − 1}

after dropping the initial tokens. The shuffled control is constructed at the sen-tence level rather than by permuting tokens. Sen-tences are sampled from documents drawn from the same corpus and concatenated subject to a sim-ple adjacency constraint that prevents consecutive sentences from coming from the same source docu-ment. This preserves local continuity while break-ing global continuity, including long-range entity tracking and discourse progression. 

C.2 MMLU Dataset 

Massive Multitask Language Understanding (MMLU) is an English-language benchmark of four-option multiple-choice questions from 57 subjects spanning STEM, the humanities, the social sciences, and other professional and academic fields (Hendrycks et al., 2021). We use MMLU only for the semantic analyses; it is not used to train the SAEs. Specifically, we use questions from abstract algebra, astronomy, business ethics, computer security, and European history. These subjects cover distinct areas while providing a subject label for every example. For the geometry analysis in Section 5.3, we sample 100 sequences across the five subjects and retain the final 20 token representations from each sequence. For contextual probing, we construct sequences by joining equal-length segments from different subjects and assign each token the sub-ject label of its source segment. Appendix F and Appendix G provide the sampling and evaluation details for the two analyses. 

C.3 AgentDojo Trajectories 

Indirect prompt injection occurs when an agent retrieves attacker-controlled text from emails, web-pages, documents, or tool outputs and follows in-structions inside that text. AgentDojo (Debenedetti et al., 2024) evaluates this setting with stateful tool environments containing 97 user tasks, 27 in-jection targets, and 629 security test cases across workspace, Slack, travel, and banking domains. Its task-specific checks evaluate whether the legit-imate user request and the attacker’s target action were completed by inspecting the agent output and changes to the environment. For our monitoring dataset, we use trajecto-ries from the banking, travel, and workspace do-mains generated by Llama-3.3-70B-Instruct. Be-nign trajectories without an injection form the neg-ative class ( y = 0 ). The positive class ( y = 1 )contains runs with an important_instructions 

or direct attack for which the AgentDojo injec-tion checker returns security=false . This value means that an injection was present but the agent did not complete the attacker’s target action. We retain these failed attacks deliberately: success-ful attacks often contain an overt malicious action, such as transferring money or sending private data, which could provide a trivial downstream classifi-cation cue. This restriction instead asks whether the monitor retains the semantic trace of an injec-tion before, or even without, an observable security failure. We do not filter by task utility, so the legiti-mate user request may either succeed or fail.
