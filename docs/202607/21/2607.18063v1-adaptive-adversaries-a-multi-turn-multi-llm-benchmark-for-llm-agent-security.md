## 摘要
基于大语言模型的智能体处理外部内容，使其面临提示注入和多轮操控的风险。大多数安全基准测试在评估防御者时，使用的是评估前收集的固定攻击池，无论是单轮还是多轮形式。我们提出了一个包含21个场景的基准测试，用于评估“针对无记忆 LLM 防御者的自适应多轮攻击”：自主的 LLM 攻击者会观察防御者之前的响应并在多轮中进行策略调整，而每次防御者的响应都被视为一次全新的交互来评估。在固定21个场景、攻击者、防御者和结构化输出评分的情况下，仅将评分限制在攻击者的第一轮，得到的攻击成功率（ASR）为0-1%；而允许15轮自适应攻击时，该成功率达到5.4-14.0%。汇集三个前沿攻击者 LLM 能够发现比最佳单个攻击者多1.4-2.2倍的独特成功攻击，并且生成的攻击与现有基准测试中的攻击具有较低的余弦相似度（0.02-0.14）。Claude Opus 4.6 和 GPT-5.4 在总体表现上持平（均为5.4%；95%置信区间重叠），但它们的弱点差异显著：在某个场景中，Opus 的攻击成功率达到60%（95% CI 36-80%），而 GPT-5.4 和 Gemini 各自停留在7%（CI 1-30%；在更大样本量的重复实验中差异依然存在）。在21个场景中，有13个场景至少能区分一对防御者，然而各场景下的排名并不一致（肯德尔协同系数 W = 0.19）。我们发布了该基准测试——包含21个评估场景、10个公开开发场景、协调器、基线工具包以及一个多攻击者命令行界面——此外还提供了来自3×3前沿模型的945份对话记录、一个攻击回放数据集，以及来自一项公开竞赛最终评分轮的18,422场 gpt-oss-20b 对战数据。

## Abstract
LLM-based agents process external content, exposing them to prompt injection and multi-turn manipulation. Most safety benchmarks evaluate defenders against fixed attack pools collected before evaluation, single-turn or multi-turn. We present a 21-scenario benchmark for \emph{adaptive multi-round attacks against memoryless LLM defenders}: an autonomous LLM attacker observes prior defender responses and pivots across rounds, while each defender response is evaluated as a fresh interaction. Holding the 21 scenarios, attackers, defenders, and structured-output scoring fixed, restricting scoring to the first attacker turn yields $0$-$1\%$ attack success rate (ASR); allowing 15 rounds of adaptive attack yields $5.4$-$14.0\%$. Pooling three frontier attacker LLMs uncovers $1.4$-$2.2\times$ as many unique successful attacks as the best single attacker, and the generated attacks have low cosine similarity ($0.02$-$0.14$) to attacks in existing benchmarks. Claude Opus 4.6 and GPT-5.4 are tied in aggregate ($5.4\%$ each; overlapping $95\%$ CIs), but their weaknesses differ sharply: on one scenario Opus reaches $60\%$ ASR ($95\%$ CI $36$--$80\%$) while GPT-5.4 and Gemini each stay at $7\%$ (CI $1$-$30\%$; the gap is preserved in a higher-$N$ replication). $13$ of $21$ scenarios distinguish at least one defender pair, yet rankings disagree across scenarios (Kendall's $W = 0.19$). We release the benchmark -- 21 evaluation scenarios, 10 public development scenarios, the orchestrator, baseline harnesses, and a multi-attacker CLI -- plus 945 transcripts from the 3$\times$3 frontier matrix, an attack-replay dataset, and 18{,}422 gpt-oss-20b battles from an open competition's final scoring rounds.

- **现有问题**：当前多数安全基准（如 JailbreakBench、HarmBench、AgentHarm 等）均使用**评估前收集的固定攻击池**，且多为单轮或攻击者不可动态调整的设定。这无法反映真实攻击中攻击方根据防御方反应实时调整策略的自适应能力。
- **核心目标**：构建一个支持**自适应多轮攻击、多攻击方大模型、每场对战动态生成攻击**的评测基准，从而更真实地评估 LLM 智能体的安全性，并揭示不同前沿模型在具体场景下的脆弱性差异。

### 2. 论文提出的方法论
- **核心思想**：将安全评测建模为“无记忆防御者 vs. 自适应攻击者”的15轮对战。攻击方 LLM 可观察防御方在前几轮的输出，并根据历史反馈动态生成新的注入攻击；防御方则**每轮被当作全新交互（无记忆）**，仅处理当前格式化输入，避免历史对话残留影响判断。
- **关键技术细节**：
  - **场景构建**：共包含21个评估场景（及10个公开开发场景），覆盖间接注入、数据泄露、不安全输出、供应链、提示提取等5类威胁。每个场景定义防御者角色、受保护资产、输入模板和**结构化输出成功判断函数 `check_success`**。
  - **成功判定**：采用基于类型化字段的严格检查，只在该场景指定的字段中出现不安全内容或受保护资产到达非攻击方时才计为成功，避免部分匹配、回显或模糊响应引发的误判。
  - **校准流水线**：通过三层筛选（平衡配置 BAL、强攻弱防 SVW、弱攻强防 WVS）评估场景的难度、能力敏感性和所需轮数，剔除无区分力或存在缺陷的场景。
  - **基准执行协议**：使用 AgentBeats 平台，攻击方每轮生成攻击文本→填充至场景模板→防御方输出结构化 JSON→执行 `check_success`。攻击方看到全部历史，防御方仅见当前轮输入。
- **算法/公式**：无显式数学公式，核心是以带条件逻辑的评测循环代替固定攻击集；攻击成功率的计算为成功对战场次占总对战次数的比例，并用 Bootstrap 计算95%置信区间。

### 6. 论文的主要结论与发现
- **自适应多轮攻击显著提升威胁**：仅对第一轮评分时 ASR 为 0–1%，允许 15 轮自适应攻击后，ASR 升高至 5.4–14.0%。这证实响应条件化是提升攻击成功率的关键驱动。
- **多攻击方联合可发现更多漏洞**：组合三个攻击 LLM 比只用最强单个攻击方多找出 **1.4–2.2 倍**的独特成功攻击。且攻击内容与现有基准的嵌入相似度仅为 0.02–0.14，表明固定攻击池遗漏了大量攻击面。
- **防御方弱点在不同场景下截然相反**：Claude Opus 4.6 和 GPT‑5.4 总体 ASR 均为 5.4%，但 Opus 在 `memleak` 场景中脆弱性突出（60% ASR），而 Gemini 在 `paperaccept` 和 `gameguru` 等“伪造权威”场景中暴漏严重漏洞（53–60%）。13/21 个场景能区分至少一对防御者，但场景间排名一致性极低（肯德尔 W = 0.19），表明**不存在单一的“最安全”模型**。
- **评测设计的重要性**：开放竞赛表明，独立提交的攻击和防御策略空间远超基线，进一步验证了动态评测协议的价值。同时，简单子串匹配的成功标准极易被攻击方利用，应采用基于实际后果的检查。

### 7. 优点
- **填补评测空白**：首次系统提出“多轮、多 LLM 攻击者、每场动态生成攻击”的智能体安全基准，弥补了现有静态攻击池的不足。
- **实验设计严谨**：场景经过三层量化校准，确保区分力；结构化字段检查避免了模糊评分和奖励黑客；包含丰富的消融和转移实验。
- **跨基准对比维度全面**：将本基准的 ASR、信息性场景比例、排名一致性等指标与6个外部基准系统比较，清晰呈现不同评测的信号特征。
- **开放与可复现**：发布完整场景、协调器、基线代码、对战记录、攻击回放集和竞赛轨迹数据（CC‑BY‑4.0 / MIT），支持研究者自行评测新攻击/防御。
- **实际发现具有启发性**：暴露了同一模型在不同场景中的尖锐脆弱性（如 Opus 的密码泄露、Gemini 的伪造权威服从），证明综合评价而非单一总分的必要性。

### 8. 不足与局限
- **样本量约束**：主 3×3 矩阵 N=5 导致个别场景的 ASR 估算置信区间较宽；虽然对关键场景做了 N=20 验证，但整体统计效力有限。
- **防御者模型设定简化**：裁判采用“无记忆”防御者，仅衡量单次输入的抗注入能力，未考虑有状态代理可能通过对话历史提升或降低安全性的复杂情况（消融显示有状态防御者平均更稳健，但部分场景反而变差）。
- **攻击方能力瓶颈**：攻击者由前沿 LLM 代理担任，受限于模型自身的安全对齐和创造力，不能完全替代人类红队可能使用的更激进或合规外的攻击手段。
- **威胁覆盖范围偏窄**：21 个场景中 11 个为间接注入、6 个为数据泄露，供应链、提示提取等类型仅 1–2 个场景，且未纳入越狱（jailbreak）等直接对齐攻击，通用性受限。
- **可重放性而非完全复现**：由于使用闭源 API 且可能模型版本漂移，精确复现结果困难；虽然提供了记录和回放机制，但仍需通过开源模型 gpt‑oss‑20b 提供唯一可完全复现的参考点。
