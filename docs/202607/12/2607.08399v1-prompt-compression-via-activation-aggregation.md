## 摘要
大型语言模型通过数十层的激活传播来处理提示，然后生成响应。我们提出问题：指令提示中包含的任务相关信息能否被压缩成单个激活向量，并重新注入模型以取代原始token序列？我们证明，通过使用在中间层提取并在目标LLM的早期层注入的学习到的激活加权和，可以实现这一目标。压缩向量保留了任务相关信息，与完整提示处理相比，准确率下降不到$2\%$。除了实际意义外（包括在固定指令提示下减少每次查询的计算量，无需重新处理原始token序列），我们的分析揭示了LLM激活空间中的结构：(i) 中间层表示可以有意义地转移到早期层，表明信息编码方式存在一定程度的跨层兼容性；(ii) 单个激活向量编码了可量化且可恢复的语义信息量；(iii) 激活的加权和是一种鲁棒的表示压缩器。

## Abstract
Large language models process prompts by propagating activations through dozens of layers before generating a response. We ask whether the task-relevant information contained in an instruction prompt can be compressed into a single activation vector and re-injected into the model, replacing the original token sequence? We show this is achievable using a learned weighted sum of activations extracted at an intermediate layer and injected at an early layer of the target LLM. The compressed vector preserves task-relevant information, incurring an accuracy drop of under $2\%$ relative to full prompt processing. Beyond its practical implications, including reducing per-query computation for fixed instruction prompts without reprocessing the original token sequence, our analysis reveals structure in the activation space of LLMs: (i) mid-layer representations transfer meaningfully to early layers, suggesting a degree of cross-layer compatibility in how information is encoded; (ii) a single activation vector encodes a quantifiable and recoverable amount of semantic information; (iii) a weighted sum of activations is a robust representation compressor.

## 1. 核心问题与整体含义
- **研究动机**：现代大语言模型（LLM）在推理时常遇到重复的提示前缀（如系统指令、示例），每次调用都会重新计算这些固定 token，带来不必要的计算开销。
- **核心问题**：能否将指令提示中的任务相关信息压缩成单个激活向量，并在后续查询中直接注入模型以替代原始 token 序列？
- **整体含义**：若可行，既能减少固定提示的推理计算量，又能从机制可解释性角度揭示 LLM 激活空间的三个特性：(i) 中间层表示可跨层迁移到早期层；(ii) 单个向量能编码可量化的语义信息；(iii) 加权求和是一种鲁棒的表示压缩器。

## 2. 方法论
### 2.1 框架总览
- 采用 **三阶段** 流程：**提取 → 压缩 → 注入**。
  1. 对给定提示 `p = [t1, ..., tT]`，在目标 LLM 的中间层 `m` 提取隐藏状态序列 `H^{(m)}(p) ∈ R^{T×d}`。
  2. 使用压缩函数 `f : R^{T×d} → R^{d}` 将序列压缩成单个向量 `v = f(H^{(m)}(p))`。
  3. 在新的查询中，在早期层 `e` 将占位符 token 的激活替换为 `v`，模型继续前向传播并生成答案。
- 目标 LLM **冻结**，只训练压缩函数 `f`（损失为交叉熵）。

### 2.2 压缩函数设计
- **Weighting MLP (W‑MLP)（主要方法）**  
  假设激活可通过简单加权求和来压缩。使用一个小的 MLP `M_θ : R^d → R`，对每个 token 的激活 `h_i` 输出标量权重 `w_i`，最终向量：
  \[
  v = \sum_{i=1}^{T} w_i h_i^{(m)},\quad w_i = M_θ(h_i^{(m)})
  \]
  MLP 结构：4 层，隐层维度 `[2048, 1024, 512, 256]`，激活函数 ReLU。
- **Transformer Compressor (TC)（对照）**  
  使用小型 Transformer 编码器 `E_φ` 直接吸收完整序列 `H^{(m)}`，通过类 `[CLS]` 令牌输出压缩向量 `v`。  
  结构：2 层，2 头，隐藏维度 64。

### 2.3 注入细节
- 提取层 `m=12`，注入层 `e=2`（经消融实验确定）。
- 占位符 token 选择 UTF‑8 替换字符 U+FFFD（渲染为 `¿`）。
- 注入方式为**替换**（非加法），位置接近占位符 token。

## 6. 主要结论与发现
1. **压缩有效**：W‑MLP 在 Toy Task 分布内准确率仅比完整提示低不到 2%（85.35% vs. 86.92%），在 ARC‑Easy 上 Llama3.1‑8B 达到 77% test accuracy，同模型的完整提示为 92%。
2. **W‑MLP 优于 TC**：TC 更易过拟合（训练准确率高但测试差），而 W‑MLP 泛化更强，说明简单的加权和结构更稳健。
3. **中间提取、早期注入最优**：提取层 12、注入层 2 的组合效果最好，暗示中间层语义更丰富，且早期注入给模型更多处理层数。
4. **单个向量容量大**：一个 4096 维向量可同时编码 9 个任务而无明显性能下降，表明激活空间信息容量远高于单任务需求。
5. **可解释性**：W‑MLP 的权重自动聚焦于语义负载词（如“capital”“industry”等），同时标点符号（如问号）常获高权重，与注意力汇聚现象一致。
6. **结构发现**：验证了 LLM 激活空间的跨层兼容性、语义可恢复性以及加权求和作为压缩器的有效性。

## 7. 优点
- **轻量高效**：无需微调 LLM，只训练一个小型压缩网络，适用于多种开源模型。
- **方法简洁**：加权求和设计直观，继承线性表示假设，计算开销小。
- **可解释性好**：权重天然提供 token 重要性指示，无需额外归因分析。
- **分析深度**：从层间迁移、多任务容量到 token 级语义对齐，形成系统性的激活空间认知。
- **实用性潜力**：可应用于重复系统提示缓存、RAG 压缩、机器人 VLA 模型的指令编码等场景。

## 8. 不足与局限
- **有损压缩**：信息丢失不可避免，高信息密度或长上下文下的保真度尚未验证。
- **任务局限**：实验集中在知识检索和简短提示，未涉及多步推理、长文档等复杂任务。

- **压缩成本不免费**：需前向传播至少到中间层（如 LLaMA‑8B 的 12 层），更适合多次复用的场景。

- **容量上限未定**：虽然 9 任务无崩塌，但真实容量边界需要更大规模任务集进一步探索。
