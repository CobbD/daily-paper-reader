## 摘要
协同感知使自动驾驶汽车能够通过聚合来自多个智能体和感知平台的检测输出（通常通过多供应商部署中的共享融合服务）来提高态势感知能力。然而，在推理时共享这些输出会暴露专有模型行为和敏感环境信息，引发重大的隐私和安全问题。在本文中，我们提出Sarus，一个基于同态加密（HE）的隐私保护多供应商感知融合框架，能够在不泄露单个供应商输出的情况下进行聚合。每个供应商将检测编码为共享空间网格上的紧凑高斯矩向量，并将加密载荷传输到融合服务器，服务器直接在加密域中聚合它们。然后，融合结果被解密并通过按类别的网格合并重建为最终检测结果。我们分析了计算复杂度，表明供应商载荷构建具有线性扩展性，服务器端融合复杂度为$O(BV)$，其中$B$为占用网格数，$V$为供应商数量，而后处理复杂度为$O(B + \sum_{c\in \mathcal{C}} B_c^2)$，其中$\mathcal{C}$表示目标类别集，$B_c$为类别$c$的占用网格数。实验表明，在实际应用中具有线性扩展性，同态加密仅带来有界常数因子开销，且解密开销在后处理成本中占主导。在KITTI数据集上使用相机（YOLOv8）和激光雷达（PointPillars, PV-RCNN）检测器的实验表明，Sarus通过有效聚合互补检测来提升场景级覆盖率，特别是在个体模态性能下降的距离依赖区域。这些结果表明，当联合利用统计压缩和空间稀疏性时，隐私保护的多供应商感知融合在实时部署中是可行的。

## Abstract
Cooperative perception enables autonomous vehicles (AVs) to improve situational awareness by aggregating detection outputs from multiple agents and sensing platforms, often via a shared fusion service in multi-vendor deployments. However, sharing such outputs at inference time exposes proprietary model behavior and sensitive environmental information, creating significant privacy and security concerns. In this paper, we present Sarus, a privacy-preserving framework for multi-vendor perception fusion via homomorphic encryption (HE), enabling aggregation without revealing individual vendor outputs. Each vendor encodes detections as compact Gaussian moment vectors over a shared spatial lattice and transmits encrypted payloads to a fusion server, which aggregates them directly in the encrypted domain. The fused result is then decrypted and reconstructed into final detections through class-wise bin merging.   We analyze the computational complexity, showing linear scaling for vendor payload construction and $O(BV)$ server-side fusion with the number of occupied bins $B$ and vendors $V$, while postprocessing scales as $O(B + \sum_{c\in \mathcal{C}} B_c^2)$, where $\mathcal{C}$ denotes the set of object classes and $B_c$ is the number of occupied bins for class $c$. Experiments demonstrate linear scaling in practice with only a bounded constant-factor overhead from HE, with decryption dominating postprocessing cost. Experiments on the KITTI dataset using camera (YOLOv8) and LiDAR (PointPillars, PV-RCNN) detectors show that Sarus improves scene-level coverage by effectively aggregating complementary detections, particularly in distance-dependent regimes where individual modalities degrade. These results indicate that privacy-preserving multi-vendor perception fusion is feasible for real-time deployment when statistical compression and spatial sparsity are jointly exploited.

## 2. 论文提出的方法论

### 2.1 核心思想
- 将每个检测框转换为 **高斯展布（Gaussian splat）**，用加权空间矩向量表示。
- 通过共享的空间网格对检测进行软分配与局部聚合，使融合操作线性化，从而可用同态加密的加法同态性实现安全聚合。
- 服务器对加密矩向量直接求和，解密后由授权方恢复融合高斯参数，再通过图聚类重建最终检测框。

### 2.2 关键技术细节与流程

**检测编码与矩表示**
- 每个检测框 $b = [x_1, y_1, x_2, y_2]$ 被建模为高斯分布：中心 $(\mu_x, \mu_y)$ 取框中心，方差 $\sigma_x^2, \sigma_y^2$ 由框尺寸与可调参数 $\kappa$ 控制。
- 给定检测置信度 $p$ 与供应商信任权重 $\alpha$，有效权重 $w = \alpha p$，编码为 7 维矩向量：
  $$ \mathbf{m} = w \cdot [1,\; \mu_x,\; \mu_x^2,\; \sigma_x^2,\; \mu_y,\; \mu_y^2,\; \sigma_y^2] $$
- 矩向量具有 **线性可加性**：多个检测的矩向量简单相加即可得到聚合高斯分布的必要充分统计量，支持直接从聚合矩恢复融合高斯参数（引理 1）。

**空间网格与软分配**
- 对每个目标类 $c$，按预设锚点和步长构建均匀空间网格。
- 检测的高斯中心通过双线性插值将贡献分配到至多 4 个相邻网格单元，权重和为 1。
- 供应商在本地将多个检测的加权矩向量按网格积累，得到 **每网格的聚合矩向**量 $\mathbf{s}_{c,(i,j)}$。

**载荷构建与同态加密**
- 供应商对每个占有类 - 网格键的聚合矩向量用 **CKKS 同态加密**，同时携带明文统计量 $M$（加权置信度质量）和 $C$（累加权重计数）。
- 每个键至多产生一个密文，极大地压缩了加密次数（$|\mathcal{H}| \ll N$）。

**服务器端加密融合**
- 服务器收集所有供应商的载荷，按键合并密文：$\mathbf{c}^{\text{fused}}_{key} = \bigoplus_{v} \mathbf{c}_{v,key}$，利用加法同态。
- 同时聚合明文统计量。复杂度为 $O(BV)$，$B$ 为融合后占用网格数，$V$ 为供应商数。

**融合检测重建**
- 授权方解密聚合矩向量，通过反推公式恢复高斯均值与方差：
  $$ \mu_x = \frac{S_{w\mu_x}}{S_w},\quad \sigma_x^2 = \frac{S_{w\sigma_x^2}+S_{w\mu^2_x}}{S_w} - \mu_x^2 $$
- 将高斯转换为边界框：$x_1 = \mu_x - \lambda\sigma_x$ 等。
- 类内构建 **空间一致性图**：相邻网格的高斯若满足中心距离门控、IoU 及马氏距离条件，则连边；连通分量内矩向量相加，融合为最终检测，并恢复置信度 $\hat{p} = \hat{m}/\hat{c}$。

### 2.3 复杂度分析摘要
- 供应商：$O(N + B_v)$，$B_v$ 为某供应商的占用网格数。
- 服务器：$O(BV)$。
- 后处理重建：$O(B + \sum_c B_c^2)$，通常受类内图构建约束，实际低于最坏情况 $O(B^2)$。

## 3. 实验设计

### 3.1 数据集与场景

- **KITTI 数据集**：使用训练集样本，同步 RGB 图像与 LiDAR 点云，评估多模态融合的态势感知。

- **等价性验证**：比较 **同态融合（HE）与明文融合（Plain）** 的输出，采用 IoU、边界框中心偏差、尺寸偏差、高斯标准差偏差。

## 6. 论文的主要结论与发现

- **同态融合的数值保真性**：HE 与明文融合输出几乎相同，IoU 均超 0.99997，偏差仅为亚像素级，证明加密未引入实质性失真。
- **线性扩展性**：供应商载荷构建时间与占用网格数 $B$ 线性相关，服务器融合时间与 $B\times V$ 线性相关，后处理亦接近线性（最坏 $O(B^2)$），且 HE 带来的开销是常数因子而非改变复杂度。
- **开销来源**：供应商端加密和解密主导后处理开销；服务器端主要开销来自密文序列化/反序列化而非实际算术。
- **态势感知提升**：Sarus 能有效聚合相机和 LiDAR 的互补检测，在远距离或 LiDAR 稀疏区域特别明显（如 PointPillars 的覆盖显著提升），即便在强基线 PV‑RCNN 下仍有小幅增益。
- **可行性声明**：当联合利用统计压缩和空间稀疏性时，基于 CKKS 的隐私保护多供应商融合在实时部署中是可行的。

- **巧妙的线性化设计**：用高斯矩替代原始检测框，配合空间网格双线性分配，将感知融合转化为纯线性加法问题，天然适合同态加密。
- **理论分析扎实**：给出了完整的复杂度表达式，并实验验证线性趋势。
- **明确的隐私威胁模型**：界定诚实但好奇的服务器和好奇但互不信任的供应商，并依据语义安全论证保护力度。
