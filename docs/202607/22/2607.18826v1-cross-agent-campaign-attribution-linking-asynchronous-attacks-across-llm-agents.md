## 摘要
LLM代理的防御通常每次只评估一个会话。但在实际部署中，攻击可以分布在独立的代理、团队和运行时中，导致每个本地护栏仅获得稀疏的片段。我们形式化了跨代理异步攻击活动归因：在没有共享运行时状态、测试时活动标签或攻击者身份先知的情况下，关联来自同一潜在对抗活动的会话。我们引入了异步归因指纹向量（$A^2FV$），这是一种轻量级代理侧参考协议，用于根据代理可观测的工具使用、时序和提示残留对成对的活动相似性进行评分。我们还构建了SCD-v1，这是一个受控的角色匹配基准，包含良性流量、孤立攻击、多会话活动、匹配的非先知规避和泄漏审计。在SCD-v1上，$A^2FV$在活动关联方面达到了0.82的成对AUC，而在相同任务下，基于每会话检测器和分块LLM判断器的仅分数适应版本表现仍接近随机水平。最强的固定信号由结构和文体学残留携带，而时序则作为更丰富代理痕迹的诊断通道得以保留。交叉风格控制显示，该信号部分对风格敏感，但不能仅归约为风格。静态和维度感知的非先知压力测试进一步表明，成对可分性在受控规避下仍然存在。这些结果将跨代理攻击活动归因确立为在真实环境中保护LLM代理的一个独特评估层。

## Abstract
LLM-agent defenses are typically evaluated one session at a time. In deployment, however, attacks can be distributed across independent agents, teams, and runtimes, leaving each local guardrail with only a sparse fragment. We formalize cross-agent asynchronous campaign attribution: linking sessions from the same latent adversarial campaign without shared runtime state, test-time campaign labels, or attacker identity oracles. We introduce Asynchronous Attribution Fingerprint Vectors ($A^2FV$), a lightweight proxy-side reference protocol for scoring pairwise campaign similarity from proxy-observable tool-use, timing, and prompt residue. We also construct SCD-v1, a controlled persona-matched benchmark with benign traffic, isolated attacks, multi-session campaigns, matched non-oracle evasion, and leakage audits. On SCD-v1, $A^2FV$ achieves 0.82 pairwise AUC for campaign linking, while score-only adaptations of per-session detectors and chunked LLM judges remain near chance under the same task. The strongest fixed signal is carried by structural and stylometric residue, while timing is retained as a diagnostic channel for richer proxy traces. Crossed-style controls show that the signal is partly style-sensitive but not reducible to style alone. Static and dimension-aware non-oracle stress tests further show that pairwise separability persists under controlled evasion. These results establish cross-agent campaign attribution as a distinct evaluation layer for securing LLM agents in the wild.

### 1. 论文的核心问题与整体含义
- **问题背景**：LLM 智能体（agent）在真实部署中的攻击常常是**异步且跨代理**的。同一攻击者会将恶意意图分散到不同时间、不同代理的独立会话中，导致单个会话的防御机制只能看到局部碎片，无法关联全局威胁。
- **核心问题**：**跨代理异步攻击活动归因（cross-agent asynchronous campaign attribution）**：在无共享运行时状态、无测试时活动标签、无攻击者身份先知的情况下，从代理侧可观测的稀疏痕迹（工具调用、时序、提示残留）中判断多个会话是否属于同一对抗活动。
- **整体含义**：填补现有单会话安全评估的空白，将**活动级关联**确立为 LLM 代理安全的一个独立评估层，使安全防护从“单点检测”升级为“跨会话关联”。

### 2. 方法论：A²FV（异步归因指纹向量）
- **整体思想**：在共享代理层（proxy）上提取每个会话的三种可观测残留通道，通过计算成对相似度来量化活动关联，是一种**无模型训练、轻量级、可解释**的参考协议。
- **三种残留通道**：
  - **结构残留（Structural）**：装饰后的工具调用 n‑gram（n=1,2,3）、失败-恢复转换模式、基于 MITRE ATT&CK 的粗粒度战术直方图。将工具序列转为 ℓ₂‑归一化向量。
  - **时序残留（Temporal）**：工具调用时间间隔 ∆ 的统计量（矩、Dip 统计量、IQR/σ 等）和负载大小统计，标准化后归一化。短会话处理为有区分度的零填充。
  - **文体学残留（Stylometric）**：对原始提示做字符 n‑gram（n=3,4,5）和字节 n‑gram（UTF‑8）的 TF‑IDF 哈希向量，ℓ₂‑归一化。
- **成对归因得分**：
  - 对于每对会话 (s, s')，通道 d 的相似度 k_d = cos(f_d(s), f_d(s'))，当向量为零块时设为 0。
  - 总分 K_w(s,s') = ∑_{d} w_d · k_d(s,s')，权重 w 满足 ∑w_d = 1。
  - 聚类用的距离矩阵 D_w(s,s') = ∑_{d} w_d · (1 - k_d(s,s'))。
- **权重选取（无推理时标签）**：在标定集上用网格搜索（分辨率 0.1）选取使 NMI 最大的权重组合，部署时固定。实验确定的固定权重为 (0.6, 0, 0.4)，即**结构 + 文体学为主，时序权重为零（仅存档为诊断通道）**。
- **聚类后端**：HDBSCAN（用于部署风格，可将低密度会话归为噪声）和固定 K 凝聚聚类（用于诊断对比）。
- **威胁模型五条件**：(a) 相同外部攻击者；(b) 独立注入；(c) 无共享运行时或内存；(d) 异步到达；(e) 代理层可观测性。
- **可选主动探针 CRF‑light**：以低概率暴露语义等价工具别名，旨在观察非自适应 harness 的额外行为特征（主实验未启用）。

### 3. 实验设计
- **主基准 SCD‑v1**：
  - 合成场景：40 个对手角色，4 种代理角色（代码切片用不同生成模型）。包含 **70%良性流量**（来自 AgentDojo/AgentBench）、**15%孤立攻击**、**15%多会话活动**。
  - 生成管理：控制角色、载荷家族、风格、时序和规避。经过泄漏审计（无标签泄漏），但表面文本可恢复风格属性，因此属于“受控”基准，非完全无加工。

  - **原生成对链接器**：直接对工具序列 n‑gram 或提示 TF‑IDF 做余弦相似度。
  - **共享运行时改编**：MAScope‑async（需共享边缘，不符合条件）。
  - **分块 LLM 法官**：将大量会话分块后用强 LLM（GPT‑5.5、Claude Opus 4.7 等）聚类。

### 6. 主要结论与发现
- 在 SCD‑v1 上，**A²FV 的成对 AUC 达 0.82**，而每会话检测器改编版仅约 0.52，分块 LLM 法官接近随机（~0.50），表明代理侧残留可有效链接异步活动。
- **结构残留与文体学残留是主要可分信号**（结构单通道 AUC 0.78，文体学 0.71，两者组合达 0.82），时序通道当前在实验中不提供额外贡献（固定权重为 0），但建议保留为诊断通道。
- 在静态非先知规避（L1/L2）下，成对 AUC 依然保持在 0.80 以上，分区质量有所下降，但**信号未被消除**。
- 风格交叉控制显示：**信号部分与生成风格相关**（硬切片上 AUC 从 0.82 降至 0.60），但结构残留在去除风格敏感成分后仍具区分度（结构单通道硬切片 AUC 0.69），因此不是纯粹的风格伪造。
- 生成器独立再生和原生框架探针证实信号并非单一生成器或特定 harness 的制品，具备一定可移植性。

### 7. 优点
- **问题新颖且实用**：首次将跨代理异步活动归因形式化为安全评估层，贴近真实分布式部署场景。
- **方法简洁、可解释**：A²FV 仅需代理侧日志，无模型训练，特征直观，权重固定，易于部署和审计。
- **基准设计严谨**：SCD‑v1 包含良性、孤立和多会话攻击，进行泄漏审计，控制角色和规避，为后续研究提供可控测试床。
- **评估全面**：对比大量单会话 baseline、LLM 法官，进行多维度消融和压力测试，结论可信度较高。
- **开源倾向**：声明发布制品与复现脚本，有望促进社区验证。

### 8. 不足与局限
- **基准的人为性**：SCD‑v1 是合成生成，角色、样式、模型均由研究者设定，真实攻击的多样性、工具链噪音、运维风格集群等未被覆盖。将结果推广至生产环境仍有风险。
- **文体学信号的风格依赖**：分布审计发现生成风格可恢复，硬切片下信号大幅下降，表明当前归因可能部分在“关联写作风格”而非真正的攻击者身份，在伪装场景下可能失效。
- **没有处理完全自适应对手**：仅测试了静态和非先知组合转换，攻击者若拥有 A²FV 评分反馈则可针对优化，本文未涉及此类 L3 威胁。
- **时序通道未能挖掘价值**：固定权重将时序贡献置零，尽管宣称保留为诊断通道，但未给出有效利用的案例，削弱了“多通道残留”的完整主张。
- **原生框架探针规模小**：仅作为兼容性检查，非大规模基准，单租户、单角色，不能代表生产多租户交叉场景。
- **计算与部署约束**：成对评分窗口内复杂度 O(m²)，需要预过滤和分片；体系未涉及跨组织隐私保护（如联邦计算），实际推广需补充隐私层。
- **缺乏完全开源的生产级数据集**：即使公开 SCD‑v1，仍非真实攻击流量；要真正落地，需要经过隐私审查的带真实活动标签的跟踪数据集。
