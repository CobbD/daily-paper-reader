## 摘要
大型语言模型（LLMs）正越来越多地被部署为专用代理，以处理客户服务和代码生成等特定领域的任务。这些代理不仅要遵守通用的安全护栏，还要遵守为其指定角色量身定制的特定用途限制。这些额外的限制扩大了攻击面，尤其是针对提示注入（PI）攻击。为了防御此类攻击，现有的检测方法主要依赖于分析输入-输出模式，但效果有限。为了解决这一局限，我们转而分析隐藏激活空间，并发现当LLMs被提示超出其指定用途的请求时，它们固有地保留着潜在的策略违反（PV）概念。特别是，PV概念捕捉了用户查询与预定义限制之间冲突的语义，隐含地反映了LLMs识别策略违反的内在意识。基于这一见解，我们提出了PVDetector，一个免训练的框架，通过在LLM推理期间测量隐藏状态与PV概念的对齐程度来检测PI攻击，这些PV概念是从策略违反与策略合规提示的对比对中离线导出的。在多个LLMs和数据集上的实验表明，PVDetector实现了低于1%的假阴性率，且辅助开销极小，始终优于最先进的方法。我们的代码可在 https://github.com/Claresigle/PVDetector 获取。

## Abstract
Large language models (LLMs) are increasingly deployed as purpose-specific agents to handle domain-specific tasks such as customer service and code generation. These agents are expected to comply with not only generic safety guardrails but also purpose-specific restrictions tailored to their designated roles. Such additional restrictions enlarge the attack surface, particularly to prompt injection (PI) attacks. To defend against such attacks, existing detection methods primarily rely on analyzing input-output patterns, yet yield limited effectiveness. To address this limitation, we turn to analyzing the hidden activation space and discover that LLMs inherently retain latent policy-violation (PV) concepts when prompted with requests beyond their designated purpose. Particularly, PV concepts capture the semantics of conflicts between user queries and predefined restrictions, implicitly reflecting LLMs' intrinsic awareness of recognizing policy violations. Building on this insight, we propose PVDetector, a training-free framework that detects PI attacks during LLM inference by measuring hidden-state alignment with PV concepts, which are derived offline from the contrastive pairs of policy-violating and policy-compliant prompts. Experiments across multiple LLMs and datasets show that PVDetector achieves <1\% false negative rate with minimal auxiliary overhead, consistently outperforming state-of-the-art methods. Our code is available at https://github.com/Claresigle/PVDetector .

### 2. 论文提出的方法论

- **核心思想**：通过**对比学习的思想**从 LLM 隐藏状态中提取出“策略违反”方向的向量（即 PV 概念），然后在推理时测量测试输入在该方向上的投影强度，从而区分正常查询与注入攻击。
- **技术框架：PVDetector**（分离线与在线两阶段）：
  - **离线阶段**：
    1. **对比对构造**：为特定代理准备 \(N\) 对正样本（违反策略的查询）和负样本（符合策略的查询），**无需攻击样本**。
    2. **策略违反概念提取**：
       - 将正负样本输入 LLM，提取每层**最后一个 token 位置的隐藏状态**作为语义表示。
       - 对每层，计算正负样本表征的**均值差向量**（`v_MD = (1/N) Σ (h⁺ᵢ − h⁻ᵢ)`），将所有层的向量堆叠为 PV 矩阵 **Vpv**。
       - 这些均值差向量被视为 **PV 向量**，代表了“策略违反”的线性方向。通过反嵌入层可将其映射为语义 token（如 “sorry”, “outside”, “conflict”），验证其可解释性。
    3. **关键层选择**：计算正负样本在每层的 PV 强度得分之差，保留差值不小于最后一层的层作为 **关键层 ℒkey**（深层往往对策略违反更敏感）。
  - **在线阶段（推理时检测）**：
    - 对输入 \(x\)，在关键层取其最后 token 的隐藏状态 hˡ_last，计算其在 PV 向量上的投影得分（L2 正则化后点积），得到逐层 **PVS 分数**。
    - 用**梯形法则**聚合关键层得分，得到总 PV 强度 \(s_{\text{aggr}}\)。
    - 若 \(s_{\text{aggr}}\) 超过预设阈值 \(\theta\)，则判为攻击（阈值基于少量良性样本校准，使 FPR ≤ 1%）。
- **方法特性**：
  - **免训练**：无需训练任何辅助模型，仅需少量对比对离线计算 PV 向量和关键层。
  - **实时高效**：检测过程仅为投影与聚合操作，可在标准 LLM 推理时同步进行。
  - **可扩展至多模态**：同样适用于视觉‑语言模型，检测图像中嵌入的恶意指令。

### 3. 实验设计

### 8. 不足与局限

- **依赖模型白盒访问**：需要获取 LLM 内部各层的隐藏状态，无法直接应用于**黑盒 API 服务**（如 GPT‑4 闭源模型），仅适用于自有部署场景。
- **对比提示依赖**：尽管只需少量样本（30 对），仍需为每个代理**手动定义正负样本**，策略变化后可能需要重新提取，自动化程度有限（迁移性分析一定程度缓解此问题）。
- **阈值敏感性**：FPR 阈值基于少量良性样本校准，当实际运行时查询分布发生**偏移**，可能需要重新校准，未讨论自适应阈值方案。
- **攻击覆盖**：主要测试了 5 种**单轮直接注入**攻击，未涉及**间接注入**（如网页内容污染）、多轮对话攻击或更复杂的语义操纵，对这些变种的检测能力有待验证。
- **评价指标单一**：仅使用 FPR/FNR，未分析检测延迟对用户体验的影响（如误拦截带来的服务中断），也未考虑与内容过滤器的协同兼容性。
- **离线计算假设**：虽然在线推理开销小，但离线构建阶段仍需运行模型前向传播并存储大量激活，对极大模型（如 70B+）的存储和计算可能产生一定负担。
