## 摘要
可重用技能正成为大型语言模型智能体的基础构建块，使能力能够在不同应用中打包、共享和重用。然而，现有的安全研究主要关注提示注入和运行时执行，而整个更广泛的技能生命周期中的安全风险在很大程度上仍未被探索。在本文中，我们提出了 SkillSec-Eval，一个具有生命周期意识的框架，用于系统评估可重用智能体技能的安全性。我们首先刻画了技能生命周期，并开发了一个威胁分类法，涵盖存储库准入、语义检索、规划器选择、执行和技能演化。然后，我们在 SkillSec-Eval 中实例化此分类法，并使用包含 327 个真实世界技能的存储库进行全面实证评估。我们的研究表明，漏洞出现在执行之外的多个生命周期阶段，凸显了对可重用智能体技能进行生命周期感知安全分析的必要性。

## Abstract
Reusable skills are becoming a fundamental building block of Large Language Model (LLM) agents, enabling capabilities to be packaged, shared, and reused across diverse applications. However, existing security research primarily focuses on prompt injection and runtime execution, leaving security risks throughout the broader skill lifecycle largely unexplored. In this paper, we present SkillSec-Eval, a lifecycle-aware framework for systematically evaluating the security of reusable agent skills. We first characterize the skill lifecycle and develop a threat taxonomy spanning repository admission, semantic retrieval, planner selection, execution, and skill evolution. We then instantiate this taxonomy in SkillSec-Eval and conduct a comprehensive empirical evaluation using a repository of 327 real-world skills. Our study demonstrates that vulnerabilities arise at multiple lifecycle stages beyond execution, highlighting the need for lifecycle-aware security analysis of reusable agent skills.

    -   **规划**：伪造推荐、误导性描述、权限欺骗、提示注入。

## 6. 论文的主要结论与发现

-   **全生命周期均存在安全威胁**：漏洞不仅出现在运行时，**存储库准入、语义检索和规划器选择**阶段同样存在严重可利用空间。
-   **语义层防护必不可少**：单纯的结构验证（签名、格式）对元数据欺骗、语义伪装等攻击几乎无效，而引入 LLM 驱动的语义一致性检查（混合防御）能将恶意入库率从 52.9% 降至 **7.9%**。
-   **检索极易被操纵**：基线检索下，女巫攻击（15 克隆）的 ASR 可达 **93.2%**，平均每次检索向规划器暴露 2.84 个恶意技能；检索验证层大幅降低了此类攻击成功率，但对基于向量空间数学特性的触发词投毒防御仍不完美。
-   **规划器易受社交工程式欺骗**：伪造推荐攻击的基线成功率高达 **45.64%**，显示 LLM 规划器对权威暗示高度敏感；元数据清洗后可显著降低此类威胁，但归根结底是阻止恶意元数据进入规划上下文，而非解决模型本身的易受骗性。
-   **运行时是最后一道防线但依然存在盲区**：在无监控的基线中，组合攻击 100% 成功。污点追踪与策略引擎拦截了 87% 的越权调用，但**当攻击者让 LLM 对敏感数据进行摘要或改写时，严格的字符串级污点追踪会失效**（污点准确率仅 66.67%，导致仍有 23% 的攻击成功），揭示了 LLM 内部隐式信息流对传统污点追踪的挑战。
-   **演化阶段呈现安全与更新的二律背反**：持续验证能发现 92.5% 的恶意更新，但同时也因行为向量距离限制而拒绝了 37% 的良性重构，说明在严防回退攻击的同时会显著降低开发者体验。
-   **整体结论**：没有任何单一的防御手段足以保障技能安全，各生命周期阶段的安全机制必须协同工作，构成纵深防御。

## 7. 优点

-   **概念新颖**：首次构建了覆盖技能**完整生命周期**的统一威胁分类与评估框架，将传统软件供应链安全、语义检索攻击、规划器操纵等孤立研究方向桥接在一起。
-   **方法论分层清晰**：将技能生命周期解构为明确的信任边界，攻击与防御均按阶段独立评估，逻辑严密，易于扩展到新出现的框架或市场。
-   **贴近真实生态**：使用真实技能数据集并模拟密集语义竞争环境，避免了过于简化的实验设定；攻击模式（如女巫攻击、关键词堆砌）贴近实际威胁。
-   **实证发现具有洞察力**：揭示了 LLM 规划器对社会工程元数据的高度敏感性，以及运行时污点追踪在应对 LLM 语义转换时的固有不足，这些结论对后续研究有启发意义。
-   **可复现性**：提供了明确的技能表示、prompt 模板和评估流程，方便其他研究者复现或扩展。

## 8. 不足与局限

-   **评估规模受限**：仅基于 327 个技能的受控仓库，未在大规模、多格式、动态更新的真实生产级技能市场中检验，实验结论的泛化能力待进一步验证。
-   **演化评估不充分**：受限于缺乏长期历史数据，仅模拟了短期版本更新，未真正观察长期信任衰减、依赖关系传染和持续隐秘投毒的复杂场景。
-   **防御组件的误报/漏报问题**：混合防御的语义验证在降低恶意入库率的同时，仍有 20% 的误报率；演化防御的误报率高达 37%，未探讨这些误报在实际运营中的具体影响与缓解手段。
-   **LLM 自身威胁未覆盖**：框架专注于技能生态的安全，未涵盖来自底层模型本身的威胁（如模型越狱、对抗微调、用户输入发起的提示注入等），而这些是实际攻击链中的常见环节。
-   **动力模型假设较为理想**：假定底层 LLM、嵌入系统和操作系统均安全无误，攻击者仅通过技能生命周期进行操纵。在真实复杂环境中，多源威胁可能相互耦合。
-   **未公开算力细节**：资源消耗与性能开销未量化，难以评估评估框架在大规模场景下的可行性（如模型 GPU 成本、延迟等）。
-   **指标有限**：部分指标仅针对成功率，未衡量时间开销或攻击隐蔽性（如语义伪装在防御下的检出难度随时间的变化）。
