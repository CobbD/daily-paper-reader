## 摘要
大型语言模型（LLMs）作为网络入侵检测分类器正受到越来越多的探索，但在现实攻击者约束下的对抗鲁棒性仍不清楚。我们提出了一种针对基于 LLM 的网络流量分类器的可控性感知黑盒迁移框架。该框架根据网络通信语义将流特征划分为直接可控（DC）、间接可控（IC）和不可控（UC）组，然后将扰动限制在 DC 特征上，同时冻结 IC/UC 特征。我们使用共享的 XGBoost 替代模型生成有限差分 PGD、贪婪坐标式和 NES 对抗样本，并将它们迁移到七个 LLM 目标和两个传统机器学习目标，这些目标跨越了从 1999 年到 2022 年的五个 IDS 基准数据集。在 27 个有效的 LLM 配置和超过 500,000 个对抗样本上，我们发现 LLM 的迁移脆弱性相当大，但依赖于数据集和比较对象。与 LightGBM 相比，LLM 在 RT-IoT2022 和 CIC-IDS-2018 上更脆弱，在 NSL-KDD 和 UNSW-NB15 上相当，在 HIKARI-2021 上较不脆弱；与平均机器学习基线相比，LLM 在所有五个数据集上显示出更高的攻击成功率（ASR）。我们进一步观察到一致的跨架构迁移层次：在所有 27 个 LLM 单元和 9/10 个 ML 单元中，基于梯度和分数的扰动比贪婪扰动迁移更有效。使用树、神经网络和线性替代模型进行跨替代验证产生了相似的 LLM ASR，减少了研究结果特定于 XGBoost 的证据。约束违反率通过构造为 0%。

## Abstract
Large language models (LLMs) are increasingly explored as network intrusion detection classifiers, but their adversarial robustness under realistic attacker constraints remains unclear. We present a controllability-aware black-box transfer framework for LLM-based network traffic classifiers. The framework partitions flow features into directly controllable (DC), indirectly controllable (IC), and uncontrollable (UC) groups according to network communication semantics, then restricts perturbations to DC features while freezing IC/UC features. Using a shared XGBoost surrogate, we generate finite-difference PGD, greedy coordinate-wise, and NES adversarial examples and transfer them to seven LLM targets and two conventional ML targets across five IDS benchmarks from 1999 to 2022. Across 27 valid LLM configurations and over 500,000 adversarial examples, we find that LLM transfer vulnerability is substantial but dataset- and comparator-dependent. Compared with LightGBM, LLMs are more vulnerable on RT-IoT2022 and CIC-IDS-2018, comparable on NSL-KDD and UNSW-NB15, and less vulnerable on HIKARI-2021; compared with the averaged ML baseline, LLMs show higher ASR on all five datasets. We further observe a consistent cross-architecture transfer hierarchy: gradient- and score-based perturbations transfer more effectively than greedy perturbations across all 27 LLM cells and 9/10 ML cells. Cross-surrogate validation with tree, neural, and linear surrogates yields similar LLM ASR, reducing evidence that the findings are XGBoost-specific. Constraint violation rate is 0\% by construction.

## 2. 论文提出的方法论

- **核心思想**：将网络流特征按照通信语义划分为**直接可控**（DC，如源字节）、**间接可控**（IC，如连接率）和**不可控**（UC，如目标字节）三类，仅对 DC 特征施加扰动，冻结 IC/UC 特征，从而模拟攻击者有限的操控能力。
- **关键技术细节**：
  - **攻击方**：使用共享的 XGBoost 替代模型（训练于公开数据）生成对抗样本，并迁移到无访问权限的目标 LLM 或 ML 模型。
  - **约束空间**：在归一化特征空间中，限制扰动仅作用于 DC 特征的 L∞ 范数内，且不改变 IC/UC 特征，约束违反率（CVR）通过构造为 0%。
  - **扰动算法**（均仅查询替代模型）：
    - **FD-PGD**（梯度法）：利用有限差分估计替代模型梯度，进行投影梯度下降（20 步，步长 α = 2.5ε/n_steps）。
    - **Greedy**（贪婪法）：每次选择使替代模型损失增加最大的单个 DC 特征进行扰动（10 步，步长 ε/2）。
    - **NES**（分数法）：使用自然进化策略，通过反制对抽样估计替代模型输出的梯度（20 个查询/步，10 步，平滑 σ=0.01）。
  - **跨架构迁移层次**：预测基于梯度和分数的攻击比贪婪攻击具有更高的跨架构迁移性，并提出用预测一致性 ρ（替代模型与目标的标签一致率）作为架构相似度代理，解释迁移差异。
- **攻击流程**：对每条良性样本 x，在替代模型上按约束生成 x’，然后直接将其序列化为文本提示送入 LLM 进行推理，记录分类变化。

## 6. 论文的主要结论与发现

- **LLM 迁移脆弱性显著但数据集/比较对象依赖**：
  - 与 LightGBM 相比：LLM 在 CIC-IDS-2018 (1.4×) 和 RT-IoT2022 (7.2×) 上更脆弱；在 NSL-KDD 和 UNSW-NB15 上相当；在 HIKARI-2021 上反而更鲁棒（LightGBM 脆弱性 1.43× 更高）。
  - 与平均 ML 基线（LightGBM + DNN-IDS）相比，LLM 在所有 5 个数据集上均显示出更高的 ASR。
- **跨架构迁移层次**：在所有 27 个 LLM 单元和 9/10 个 ML 单元中，基于梯度（FD-PGD）和分数（NES）的攻击的迁移效果显著优于贪婪坐标式攻击，该层次与替代-目标架构相似度（预测一致性 ρ）相关，ρ 接近 1（如树→树）时层次消失，ρ 较低时层次强烈。
- **方法稳健性**：使用不同架构的替代模型（树、神经网络、线性模型）均能产生有效的迁移攻击，排除了结论唯一依赖于 XGBoost 的可能性。
- **防御局限性**：对 LightGBM 有效的特征压缩和对抗训练，直接在 LLM 上应用效果较差甚至适得其反，表明防御也需适配目标架构。
- **评估建议**：评估 LLM-IDS 鲁棒性应基于可控性约束、解耦目标架构，并避免使用平均 ML 基线来替代针对性比较。

## 8. 不足与局限

- **约束模型非完整流量级**：DC/IC/UC 划分仅保证了特征级可控性，未验证整数型、类别型约束及特征间依赖，也未保证攻击功能保持（即修改后的流量仍保持恶意行为），因此尚不能等同于完全可回放的流量攻击。
- **特征分类的主观性**：某些特征（如持续时间）的归属可能因部署环境而异，分类依赖专家判断，可能影响某些数据集上的结论一致性（例如 HIKARI-2021 的相反表现）。
- **机制解释未完全验证**：数据集依赖的脆弱性差异（如 HIKARI 上的回调）归因于 tokenization 或注意力机制的解释仍为假设，缺乏通过匹配的表格 Transformer 与文本序列化 LLM 的严格消融实验加以证实。
- **防御分析不全面**：防御实验仅涉及 2 个 LLM 配置，未深入探讨 LLM 原生防御方法（如对抗微调、提示工程），结论的普适性有限。
- **算力细节缺失**：未说明训练和推理使用的硬件资源，复现时可能难以估计实际开销。
- **跨替代迁移的覆盖度**：虽验证了三种替代模型，但在更大的 LLM 和数据集组合上的验证仍可扩展。
