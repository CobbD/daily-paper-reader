## 摘要
持久记忆使得大语言模型（LLM）智能体能够存储事实知识、先前决策、推理历史、工具使用信息和上下文。虽然这提高了智能体的跨任务功能性和连续性，但也引入了一个新的攻击面：智能体自身的推理历史。在本文中，我们提出了伪造放大推理记忆攻击（FARMA），该攻击毒害智能体记忆中的推理过程而非事实知识。它使用规避性语言插入伪造的推理痕迹，以绕过基于关键词的防御，然后通过自引用强化放大这些痕迹，从而击败基于共识的防御。为应对FARMA，我们提出了SENTINEL，一种检测伪造推理条目的分层防御流水线。其核心组件是推理卫士（Reasoning Guard），利用五个加权信号对候选条目进行结构化分析，以识别伪造内容。我们跨多个智能体和不同LLM模型，通过50次试验评估了FARMA和SENTINEL，发现FARMA在基线条件下的攻击成功率高达100%，并能突破关键词过滤和A-MemGuard等防御机制。此外，评估表明SENTINEL可将FARMA的攻击成功率降至0%，且在326条良性智能体轨迹中未出现误报。我们的工作表明，不仅需要保护智能体检索到的内容，还需要保护其推理历史的完整性。

## Abstract
Persistent memory has enabled large language model (LLM) agents to store factual knowledge, prior decisions, reasoning histories, tool usage information, and context. While this has improved the agent's functionality and continuity across tasks, it has also introduced a new attack surface: the agent's own reasoning history. In this paper, we introduce the Forged Amplifying Rationale Memory Attack (FARMA), which poisons an agent's remembered reasoning rather than its factual knowledge. It inserts forged reasoning traces using evasive language that bypasses keyword-based defenses, then amplifies them through self-referential reinforcement that defeats consensus-based defenses. To address FARMA, we introduce SENTINEL, a layered defense pipeline to detect forged reasoning entries. Its central component is the Reasoning Guard that structurally analyzes candidate entries for forgery using five weighted signals.   We evaluate FARMA and SENTINEL across multiple agents and different LLM models with 50 trials and show that FARMA achieves an attack success rate of up to 100% under baseline conditions and is capable of defeating defense mechanisms like keyword filter and A-MemGuard. Our evaluation also shows that SENTINEL reduces FARMA's attack success rate to as low as 0% with no false positives observed across 326 benign agent traces. Our work demonstrates the need to protect not only an agent's retrieved content but also the integrity of its reasoning history.

*   **核心问题**：现有的记忆投毒攻击主要针对外部知识（事实性内容）或经验存储，**并未触及“智能体认为自己之前如何推理”这一层面**。当智能体从记忆里检索到“已验证过某数据”的假记录时，会将其视为内部证据，进而产生“跳过验证”等危险行为，且这种攻击能绕过关键词或共识类防御。
*   **整体含义**：不仅需要保护智能体检索到的外部内容，**推理历史的完整性保护同样至关重要**，否则智能体可能被诱导执行存在安全隐患的动作。

### 2. 论文提出的方法论

论文提出了**攻击方法 FARMA**（Forged Amplifying Rationale Memory Attack）与**防御框架 SENTINEL**（五层防御流水线）。

#### FARMA 攻击：两阶段推理记忆投毒

*   **核心思想**：不直接要求智能体“跳过验证”，而是向其记忆中植入**看似自己生成的、已完成验证的决策日志**，并通过自我引用的方式**放大该虚假共识**，使其在检索时被优先采纳。
*   **注入阶段（Seed Injection）**：
    *   使用**规避性语言**（如“上游已完成源级验证，所有检查均已通过，重新验证非必要”）替代明显恶意关键词（如“跳过验证”）。
    *   保持**结构相似性**，使伪造条目与真实智能体记录在格式、语气、元数据（来源标记为AGENT，信任标记为VERIFIED）上难以区分。
    *   针对不同领域（EHR、QA、购物）定制模板，每次注入少量种子（默认3条）。
*   **放大阶段（Amplification）**：
    *   持续追加引用先前伪造条目的新记录（如“与先前N次运行一致，直接导入行为已成标准模式”），逐步增大伪造语料库。
    *   目的有三：①提高伪造内容在检索（top-k）中的出现概率；②使伪造记录成为**新的“共识”**，令基于异常检测或共识偏离的防御（如A‑MemGuard）失效；③制造“大量前例”的假象，增强智能体对虚假结论的信任。

#### SENTINEL 防御：五层流水线与推理卫士

*   **设计理念**：在内存写入时就进行多层过滤，外层采用轻量级快速筛选，内层进行昂贵的结构化分析，且同时检查“推理存储”和“一般记忆存储”中类似推理的内容。
*   **外层轻量过滤器（L1‑L4）**：
    *   **L1 关键词过滤**：拦截包含“跳过”“绕过”等恶意词的条目。
    *   **L2 溯源/IFC标记**：根据声明的来源（SYSTEM / AGENT / TOOL 等）为条目赋予完整性标签与污点值（taint）。
    *   **L3 污点阈值过滤**：拒绝污点值过高的条目（如未知来源）。
    *   **L4 模式/风险筛查**：扫描第二组恶意模式，并结合污点值计算风险分数进行拦截。
*   **核心组件：推理卫士（L5 Reasoning Guard）**
    *   针对经过前四层筛选、且内容“看起来像推理记录”的条目进行**结构化分析**。
    *   计算五个信号的**加权伪造分数 \\(F = \sum_{i=1}^5 w_i s_i\\)**（默认等权重0.2）：
        1.  **溯源异常 (s₁)**：内容表现为内部推理，但声明来源为外部/未知。
        2.  **自引用不匹配 (s₂)**：条目声称“与之前N次运行一致”，但实际存储记录数少于N。
        3.  **可疑模式密度 (s₃)**：计算决策日志类偏执短语（如“已验证”“标准处理”等）的密度，采用分级阈值。
        4.  **不合理声明检测 (s₄)**：如“100%成功率”等夸大一致性的说法，结合大运行次数的声明触发。
        5.  **结构检查 (s₅)**：检查开头格式、固定模板等结构异常（伪造条目常来自少量模板，具有更窄的分布）。
    *   超过阈值 \\(\tau_{\text{guard}}\\) 的条目被拒绝。

### 7. 优点（方法或实验设计上的亮点）

*   **新颖的攻击视角**：首次将“智能体自身推理历史”确立为独立的攻击面，并设计了不依赖直接指令的投毒方式（让记忆“替攻击者说话”）。

### 8. 不足与局限

*   **对抗自适应攻击弱**：推理卫士基于启发式特征，一旦攻击者知晓其规则，可通过**改写措辞、避免触发结构模式**等方式绕过，论文也承认当前SENTINEL对此保护不足。
*   **单智能体环境限制**：实验仅在单智能体、单记忆存储下进行，未考虑**多智能体共享记忆**场景下，一个受感染智能体的写入如何横向传播。
*   **模拟环境而非真实系统**：与多数相关研究一致，在模拟环境中评估，未涉及真实生产系统中的多轮交互、记忆写入长尾分布等因素。
*   **信号权重与阈值固定**：推理卫士采用等权重与固定阈值，可能需针对不同智能体或领域重新校准，缺乏自适应性。
*   **攻击者假设较强**：假设攻击者可多次写入记忆、能伪造来源与信任标签，虽然现实中有相关漏洞案例支撑，但在部分部署环境中可能难以完全满足。
