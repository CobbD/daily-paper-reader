## 摘要
大型语言模型（LLM）作为网络入侵检测分类器的探索日益增多，但在现实攻击者约束下的对抗鲁棒性尚不明确。我们提出了一种面向基于LLM的网络流量分类器的可控性感知黑盒迁移框架。该框架根据网络通信语义将流特征分为直接可控（DC）、间接可控（IC）和不可控（UC）三组，然后限制仅对DC特征进行扰动，而冻结IC/UC特征。使用共享的XGBoost替代模型，我们生成了有限差分PGD、贪心逐坐标和NES对抗样本，并将其迁移到七个LLM目标和两个传统ML目标上，在五个从1999年到2022年的IDS基准数据集上进行测试。在27个有效的LLM配置和超过50万个对抗样本上，我们发现LLM的迁移脆弱性相当大，但依赖于数据集和比较对象。与LightGBM相比，LLM在RT-IoT2022和CIC-IDS-2018上更脆弱，在NSL-KDD和UNSW-NB15上相当，在HIKARI-2021上更不脆弱；与平均ML基线相比，LLM在所有五个数据集上表现出更高的攻击成功率（ASR）。我们进一步观察到一致的跨架构迁移层次：在所有27个LLM单元和9/10个ML单元中，基于梯度和得分的扰动比贪心扰动迁移更有效。使用树、神经网络和线性替代模型进行交叉替代验证，获得了相似的LLM ASR，减少了这些发现是XGBoost特有的证据。约束违反率通过构造为0%。

## Abstract
Large language models (LLMs) are increasingly explored as network intrusion detection classifiers, but their adversarial robustness under realistic attacker constraints remains unclear. We present a controllability-aware black-box transfer framework for LLM-based network traffic classifiers. The framework partitions flow features into directly controllable (DC), indirectly controllable (IC), and uncontrollable (UC) groups according to network communication semantics, then restricts perturbations to DC features while freezing IC/UC features. Using a shared XGBoost surrogate, we generate finite-difference PGD, greedy coordinate-wise, and NES adversarial examples and transfer them to seven LLM targets and two conventional ML targets across five IDS benchmarks from 1999 to 2022. Across 27 valid LLM configurations and over 500,000 adversarial examples, we find that LLM transfer vulnerability is substantial but dataset- and comparator-dependent. Compared with LightGBM, LLMs are more vulnerable on RT-IoT2022 and CIC-IDS-2018, comparable on NSL-KDD and UNSW-NB15, and less vulnerable on HIKARI-2021; compared with the averaged ML baseline, LLMs show higher ASR on all five datasets. We further observe a consistent cross-architecture transfer hierarchy: gradient- and score-based perturbations transfer more effectively than greedy perturbations across all 27 LLM cells and 9/10 ML cells. Cross-surrogate validation with tree, neural, and linear surrogates yields similar LLM ASR, reducing evidence that the findings are XGBoost-specific. Constraint violation rate is 0\% by construction.

### 2. 论文提出的方法论
*   **核心思想**：根据网络通信语义，将流量特征向量划分为三组：
    *   **直接可控（DC）**：攻击者端点可直接设定的特征（如源字节数、发包数）。
    *   **间接可控（IC）**：攻击者可通过持续行为间接影响的聚合统计量（如连接速率）。
    *   **不可控（UC）**：由远端、网络路径或监控系统决定的特征（如目的字节数、响应延迟）。
    对抗扰动**严格限制在 DC 特征子空间内**，IC 和 UC 特征被冻结，从而生成更符合攻击者能力的“现实”对抗样本。
*   **关键技术细节**：
    *   **威胁模型**：黑盒迁移设定。攻击者仅能基于公开数据训练一个本地替代模型（默认 XGBoost），生成对抗样本后直接迁移至目标模型，**不得查询目标模型**。
    *   **约束空间定义**：\( \mathcal{S}(\mathbf{x}, \varepsilon) = \{\mathbf{x}' \in [0,1]^d \mid \|\mathbf{x}'_{I_{DC}} - \mathbf{x}_{I_{DC}}\|_\infty \le \varepsilon, \mathbf{x}'_{I_{IC}} = \mathbf{x}_{I_{IC}}, \mathbf{x}'_{I_{UC}} = \mathbf{x}_{I_{UC}}\} \)，其中 \(\varepsilon\) 为 \(L_\infty\) 扰动预算。
    *   **扰动生成算法**（均仅在替代模型上运算）：
        *   **有限差分 PGD**：通过前向有限差分估计梯度（\(\hat{g}_j = \frac{\mathcal{L}(f_S, \mathbf{x} + \delta \mathbf{e}_j, y) - \mathcal{L}(f_S, \mathbf{x}, y)}{\delta}\)），然后沿梯度符号方向更新 DC 特征。
        *   **贪心逐坐标**：每次迭代选出使替代模型损失上升最大的单个 DC 特征进行扰动。
        *   **NES**：采用自然进化策略，通过对偶采样估计替代模型输出的梯度方向。
*   **公式/算法流程**：算法 1 总结了端到端流程：训练 XGBoost 替代模型 → 对每个正确分类的样本，使用指定算法在 DC 特征上生成扰动 → 冻结 IC/UC 特征并裁剪 → 统计攻击成功率（ASR）。

    *   **攻击算法**：有限差分 PGD、贪心逐坐标、NES 三者相互对比。
    *   **对抗性基线**：在相同约束下对比 LLM 与 LightGBM、LLM 与 DNN‑IDS，以及 LLM 与两者的**平均 ML 基线**。

    *   **防御评估**：初步测试了特征压缩和对抗训练对 LightGBM 及部分 LLM 的效果。

### 5. 实验数量与充分性
*   **规模巨大**：涵盖 5 个数据集、7 种 LLM + 2 种 ML、3 种攻击算法、5 个 \(\varepsilon\) 预算（0.01–0.50）、5 个随机种子，累计超过 **50 万**个对抗样本，共 **27 个有效 LLM 评估单元格**。
*   **充分性与公平性**：
    *   通过统一替代模型和固定约束空间，LLM 与 ML 在 **完全相同的对抗样本** 上比较，控制了变量。
    *   设置了多层消融：替换替代模型架构（树/神经网络/线性）、不同扰动预算、按分类器能力（F1 ≥ 0.60, 0.70）分层分析、提示格式（KV/JSON）消融，并进行了统计显著性检验（Mann‑Whitney U、数据集级配对 Wilcoxon）。

### 6. 论文的主要结论与发现
*   **脆弱性并非普适**：LLM 的迁移脆弱性**强烈依赖于数据集和对比的 ML 基准**。
    *   与 LightGBM 相比：在 RT‑IoT2022（高 7.2 倍）和 CIC‑IDS‑2018（高 1.4 倍）上更脆弱；在 NSL‑KDD、UNSW‑NB15 上**相当**；在 HIKARI‑2021 上**更不脆弱**。
    *   若与平均 ML 基线（LightGBM + DNN‑IDS）比较，LLM 在所有五个数据集上 ASR 均更高，但这部分是由于 XGBoost→DNN 迁移极弱造成的伪影。
*   **跨架构迁移层次**：**基于梯度和分数的扰动（PGD, NES）迁移成功率远超贪心逐坐标扰动**。这一规律在 27/27 个 LLM 单元格和 9/10 个 ML 单元格中成立。唯一例外（CIC‑IDS‑2018 上 XGBoost→LightGBM）出现时替代模型与目标模型预测一致性极高（\(\rho = 0.98\)），解释了贪心攻击的特例。
*   **结果稳健性**：使用 MLP、逻辑回归作为替代模型得到的 LLM 攻击成功率与 XGBoost 相近，否定了结果仅与 XGBoost 特定相关的假设。
*   **防御差异**：特征压缩对 LightGBM 有效，但对文本序列化后的 LLM 防御效果微乎其微甚至反增，表明防御不能直接从树模型迁移到 LLM。

### 7. 优点
*   **威胁模型现实性**：首次将“可控性”形式化为 DC/IC/UC 特征分区，强制扰动仅在攻击者物理可操控的范围内，比无约束特征攻击更贴近真实网络逃避场景。
*   **公平的对比框架**：所有目标（LLM 或 ML）均面对**同一个替代模型生成且经同一可控性掩码处理**的对抗样本，排除了攻击条件差异带来的混淆。
*   **发现新颖**：系统性揭示了 LLM 与传统 ML 在迁移脆弱性上的“对手依赖”性，并提出了由架构相似性驱动的**迁移攻击层次**。
*   **实验严谨性高**：多数据集、多模型、多攻击算法、多替代模型、多预算的全面验证，并辅以假设检验，信度较高。

### 8. 不足与局限
*   **问题空间完备性不足**：约束仅实现了**特征组掩码**，未保证全数据包级别可重放性。缺乏对整数类型强制、特征间依赖性（如包长与总字节关系）和攻击功能保持的验证，因此仍是“特征空间”对抗样本的增强版。
*   **LLM 防御研究薄弱**：特征压缩和对抗训练仅初步测试，且仅在个别模型/数据集上进行，未能为 LLM‑IDS 提供实用的防御方案。
*   **根因未完全隔离**：论文猜测 LLM 在某些数据集上的高脆弱性可能源于 Token 化放大或全局注意力传播，但未通过“表格化 Transformer（相同架构但直接输入数值）”与“文本序列化 LLM”的对照实验将序列化效应与架构效应解耦。
*   **分析局限**：HIKARI‑2021 的反转（LLM 更稳健）仅给出可能解释，未严格实证其机制；特征划分依赖专家知识，通用性可能受数据集影响。
