## 摘要
本文针对用于战场物联网(IoBT)任务控制的策略感知LLM检索增强生成(PA-LLM-RAG)框架进行对抗安全研究。我们提出一种查询无关的语义检索投毒攻击，该攻击将语义构造的规则注入IoBT知识库，在所有操作员查询类型中取得高检索排名，且无需了解运行时提示。通过注入单条规则（投毒率1.6%），该攻击可实现85%的LLM上下文污染，并在投毒率7.7%时达到饱和，表明即使极小的知识库受损也足以破坏任务决策。为应对这一威胁，我们提出了CLD-KB（知识库的网络分层防御），一个双重检测异常检测框架，该框架结合了单类SVM边界检测和一种新颖的基于成员的类别散布分析，利用三类IoBT策略分类法在有毒规则到达决策LLM之前将其识别。在投毒检测和知识保留方面，CLD-KB显著优于DBSCAN、LOF、K-Means、孤立森林和单类SVM等五种基线方法，且每次任务仅增加7毫秒的计算开销，使其成为LLM驱动的IoBT任务系统的有效且可边缘部署的防御方案。

## Abstract
This paper presents an adversarial security study of the Policy-Aware LLM Retrieval-Augmented Generation (PA-LLM-RAG) framework for Internet of Battlefield Things (IoBT) mission control. We propose Query-Agnostic Semantic Retrieval Poisoning, a novel attack that injects semantically crafted rules into the IoBT knowledge base achieving high retrieval ranking across all operator query types without requiring knowledge of runtime prompts. The attack achieves 85% LLM context corruption from a single injected rule (1.6% poisoning rate) and saturates at 7.7% poisoning, demonstrating that even minimal knowledge base compromise is sufficient to corrupt mission decisions. To counter this threat, we propose CLD-KB (Cyber-Layered Defense for Knowledge Base), a dual-detector anomaly detection framework combining One-Class SVM boundary detection with a novel Member-Based Category Spread analysis that exploits the three-category IoBT policy taxonomy to identify poisoned rules before they reach the decision LLM. CLD-KB significantly outperforms five baseline methods including DBSCAN, LOF, K-Means, Isolation Forest, and One-Class SVM in both poisoning detection and knowledge preservation, with only 7ms computational overhead per mission, establishing it as an effective and edge-deployable defense for LLM-driven IoBT mission systems.

### 2. 论文提出的方法论
- **攻击方法：查询无关语义检索投毒（Query‑Agnostic Semantic Retrieval Poisoning）**
  - **核心思想**：攻击者无需获知运行时刻的操作员查询，而是构造包含全任务通用词汇（如 gate, drone, UAV, perimeter 等）的恶意规则，使其嵌入向量接近查询分布的中心，从而在所有查询中都能被检索到并排进 top‑k 上下文。
  - **技术细节**：
    - 利用嵌入模型 `nomic-embed-text`，以余弦相似度检索。
    - 恶意规则融合三类策略（工作流 WF、交战规则 ROE、能力 CAP）的术语，使其嵌入向量与各策略类别质心等距，同时对各类查询保持高相似度。
    - 不需要修改现有规则，只需向 KB 追加少量规则即可生效。
- **防御方法：CLD‑KB（Cyber‑Layered Defense for Knowledge Base）**
  - **双检测器架构**：
    - **检测器 1：单类 SVM（OC‑SVM）**  
      用合法规则嵌入训练一个 RBF 核的 OC‑SVM，学习正例边界；任何位于边界外的候选规则被判定为异常。
    - **检测器 2：基于成员的类别散布分析（Member‑Based Category Spread）**  
      利用 IoBT 策略三类结构：计算候选规则到每类内各成员的最小余弦距离，再求最大最小距离差（spread）。合法规则 spread >0.20，恶意规则 spread <0.15，由此分离。
  - **判决逻辑**：任一检测器报警即视为恶意，实行过滤‑补录（ret->10 候选取 top‑3 干净规则）和自适应黑名单，被拦截规则从 KB 中永久移除，实现知识库自愈。
  - **领域特性利用**：仅使用策略分类结构，无需额外 LLM 调用，插入开销极低。

### 6. 论文的主要结论与发现
- 攻击极其高效：仅注入 1 条规则（投毒率 1.6%）即可使 85% 的任务上下文被污染；投毒率 7.7% 时饱和，平均 2.65 条恶意规则进入上下文，此后继续加毒收益甚微。
- 传统异常检测（DBSCAN、LOF、K‑Means、孤立森林）无法完全检测此类毒化，因恶意嵌入混合了多类词汇，位于正例分布的中心而非异常区域。

- 防御引入的延迟仅为单次任务 7ms，完全满足边缘实时决策需求；自适应黑名单机制进一步赋予系统自恢复能力。
