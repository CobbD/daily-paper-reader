## 摘要
自主网络代理有望自动化日常浏览任务，但也继承了网络最古老的攻击面之一。跨站脚本（Cross-Site Scripting）已证明，即使在良性页面上，混合可信与不可信内容依然危险。代理通过将自然语言解释为指令，重新浮现了这种风险，使得第三方和用户生成的内容能够通过提示注入劫持代理。核心挑战在于，推导特定任务的安全策略需要对页面结构进行推理，而页面结构又与攻击者内容纠缠在一起。我们提出了Prismata，这是一种为网络代理实施上下文最小权限的防御措施，限制代理所见和所能操作的内容。Prismata的动态信任推导为页面内容生成权限标签，并提供结构限制保证，其灵感来源于经典的完整性模型，可以约束任何标签错误，使得标签权限只能降低且错误标签范围有限。Prismata的机制限制通过编辑内容和限制代理能力来执行这些标签。重要的是，这些机制无需开发者标注，因此Prismata支持大量长尾网站。在最近发布的网络代理攻击中，包括自适应变体，Prismata显著降低了攻击成功率，同时保持了良性任务的效用。

## Abstract
Autonomous web agents promise to automate everyday browsing tasks, but inherit one of the web's oldest attack surfaces. Cross-Site Scripting proved that mixing trusted and untrusted content is dangerous, even on benign pages. Agents resurface this risk by interpreting natural language as instructions, allowing third-party and user-generated content to hijack the agent via prompt injection. The core challenge is that deriving a task-specific security policy requires reasoning over page structure that is entangled with the attacker's content.   We present Prismata, a defense enforcing contextual least privilege for web agents, constraining both what the agent sees and what it can do. Prismata's dynamic trust derivation produces permission labels for page content, with structural confinement guarantees, inspired by classical integrity models, that bound any labeling errors so that labels can only decrease in privilege and mislabelings are bounded. Prismata's mechanical confinement enforces these labels by redacting content and restricting agent capabilities. Importantly, these mechanisms require no developer annotations, so Prismata supports the long tail of websites. Across recent published web agent attacks, including adaptive variants, Prismata substantially reduces attack success while preserving benign task utility.

- **核心挑战**：确定某个页面操作的安全含义需要解析页面结构，但页面结构本身又与不可信内容纠缠在一起（**Web Entanglement Problem**）。现有防御要么依赖模型层面抵抗（容易被自适应攻击绕过），要么需要开发者预先构建站点地图，无法扩展至海量网站。
- **目标**：为网络代理实现**上下文最小权限**防御，约束代理所见和所能操作的内容，且无需任何开发者标注，支持长尾网站。

## 2. 论文提出的方法论
### 2.1 核心思想
- **动态信任推导 + 机制限制**：从DOM透明抓取权限标签，通过模仿经典完整性模型（Biba模型）的结构限制，确保标签权限只降不升，错误标签影响范围局部化；再通过确定性机制限制编辑内容、约束代理能力。
- **关键部件**：
  - **动作门（Action Gate）**：对每个可交互元素的“关键路径”（从DOM根到该元素的祖先链）独立评估，判断其是否在任务范围内。若攻击内容不在此路径上，则评估不受攻击影响（**注入无染**）。
  - **Biba解析（Biba Parsing）**：沿关键路径递归分类内容来源（开发者、用户、托管方、外部）。借鉴Biba完整性模型的**不可向下读、不可向上写**原则，逐层锁定标签，一旦识别出结构线索（如标题“Reviews”、aria-label、class名），就将整棵子树标记为不可信，并赋予受限能力（如只读）。
  - **安全策略模型**：仅根据任务描述决定需要哪些内容来源，不暴露DOM内容，从而避免攻击者影响策略。
  - **执行层（Mechanical Confinement）**：根据标签和能力图，从代理观察中移除越权内容，降级或禁用越权操作，并在代理发出工具调用时拦截检查，确保操作不超出许可。

### 2.2 安全机制的形式化说明
- 定义元素能力 `cap(e) = min(cap_gate(e), cap_biba(e))`，只要任一机制能在不受攻击影响下做出决定，标签即为“注入无染”。
- **三种场景分析**：
  - **Case 1**：攻击内容不在关键路径上 → 动作门决定不受影响。
  - **Case 2**：攻击内容在关键路径上，但存在前置结构线索 → Biba解析在暴露攻击内容前锁定标签。
  - **Case 3**：攻击内容在关键路径上且无前置结构线索 → 既有机制均可能受影响，但经验数据显示此类情形极罕见（<0.1%路径）。
- 执行层为确定性代码，不依赖LLM推理，因此**不可绕过**。

### 2.3 实现架构
- Prismata处于代理与浏览器之间（基于BrowserGym框架），拦截DOM，经处理后将受限的观察（基于无障碍树）传递给代理，并在代理行动时进行拦截。
- 流程：`Action gate` → `Biba parsing` → `Enforcement`。处理后的观察中，越权节点被移除或降级。

## 7. 优点
- **无需开发者标注**：自动从页面结构推导信任标签，支持任意网站。
- **结构化安全保证**：借鉴经典完整性模型，提供可解释的局限性分析，而不是纯经验性防御。
- **攻击与防御解耦设计**：采用动作门和Biba解析双保险，即使一处受攻击影响，另一处仍可能提供保护。
- **低开销**：通过缓存，成本增量小，且不影响代理的基本可用性。
- **模块化评估框架**：提供的评估框架可扩展新攻击和防御，促进后续研究。

## 8. 不足与局限
- **模型依赖**：信任推导依赖底层LLM的准确性，虽然精度高，但仍可能偶发标签错误；在低资源模型上可能精度下降。
- **不完全覆盖**：无法防御完全落在任务权限范围内的攻击（如读取评论时被虚假评论误导），这类攻击需模型层面的推理能力。
- **模态限制**：仅处理文本输入，未扩展至图像、语音等多模态攻击。
- **环境假设**：假设网站已实施XSS防护，且攻击者无法执行任意JavaScript（否则形成传统XSS）。此外，依赖DOM结构信息（如class名），若网站未遵循良好实践（如缺乏语义标签），可能影响标签准确性。
- **实验代表性**：仅在WebArena上评估，网站数量有限，且对抗测试只包含Pop-up类攻击，未覆盖更复杂的诱导（如多步对话劫持）。
- **状态漂移**：虽然Prismata过滤的是捕获观察，但代理与真实页面仍存在固有漂移，可能影响某些实时性强的任务。
