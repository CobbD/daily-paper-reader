## 摘要
个人AI代理在用户的本地机器上运行，例如OpenClaw，可以自动化日常任务，包括网络搜索、电子邮件和文件管理。它们对计算机资源（包括网络、文件系统和shell）的访问使它们暴露于间接提示注入（IPI）攻击之下。先前的双LLM防御通过用代理可以引用但无法读取的符号替换不可信数据来阻止IPI。然而，它们仅在代理的上下文中跟踪不可信数据，因此当代理保存并稍后重新读取不可信数据时，该数据（可能是攻击者的提示）可以作为可信数据而不是符号返回，我们称之为存储IPI。在由人类和程序共享的用户真实环境中操作，正是使像OpenClaw这样的代理实用的原因，也正因如此，忽略这一点的防御是不完整的。在这样的环境中保留符号是困难的，因为人类和程序需要原始数据。我们提出了DualView，它通过为每个通道提供两个视图，将不可信数据跟踪从代理的上下文扩展到用户的环境，包括文件系统、shell、网络和其他代理。在AgentView中，代理即使在写入和读回之后，也将不可信数据视为符号，从而阻止存储IPI，而HumanView为人类和工具保留原始数据。DualView将每个工具调用路由到正确的视图，并在两个视图之间同步数据。DualView作为OpenClaw插件部署，仅使用工具钩子，而无需更改代理的工具调用逻辑或工具实现。由于DualView通过设计隔离不可信数据，其保护不限于已知的攻击模板。在我们对IPI基准测试和PinchBench的评估中，DualView阻止了每一次IPI攻击，包括存储IPI，同时保持了接近未受保护基线的效用。

## Abstract
Personal AI agents that run on the user's local machine, such as OpenClaw, automate daily tasks including web search, email, and file management. Their access to computer resources, including the network, file system, and shell, exposes them to indirect prompt injection (IPI) attacks. Prior Dual LLM defenses block IPI by replacing untrusted data with symbols that the agent can reference but not read. However, they track untrusted data only inside the agent's context, so when the agent saves and later rereads untrusted data, that data, possibly an attacker's prompt, can return as trusted data rather than as a symbol, which we call stored IPI.   Operating on the user's real environment, which humans and programs share, is what makes agents like OpenClaw practical, and is exactly why a defense that ignores it is incomplete. Preserving symbols in such an environment is hard, because humans and programs need original data. We present DualView, which extends untrusted data tracking from the agent's context to the user's environment, including the file system, shell, network, and other agents, by giving each channel two views. In AgentView, the agent sees untrusted data as symbols even after writing it out and reading it back, blocking stored IPI, while HumanView preserves original data for humans and tools. DualView routes each tool call to the right view and synchronizes data across the two views. DualView deploys as an OpenClaw plugin using only tool hooks, without changing the agent's tool-call logic or tool implementations. Since DualView isolates untrusted data by design, its protection is not limited to known attack templates. In our evaluation on an IPI benchmark and PinchBench, DualView blocked every IPI attack, including stored IPI, while keeping utility close to the unprotected baseline.

-   **整体含义**：现有防御在“安全性”、“代理本身效用”和“人类用户与环境程序效用”三者之间存在设计缺陷，无法同时兼顾。DualView 旨在扩展数据追踪的边界，从代理上下文延伸至整个用户计算环境，以**彻底阻断存储型IPI，同时保持代理和人类用户的效用不受影响**。

### 2. 论文提出的方法论：核心思想、关键技术细节与流程
DualView 的核心思想是为用户环境提供**双重视图（Dual Views）**，并依据操作类型进行路由和数据同步。

-   **双重视图机制**：
    *   **AgentView（代理视图）**：提供给AI代理的受保护视图。在此视图中，所有不可信数据（无论是从网络首次接收，还是从文件重新读回）都保持为**符号（Symbols）**。攻击者的文本永远不会以原始形式抵达决策层的大型语言模型，从而阻断即时和存储型IPI。
    *   **HumanView（人类视图）**：提供给人类用户和非代理程序的视图。在此视图中，所有数据恢复为**原始内容**，确保用户、第三方工具及远程端点能正常读写、使用代理生成的结果。
-   **工具路由与数据同步**：
    *   **工具路由**：框架将AI代理的工具调用路由至合适的视图。**文件读写、本地局部Shell命令**等能直接处理符号的工具，路由至`AgentView`；**网络请求、需原始数据的Shell命令**等需要真实数据的工具，路由至`HumanView`。路由至HumanView的工具，其输入输出会经历**去符号化**和**重新符号化**的过程。
    *   **视图同步**：在执行文件或Shell工具前，系统首先将HumanView中的人类编辑同步到AgentView；工具执行后，再将AgentView中代理的改动同步回HumanView，并在此过程中将符号恢复为原始数据。这样，人类和代理看到同一个文件的不同表现，且保持状态一致。
-   **双策略机制**：
    *   **数据信任策略（Data Trust Policy）**：定义结构化数据中哪些字段是可信的（保留原文），哪些不可信（转为符号），避免将元数据等可信信息错误符号化。
    *   **数据使用策略（Data Usage Policy）**：一种安全辅助层。当HumanShell要将符号所代表的不可信数据作为**可执行命令或代码**运行时，会触发用户审批，防止高风险的命令执行。
-   **部署方式**：作为一个**OpenClaw插件**实现，仅通过**工具钩子（Tool Hooks）**进行拦截和修改，无需改动底层AI模型、代理逻辑或现有工具的实现。

-   DualView是唯一同时阻止了**即时IPI和存储IPI（攻击成功率0%）**，并且保持**高任务效用（与未防护基线相比仅下降几个百分点）**的防御方案。

### 7. 优点：方法或实验设计上的亮点
-   **新颖的威胁模型**：首次系统性地定义和解决了“存储型间接提示注入”，填补了双LLM防御模式的一个重大空白。
-   **创新的“双视图”架构**：巧妙解决了代理（需符号、安全）与人类/环境（需原文、实用）之间的根本矛盾，通过视图隔离与同步实现安全与效用的平衡。
-   **设计上的安全性**：其防护基于数据流追踪和隔离，是确定性的，而非概率性的，这从根本上杜绝了攻击者文本进入决策模型。

-   **性能与成本开销**：DualView增加了显著的Token消耗。为解决此问题，作者提出的使用更小、更便宜的模型进行隔离数据处理的方案，文中并未进行实验验证。
