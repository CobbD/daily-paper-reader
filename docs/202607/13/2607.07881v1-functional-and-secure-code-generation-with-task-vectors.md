## 摘要
大型语言模型（LLM）越来越多地被用于代码生成，但它们难以生成功能正确且无安全漏洞的代码。先前旨在提升此类编码 LLM 安全代码生成能力的工作，大多侧重于使用不同数据集分别评估代码功能性和安全性，或在生成后查找漏洞。与此同时，文本生成领域在对齐技术方面已取得显著进展，即通过微调使模型输出展现出某些特性（如有用性、无害性）。尤其值得关注的是任务向量算术，该方法通过对 LLM 权重进行线性操作，可在仅带来极小计算开销的情况下任意增强对齐效果。我们开发了一种新颖的方法 SecVecCoder，利用任务向量生成既功能正确又安全可靠的可信代码，无需生成后调整。在 CodeGuard+ 基准测试中，针对来自三个系列的六个编码 LLM，SecVecCoder 将可信代码补全率相较于基础模型提高了 2.1 至 36.0 个百分点，在未见过的 CWE 类型上提升幅度最高可达 39.1 个百分点。由于编码 LLM 的有效性仅依赖模型权重的改变，SecVecCoder 无需特定解码方法，因此平均解码延迟保持在基础模型的 0.6% 以内。

## Abstract
Large language models (LLMs) are increasingly used for code generation, but they struggle to generate functional code free of security vulnerabilities. Prior work to improve the secure code generation abilities of such coding LLMs has largely focused on evaluating code functionality and security separately using different datasets, or focused on finding vulnerabilities post-generation. At the same time, the text-generation domain has seen significant work on alignment techniques, where models are tuned such that their outputs exhibit certain qualities (e.g., helpfulness, harmlessness). Of particular interest is task-vector arithmetic, where linear operations on LLM weights can be used to arbitrarily enhance alignment while incurring only minimal computational overhead. We develop a novel method, SecVecCoder, leveraging task vectors to produce trustworthy code that is simultaneously functional and secure without the need for post-generation adjustment. Across six coding LLMs from three families on the CodeGuard+ benchmark, SecVecCoder improves the rate of trustworthy code completions by 2.1-36.0 percentage points over the base model, with improvements on unseen CWE types reaching up to 39.1 percentage points. Since the effectiveness of the coding LLM relies only on changing the model weights, SecVecCoder requires no method-specific decoding and hence achieves a decoding latency within 0.6% of the base model's, on average.

## 2. 方法论
- **核心思想**：将编码 LLM 在安全编码方向与脆弱编码方向上的权重变化建模为任务向量，通过向量算术直接修改基础模型权重，摆脱对特定解码策略的依赖。
- **关键技术细节**：
  - **模型构建**：
    1. 在基础模型 \(W_{\text{base}}\) 上分别用安全补全和漏洞补全进行微调，得到 \(W_{\text{sec}}\) 和 \(W_{\text{vul}}\)。
    2. 定义安全任务向量 \(\tau_{\text{sec}} = W_{\text{sec}} - W_{\text{base}}\) 和漏洞任务向量 \(\tau_{\text{insec}} = W_{\text{vul}} - W_{\text{base}}\)。
    3. 通过 `W_steered = W_base + α τ_sec - θ τ_insec` 合成新模型。
  - **微调策略**：比较了三种方案构建 \(W_{\text{sec}}\) 和 \(W_{\text{vul}}\)：
    - SFT：分别对安全/漏洞补全做标准监督微调。
    - SafeCoder 风格：三目标（功能损失 + 安全补全损失 + 漏洞补全 unlikelihood 损失）。
    - LPO（Localized Preference Optimization）：在偏好优化中突出安全与漏洞实现差异的 token 位置。
  - **任务向量相似性分析**：LPO 得到的 \(\tau_{\text{sec}}\) 与 \(\tau_{\text{insec}}\) 呈高度负相关且符号一致性大幅下降，表明它能有效分离安全与漏洞方向，因此被选为主要来源。
  - **操作符与强度**：
    - Negation：\(W_{\text{base}} - θ τ_{\text{insec}}\)。
    - Contrast：\(W_{\text{base}} + θ(τ_{\text{sec}} - τ_{\text{insec}})\)。
    - Secure-Anchored（本文最终采用）：\(W_{\text{base}} + τ_{\text{sec}} - θ τ_{\text{insec}}\)，θ 在实验中取 1.5。
  - **最终方案**：SecVecCoder 使用 LPO 衍生向量、Secure-Anchored 操作、θ=1.5，直接修改权重后以标准自回归解码生成代码。

## 3. 实验设计
- **数据集**：
  - **训练集**：来自 SVEN 的 803 对安全/漏洞补全数据，覆盖 9 个常见 CWE 类别（Python 380 对、C/C++ 423 对）。
  - **评估基准**：CodeGuard+，支持在同一补全上联合测量功能性和安全性（通过单元测试和 CodeQL 安全规则），包含 17 个主 CWE 测试场景和 12 个未见 CWE 场景。
- **对比方法**：
  - 训练期方法：SVEN（前缀微调）、SafeCoder（指令微调）、SCoDE（安全导向矩阵 + 对比解码）、DeepGuard（多层级安全模块）。
  - 推理期方法：CoSec（共同解码）、SCoDE 的对比解码形式。
  - 基础模型：Qwen2.5-Coder（3B/7B）、DeepSeek-Coder（1.3B/6.7B）、StarCoder2（3B/7B），共 6 个。
- **评估指标**：
  - `sec-pass@k`：在 k 个样本中至少有一个既功能正确又安全的概率（核心指标）。
  - `pass@k`、`sec@kpass`、SVEN-SR（原始安全率）作为辅助指标。
  - 所有主实验使用 100 个样本，温度 0.1，top-p 0.95，最大生成 512 tokens。

## 6. 主要结论与发现
- 使用 LPO 衍生任务向量 + Secure-Anchored 操作能一致地提升可信代码生成率：6 个模型上 sec-pass@1 绝对提升 2.0 ~ 39.1 个百分点（相对提升最高 106.5%），并在 5 个模型中达到最优。
- 与训练密集型方法（SafeCoder、SCoDE）相比，训练成本降低 2.6–12.4 倍，推理延迟与基模型几乎相同，无需额外解码开销。
- 对未见 CWE 类型仍表现出强泛化能力（提升 4.3%–106.5%），说明学到的是安全编码的方向性行为而非单纯记忆。
- 仅靠安全率（SVEN-SR）无法反映真实可信度，必须联合功能与安全评测；论文同时指出部分先前方法在单独安全指标上表现好但在 sec-pass@1 上不足。

## 7. 优点
- **方法创新**：首次将任务向量算术引入可信代码生成，为 LLM 安全对齐提供轻量、可调的范式。
- **高效低耗**：训练成本远低于传统训练期防御，推理完全无额外开销，便于实际部署。
- **统一评测**：采用 CodeGuard+ 对功能与安全进行联合测量，更符合“可信代码”的实际需求，揭示了以往分开评测可能高估模型安全性。
- **实验全面**：涵盖 3 个模型家族、6 个模型尺寸，多操作符与多强度网格搜索，泛化、消融、效率多维度验证。

## 8. 不足与局限
- **训练数据规模与覆盖面**：仅使用 803 对样例、9 类 CWE，且主要是小型代码片段，复杂项目级漏洞模式可能未被捕捉。
- **部分模型鲁棒性**：DeepSeek-Coder-6.7B 上 sec-pass@1 未超越 SafeCoder 和 SVEN，暗示 LPO 向量的最优强度可能因模型而异，需要更自适应的超参策略。
- **对未见 CWE 的不对称表现**：在 StarCoder2 上未见 CWE 的安全性劣于 SafeCoder，可能因 SafeCoder 训练集直接包含相关 CWE，暴露了依赖 LPO 微调数据的冷启动问题。
- **安全检测单一**：安全判定仅依赖 CodeQL 静态分析规则，可能会遗漏逻辑漏洞或运行时安全问题。
- **缺乏用户研究与动态评估**：未考虑开发人员与模型交互的真实场景（如多轮补全），也未评估在实际软件工程流程中的可用性。
- **未探索多安全属性联合**：当前仅处理“安全/漏洞”二元任务向量，未扩展到多种安全需求（如输入验证、资源管理、权限控制等）的同时调制。
