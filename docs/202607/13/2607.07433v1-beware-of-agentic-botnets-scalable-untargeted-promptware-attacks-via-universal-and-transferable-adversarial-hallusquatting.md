## 摘要
随着智能体大型语言模型（LLM）应用的日益普及，一种先前被称为提示件（promptware）的新威胁已经出现。虽然先前工作已证实，在弱威胁模型下，对手可以利用直接通道对LLM应用实施提示件攻击，但许多应用并不提供任何可直接用于提示注入的通道（互联网渠道除外）。这引出一个问题：在实际威胁模型中，攻击者能否在没有任何直接通道的情况下大规模利用LLM应用？本文中，我们展示了LLM固有的资源标识符幻觉倾向可被利用来放大非定向提示件攻击，这些攻击能大规模拉取对抗性提示，并可被用于组建僵尸网络。我们提出了对抗性幻觉抢占技术，攻击者识别流行资源（如热门仓库、热门技能等），计算LLM对流行资源名称的幻觉分布，并抢先注册这些名称以托管对抗性提示。通过利用幻觉在不同基础LLM之间以及向应用层的可预测性和可迁移性，对手能够在弱威胁模型下显著放大非定向提示件的触及范围，并通过利用LLM应用在拉取受感染幻觉资源的设备上安装僵尸程序，从而组建僵尸网络。我们通过实验证明，幻觉资源生成的发生率很高，在仓库克隆场景下高达85%，在技能安装场景下高达100%，且这些幻觉在不同基础模型和不同提示之间具有可迁移性。我们展示了对抗性幻觉抢占针对多种生产环境LLM应用的实用性，这些应用的工具集中集成了终端，我们实现了远程工具执行和远程代码执行。

## Abstract
The growing adoption of agentic LLM applications has introduced a new threat previously named as promptware. While prior work has established that adversaries can exploit direct channels to LLM applications to apply promptware under weak threat models, many applications do not provide any direct channels that could be exploited for prompt injection beyond the Internet. This raises a question: can attackers exploit LLM applications at scale without any direct channels in practical threat models? In this work, we show that the inherent tendency of LLMs to hallucinate resource identifiers can be exploited to amplify untargeted promptware attacks that pull adversarial prompts at scale and could be exploited to establish a botnet. We introduce adversarial hallucination squatting, a technique in which attackers identify trending resources (e.g., popular repositories, popular skills, etc.), compute the LLM distribution of hallucinations on the trending resource names, and preemptively register them to host adversarial prompts. By leveraging the predictability and transferability of hallucinations across foundational LLMs and to application layers, adversaries can significantly amplify the reach of untargeted promptware under weak threat models and establish a botnet by exploiting LLM applications to install a bot on the device that pulled the compromised hallucinated resource from the Inter. We empirically demonstrate that hallucinated resource generation occurs at high rates, up to 85% in repository cloning scenarios and up to 100% in skill installation, and that these hallucinations transfer between foundational models and different prompts. We demonstrate the practicality of adversarial hallucination squatting against various production LLM applications with integrated terminals in their set of tools, achieving remote tool execution and remote code execution.

- **背景**：基于大型语言模型（LLM）的智能体应用日益普及，其可调用工具（如终端、文件系统）带来了新的安全威胁——“提示件”（promptware）。现有提示件攻击大多依赖直接或间接通道（如邮件、共享文档）向目标应用注入恶意指令，且多为“有目标”的单点攻击。
- **核心问题**：在**无直接交互通道**的弱威胁模型下，攻击者能否**大规模利用LLM应用**实施非定向攻击？本文探索将LLM在资源标识符上的**固有幻觉倾向武器化**，通过预先“抢占”幻觉高频资源，实现无定向的大规模提示注入，乃至组建**智能体僵尸网络**（Agentic Botnet）。

### 2. 方法论
#### 核心思想：对抗性幻觉抢占（Adversarial HalluSquatting）
攻击者无需直接接触受害者应用，而是利用LLM在获取流行资源时**预测并抢占可能出现的幻觉标识符**（如不存在的仓库owner/name组合、技能slug），预先注册这类资源并嵌入恶意提示，当智能体自主拉取这些资源时即被植入“提示件”。

#### 关键技术步骤
1. **资源流行度识别**：如GitHub Trending仓库、ClawHub热门技能。
2. **幻觉候选分布计算**：
   - 使用探针（如基础LLM API或模拟应用）对目标资源名反复查询（如“print a shell command to clone <repo>”），统计LLM返回的错误资源标识符（如`repo-name/repo-name`）及其频率。
   - 算法1：单模型多次采样，归一化得到候选概率分布。
   - 算法2：跨多模型聚合归一化分数，按共识得分排序，得到**通用抢占候选**（universal squatting candidate）。
3. **资源抢占与恶意植入**：
   - 注册高分候选资源（如GitHub仓库、ClawHub技能），在其中嵌入对抗性提示（如指示执行恶意脚本）。
4. **攻击触发**：用户发出模糊请求（如“clone librepods”），智能体自主规划时产生的幻觉使其拉取了被抢占的资源，上下文被污染，进而执行恶意动作（RCE或工具调用），实现大规模扩散。

#### 漏洞类别（针对技能安装场景）
- **V1（词删除）**：智能体解析技能名时丢弃部分词（如“skill-vetter”被解析为`vetter`），攻击者注册遗漏的短名即可拦截安装。
- **V2（显示名与slug偏离）**：技能展示名与注册slug不一致，智能体使用展示名搜索时极易落入攻击者精心构造的虚假技能中；尤其当原技能描述非英文时，攻击者通过英文描述的同名虚假技能可轻易占据搜索榜首。

### 6. 主要结论与发现
- **高幻觉率**：对于近期（2025年）创建的仓库，基础模型平均幻觉率高达92.4%，而历史仓库近乎无幻觉；技能安装场景下幻觉率在很多组合达100%。
- **可迁移性**：幻觉主要表现为“自我参照”模式（如`repo-name/repo-name`），且在不同LLM家族、不同提示以及模型层到应用层之间高度转移，使攻击者能轻松预测通用抢占目标。
- **攻击可行性**：通过注册漏洞利用仓库或技能，成功在生产应用中实现远程代码执行（RCE）或上下文窃取，成功率范围20%–100%不等（见表1），验证了非定向提示件攻击的实用威胁。
- **僵尸网络潜力**：攻击可一步到位使智能体拉取并执行恶意代码，从而将设备纳入攻击者控制，具备组建大规模僵尸网络的潜力。
- **缓解关键**：强制智能体在获取外部资源前先进行web搜索可大幅降低幻觉率，但搜索行为高度依赖模型与提示措辞，无法从根本上杜绝攻击。

### 7. 优点
- **新颖性**：首次系统性地将LLM幻觉武器化为可扩展的非定向提示件攻击，并提出“对抗性幻觉抢占”这一攻击范式。
- **系统性与深度**：覆盖仓库和技能两种场景、基础模型与应用两层，充分探索了跨模型、跨提示、跨应用的可迁移性，解释了漏洞的深层原因（如skill.md中“skill”词频对V1的影响）。
- **量化方法**：提出通过多次采样及跨模型归一化计算通用抢占候选的算法，使攻击策略具备可操作性和可度量性。
- **现实贴合度**：攻陷多个知名生产应用（Cursor、Windsurf、GitHub Copilot等），切实证明了威胁的紧迫性。
- **负责披露**：遵循IRB审查，向所有受影响的厂商通报，实验过程中采取安全措施防止真实用户受害。

### 8. 不足与局限
- **平台局限**：仅聚焦GitHub和ClawHub，未扩展到Docker Hub、Hugging Face等其他常见资源平台。
- **样本范围**：仓库实验以10个2025年热门仓库为主，技能实验部分重复次数仅为10次，可能遗漏低频但危险的幻觉模式。
