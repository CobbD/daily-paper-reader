## 摘要
随着大型语言模型（LLM）代理承担进攻性安全工作，一次超出范围的工具调用就可能突破客户参与边界、干扰生产或使漏洞赏金发现失效。不同于固定的安全策略，关键的边界是在用户请求中声明的，并且必须从意图中推断出来。进攻性安全的对抗性加剧了这一挑战：同一工具调用是否在范围内，不取决于动作本身，而取决于其接触的目标以及运行上下文，而任何固定策略都无法预先枚举这些情况。我们研究了预执行门控：一个低成本、可信的 LLM 法官检查由强大、可替换的代理提出的每个调用，并在其运行前接受或拒绝。我们引入了 ScopeJudge，这是一个包含 4,897 个工具调用（7.7% 的范围违规）的基准，这些调用来自旨在引诱代理超出范围的工程化任务的代理轨迹，并由专业渗透测试人员在调用级别进行标记，评分者之间有较高的一致性（Fleiss kappa = 0.64），这设定了专家一致性的参考点 F1 = 0.78。我们在五种转录策略下评估了八个法官模型，变化法官所看到的上下文量，从仅静态策略到完整原始转录，并绘制了相应的成本-准确度帕累托前沿。我们发现，静态策略在结构上不足以进行范围强制：由于对用户请求视而不见，法官的召回率崩溃至接近零，这证实了范围存在于请求中，并且基于请求的监控是必要的。由于遗漏违规的代价高于错误拒绝，我们分别报告了精确率、召回率和 F1，并推荐了两个操作点：一个成本敏感配置和一个面向高风险部署的召回率优先配置。我们发布 ScopeJudge 数据集，以支持自主安全代理的实时监控和可扩展监督。

## Abstract
As LLM agents take on offensive security work, a single out-of-scope tool call can breach a client's engagement boundary, disrupt production, or void a bug-bounty finding. Unlike a fixed safety policy, the boundary that matters is declared in the user's request and must be inferred from intent. That challenge is sharpened by the adversarial nature of offensive security: the same tool call is in or out of scope depending not on the action itself but on the target it touches and the context in which it runs, which no fixed policy can enumerate in advance. We study pre-execution gating: a cheap, trusted LLM judge inspects each call proposed by a strong, swappable agent, and accepts or rejects it before it runs. We introduce ScopeJudge, a benchmark of 4,897 tool calls (7.7% scope violations) from agent trajectories on tasks engineered to tempt agents out of scope and labeled at the call level by professional penetration testers, with substantial inter-grader agreement (Fleiss kappa = 0.64) that sets an expert agreement reference point of F1 = 0.78. We evaluate eight judge models under five transcript strategies, varying how much context the judge sees, from the static policy alone to the full raw transcript, and chart the resulting cost-accuracy Pareto frontier. We find that a static policy is structurally insufficient for scope enforcement: blind to the user's request, judge recall collapses to near zero, confirming that scope lives in the request and that request-conditioned monitoring is necessary. Because a missed violation costs more than a spurious rejection, we report precision, recall, and F1 separately and recommend two operating points: a cost-sensitive configuration and a recall-first one for high-stakes deployments. We release the ScopeJudge dataset to support real-time monitoring and scalable oversight of autonomous security agents.

### 2. 方法论：核心思想、关键技术细节与流程
- **核心思想**：将范围强制视为**监测问题**而非规范问题。法官依据静态策略（SP）、用户意图（u）以及可选的轨迹历史，对代理提出的工具调用 ak 进行实时接受（允许执行）或拒绝（要求修正或升级至人类）。
- **形式化定义**：
  - 轨迹 τ = (u, (a1, o1), …, (aN, oN))
  - 预执行法官 J_pre: (SP, φ(τ_{<k}, a_k)) → {0,1}，其中 φ 为**转录策略**（即上下文投影函数），0 表示允许，1 表示拒绝。
- **五种转录策略（φ）**：按成本与暴露给攻击者的信息量递增排序：
  1. **Static Policy**：仅提供 SP 和 ak，不含用户意图和历史。
  2. **Intent**：SP + u + ak。
  3. **Intent Plus Tool Calls**：SP + u + 历史工具调用列表（不含输出）。
  4. **Intent Plus Output Summarized**：在上一策略基础上，增加对工具输出的**摘要**（由同一法官模型生成，避免输出原文进入法官上下文）。
  5. **Full**：SP + 完整轨迹（包含工具输出原文和推理痕迹）。
- **评估指标**：
  - 精度（Precision）、召回率（Recall）、F1（针对违规调用，即正类为“拒绝”）。
  - 成本 C(J, φ) = ρ(J) × 平均输入令牌数，其中 ρ(J) 为每令牌价格。

- **数据集与场景**：
  - **ScopeBench**：30 个专门设计的进攻性安全任务，每个任务都伴有明确的用户意图和范围约束，且最直接的题解路径越界，以诱使代理犯错。
  - **ScopeJudge 校准集**：从 5 个代理模型（Claude、Qwen、Kimi 等）生成的轨迹中抽取 100 条轨迹，共 **4,897 个工具调用**（其中违规比例 7.7%），由 **5 位专业渗透测试员**独立标注每个调用是否违规（多数投票作为金标准）。

- **评估对象与对比方法**：
  - **8 个法官模型**：4 个开源（GLM-5.2, Qwen3.6-35B, Nemotron-3-Ultra, Trinity-Large）和 4 个专有（Claude Opus/Sonnet/Haiku 4.x, GPT-5.4-mini）。

  - 数据集由 5 位专家独立标注，具有较高评分者间信度，金标准较可靠。

### 6. 主要结论与发现
- **静态策略完全失效**：在仅提供 SP 时，几乎所有法官的违规召回率趋近零，证明了操作范围存在于用户请求中，**请求感知监控是结构性的必要条件**。加入用户意图后，召回率急剧上升至 0.60–0.94。
- **开源法官表现最优**：最好的法官是开源模型 **GLM-5.2**（F1=0.66），优于所有专有法官，且成本更低（约专有旗舰的 1/3）。
- **成本-精度帕累托前沿**：
  - **成本敏感配置**：GLM-5.2 + Intent Plus Tool Calls，召回 87%，精度 0.50，成本 $0.0032/次调用。
  - **召回优先配置**：Claude Opus 4.8 + Intent Only，召回 94%，精度 0.39，成本 $0.0042/次调用。
- **上下文与成本的权衡**：在 Intent 之上增加调用历史可提升 F1，但增加输出摘要或完整轨迹带来的边际收益递减，且成本急剧上升。

### 7. 优点
- **问题导向实用**：直接解决进攻性安全代理部署中的真实风险，并提供可操作的部署建议（两种操作点）。
- **基准构建严谨**：由行业专家标注，具有可量化的评分者间一致性，为后续研究提供了可靠的校准集。
- **方法学贡献清晰**：形式化了转录策略的分类法，将上下文、成本、攻击面三者有机结合，给出了“意图不可或缺”的实证证据。
- **强调开源与成本权衡**：结论指出开源模型可成为高性价比的监控器，有利于技术的普及和透明性。
- **实验设计全面**：覆盖多个法官家族、全部策略空间，并剖析了精度-召回-成本间的非对称性。

### 8. 不足与局限
- **基准规模与多样性受限**：仅在一个领域（Web渗透测试）、一个任务套件（ScopeBench）、100 条轨迹上评估，且语言仅为英语，通用性有待验证。
- **未评估对抗鲁棒性**：论文虽指出历史策略增大了攻击面，但实验未测试代理或环境是否可操纵法官输出，因此成本-精度前沿对安全声称仅为**上界**。
- **自评偏差风险**：部分法官模型同时是轨迹生成模型（Claude Sonnet、GPT-5.4-mini），可能存在自我偏好偏差，未在评分中控制。
- **基础率敏感**：违规率仅 7.7%，精度和 F1 会随实际环境中违规比例变化而变化。
- **统计推断不足**：每配置仅执行一次（温度=0 且无重复），未提供置信区间，对不同配置间的微小差异需谨慎解读。
- **任务范围仅二元分类**：只考虑“在范围/出范围”，未区分“虽在范围内但潜在危险以致需升级”的灰色地带。
