## 摘要
键值（KV）缓存可降低大语言模型（LLM）的推理延迟。传统的基于前缀的重用由于要求精确的令牌和位置匹配，导致跨推理请求的缓存命中率较低。为了提高效率，近期的系统优化引入了位置无关的KV重用，允许在出现相同文本块时重用KV缓存，而不管其在序列中的位置。我们发现这种设计引入了一种新的威胁，即KV缓存劫持。由于KV缓存通过令牌匹配来检索，但其编码了最初计算时的上下文，因此与看似良性的令牌块相关联的KV可能编码了攻击者控制的前缀。当随后在受害查询中重用时，这种被污染的KV会悄悄劫持模型的行为，即使输入中没有出现攻击者控制的文本。我们提出了HIJACKKV，这是第一个系统性地利用这一漏洞的攻击框架，展示了其严重性和实用性。HIJACKKV优化攻击者控制的前缀，使得后续常见良性文本计算出的KV编码了攻击者的目标，同时文本本身保持不变以便将来缓存命中。HIJACKKV在单次尝试中实现了平均94%的成功率，在现实约束下（包括低命中率（10%）和频繁重计算（50%））仍保持有效，能够持续跨多轮交互，并在黑盒设置下跨模型迁移。我们还为构建安全的KV重用系统提供了设计见解。

## Abstract
Key-Value (KV) cache reduces inference latency in large language models (LLMs). Traditional prefix-based reuse has low cache hit rates across inference requests because it requires exact token and position matches. To improve efficiency, recent system optimizations introduce position-independent KV reuse, allowing KV cache to be reused whenever identical text chunks appear, regardless of their position in the sequence.   We show this design introduces a new threat, KV Cache Hijacking. Since KV caches are retrieved by token match but encode the context in which they were originally computed, the KV tied to a benign-looking token chunk may encode an attacker-controlled prefix. When later reused in a victim query, this contaminated KV silently hijacks the model's behavior, even if no attacker-controlled text appears in the input.   We introduce HIJACKKV, the first attack framework that systematically exploits this vulnerability, demonstrating its severity and practicality. HIJACKKV optimizes an attacker-controlled prefix, so that the KV computed for a subsequent common benign text encodes the attacker's goal, while the text remains unchanged for future cache hits. HIJACKKV achieves an average 94% success rate in a single attempt, remains effective under realistic constraints including low hit rates (10%) and frequent recomputation (50%), persists over multi-turn interactions, and transfers across models in black-box settings. We further provide design insights for building secure KV reuse systems.

### 1. 论文的核心问题与整体含义
- **研究背景**：大语言模型（LLM）推理为降低延迟广泛采用KV缓存技术。传统前缀重用要求令牌和位置精确匹配，命中率低。为提升效率，近期系统引入了“位置无关KV重用”，允许共享文本块（如常见文档、FAQ）在任何位置被重用其KV缓存，以大幅减少重复前向计算。
- **核心问题**：位置无关KV重用破坏了内部状态与上下文的因果一致性——相同文本在不同前缀下会计算出不同的KV状态。攻击者可以构造恶意前缀，污染共享缓存中常见文本块的KV状态，当受害者查询重用到该缓存时，即使其输入不含任何攻击性文本，模型输出也会被暗中劫持。
- **整体含义**：论文首次揭示并形式化了这一新型“KV缓存劫持”威胁，表明现有效率优化带来了跨用户完整性攻击面，且现有重计算、压缩、防御机制不足以消除该风险。

### 2. 方法论（核心思想与关键技术细节）
- **攻击建模**：攻击者目标为找到前缀 `p`，使得将 `p` 与常见良性文本块 `X̃` 拼接后送入LLM，得到 `X̃` 对应的KV缓存 `T_{p}^{X̃}`；当受害者查询中包含相同文本块 `X` 且发生位置无关缓存命中的情况下，重用 `T_{p}^{X̃}` 将使模型输出攻击者指定的恶意答案 `r̃`。形式化为最小化交叉熵损失：
  \[
  p^* = \arg\min_p \mathcal{L}_{\mathrm{CE}}\big(\mathrm{LLM}(X \oplus q \mid T_{p}^{\tilde{X}}), \tilde{r}\big)
  \]
- **优化算法**：采用Greedy Coordinate Gradient（GCG）对离散前缀令牌进行优化。
  - 通过one-hot编码的连续松弛计算梯度。
  - 每步迭代：对每个位置取top-k梯度下降方向候选令牌，随机选取 `B` 个位置替换为候选令牌形成候选集合，从中选出损失最小的前缀更新。
- **损失计算模拟**（`compute_loss`）：
  1. 将前缀嵌入与目标文本嵌入拼接，通过LLM计算全序列KV。
  2. 提取对应目标文本块的那部分KV，作为恶意缓存（丢弃前缀自身的KV，但前缀的影响已编码其中）。
  3. 用该恶意缓存模拟受害者推理：输入 `X̃ ⊕ q`，但重用缓存的KV，得到输出概率分布。
  4. 计算输出分布与目标恶意答案的交叉熵。
- **攻击流程**：攻击者本地使用代理模型（白盒或黑盒替代模型）优化出前缀，再将 `p ⊕ X̃` 作为一次查询发送给目标LLM服务，服务计算KV并存入共享缓存。后续任何用户查询中出现相同文本块即可能触发缓存重用，导致输出被劫持。整个过程不接触服务端内部缓存，无需系统权限。

### 6. 主要结论与发现
- 位置无关KV重用机制存在本质安全缺陷：相同文本在不同前缀下KV状态显著偏离（键偏差约50%，值偏差约25%），足以劫持模型输出。
- HIJACKKV攻击高度有效：默认设定下平均T-ASR达94%，U-ASR近乎100%。
- 强鲁棒性：在缓存命中率低至10%或重计算比例高达50%时，攻击仍维持较高成功率（T-ASR分别约67%和39%）。
- 持久性：即使在恶意缓存后插入2048个无关令牌，T-ASR仍保持在47%左右。
- 跨模型迁移性：在黑盒设定下，用LLaMA3.1-8B优化的前缀迁移到LLaMA-3.3-70B、Qwen3-8B/14B等模型仍有效（T-ASR可达37–40%，U-ASR更高），中等大小模型迁移精度略降。
- 现有防御不足：RobustKV和CachePrune等KV级防御或压缩方法无法有效缓解，因为恶意缓存仍维持高注意力得分，且污染不具有固定模式。
- 建议方向：混合重计算高影响力KV状态并辅以偏差阈值拒绝策略。

- **攻击框架实用且强大**：GCG优化前缀长度短（32 tokens）、效果稳定，能在多种现实约束下生效，具备跨模型迁移能力。

### 8. 不足与局限
- **缓存可用性依赖**：攻击成功依赖于恶意缓存被写入并与受害者查询匹配，存在缓存替换策略（如LRU/LFU）和插入窗口不确定性，未在真实工作负载跟踪中验证。
- **攻击者知识假设**：需知目标系统使用位置无关重用，且需选择常见共享文本块作为投毒目标；对缓存占用情况未知时可能需要侧信道探测，可能引入自污染。

- **实验环境局限**：未在生产级多租户服务上实测，缓存替换、并发请求等复杂因素未纳入。
- **防御措施不彻底**：重计算虽能部分缓解但带来额外计算开销，且攻击者可通过自适应优化对抗，安全与效率的平衡仍需探索。
- **范围限制**：主要针对实时多租户位置无关缓存重用场景，对于先分块后缓存的独立编码RAG流水线适用性较低。
