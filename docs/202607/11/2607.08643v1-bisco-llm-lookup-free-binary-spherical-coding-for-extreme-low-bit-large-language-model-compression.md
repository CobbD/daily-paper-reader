## 摘要
大型语言模型（LLMs）在部署过程中越来越受到内存容量、权重带宽和检查点存储的限制。现有的低比特压缩方法主要遵循两个方向。标量或分组量化简单且与高效的低精度内核兼容，但当目标预算接近每权重2比特时，其表示能力变得有限。矢量量化权重压缩提供了更丰富的块级表示，但通常引入了显式码本、索引查找和额外的存储开销。本文提出了BiSCo-LLM，一个无码本的二进制球面编码框架，用于极低位LLM权重压缩。核心流程建立在三个组件之上。首先，局部权重块被映射到单位超球面上，并二值化为紧凑的球面编码，因此主要载荷是位打包的符号流，而非显式的VQ中心。其次，残差BSQ阶段对基础球面编解码器留下的重构误差进行编码，提供了一条显式的率失真路径，而无需存储码本。第三，在替换每个Transformer模块类别后，执行按类别恢复蒸馏，减少局部权重重构与组装模型行为之间的不匹配。一个小的8位保护通道路径被用作敏感通道的辅助稳定机制，并与BSQ载荷分开计算。报告的存储预算包括二进制编码、神经解码器、保护通道载荷、LoRA适配器和元数据。

## Abstract
Large language models (LLMs) are increasingly constrained by memory capacity, weight bandwidth, and checkpoint storage during deployment. Existing low-bit compression methods mainly follow two directions. Scalar or group-wise quantization is simple and compatible with efficient low-precision kernels, but its representation capacity becomes limited when the target budget approaches 2 bits per weight. Vector-quantized weight compression provides a richer block-level representation, but usually introduces explicit codebooks, index lookup, and additional storage accounting. This paper presents BiSCo-LLM, a codebook-free binary spherical coding framework for extreme low-bit LLM weight compression. The core pipeline is built on three components. First, local weight chunks are mapped onto a unit hypersphere and binarized into compact spherical codes, so that the main payload is a bit-packed sign stream rather than explicit VQ centroids. Second, a residual BSQ stage encodes the reconstruction error left by the base spherical codec, providing an explicit rate-distortion path without stored codebooks. Third, category-wise recovery distillation is performed after replacing each Transformer module category, reducing the mismatch between local weight reconstruction and assembled model behavior. A small 8-bit protected-channel path is used as an auxiliary stabilization mechanism for sensitive channels and is counted separately from the BSQ payload. The reported storage budget includes binary codes, neural decoders, protected-channel payloads, LoRA adapters, and metadata.

### 2. 论文提出的方法论
BiSCo‑LLM 包含三个核心组件：
- **a. 单位球面二值权重映射**
  - 将局部权重块 `x` 编码为隐向量 `z`，并归一化投影到单位超球面，再通过符号函数二值化为球面码 `q ∈ {±1/√b}^b`。
  - 主存储载荷为**位打包的符号流**，而不存储显式码本。训练使用直通估计器（STE），损失函数结合重构 MSE、潜变量承诺损失和熵正则化项，以鼓励码字利用的多样性。
- **b. 第二阶段残差 BSQ 压缩**
  - 基础编解码器先重构出近似权重，然后对重构残差施加第二个 BSQ 阶段，以此提升额外的编码容量并形成显式的率‑失真路径。
  - 设计动机：由于固定权重块的数量远小于隐式二进制空间，单一大码空间的收益并不单调，因此将额外比特分配给残差更能有效利用容量。
- **c. 类别级恢复蒸馏与辅助机制**
  - **按类别压缩与恢复**：对同一类别的全部线性层（如 q_proj、down_proj 等）统一训练编解码器，替换后执行模块类别的蒸馏恢复，蒸馏目标结合输出分布对齐与隐藏态匹配。
  - **8‑bit 敏感通道保护**：基于激活统计和权重大小选出少量高敏感通道（默认为输入通道或耦合 MLP 中间通道），以 8‑bit 单独存储，保护比仅设为 1%，其存储被显式计入 `S_8bit`。
  - **可选的 LoRA 补偿**：恢复阶段可插入低秩适配器，其参数量同样计入最终预算。
  - **工程优化**：采用类别批量化编解码训练和并行残差解码以降低实际开销。

- BiSCo‑LLM 在 **Qwen3‑8B 2‑bit 压缩**上取得 WikiText‑2 困惑度 **10.18**（全精度 9.73），平均下游准确率 **68.05**（全精度 69.92），较矢量基线（AQLM、QuIP#、UniSVQ）有不同程度的提升，并大幅优于标量方法。
- **残差 BSQ 阶段**能在多数类别上进一步降低困惑度，而仅加大单阶段粒度并不保证单调提升。
- **1% 的敏感通道保护**即可捕获大部分保护收益，取得显著的降模损失下降（尤其在 v_proj 和 down_proj 上），且低精度保护格式（FP8/INT8）与 BF16 性能基本持平。
- **类别级恢复蒸馏**比纯层局部恢复更能维持宏观模型能力，缓解误差累积。
- 总体表明：无码本球面编码与残差、保护、蒸馏的组合，在显式存储预算下是一种可行的 2‑bit LLM 权重压缩方案。

### 7. 优点
- **无码本设计**：避免了传统 VQ 中码本与索引的存储和查找开销，使主要载荷为简洁的二进制流。
- **存储透明性**：明确列出了 BSQ 码流、解码器、保护通道、LoRA 及元数据等所有部件，并与基线进行公平的真实比特率比较。
- **残差编码策略**：针对有限数据无法填充巨大理论码空间的现实问题，将额外容量分配给残余结构，提高了率‑失真效率。
- **多层次恢复**：从类别级蒸馏到端到端微调，兼顾局部重构与全局行为，帮助维持任务性能。
- **量化与保护分离**：敏感通道仅占极小比例并单独计储，提升了极端压缩的安全性，且保护精度可灵活降低。
