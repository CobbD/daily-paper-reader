## 摘要
自主Web代理有望自动化日常浏览任务，但也继承了Web最古老的攻击面之一。跨站脚本已证明，即使在良性页面上，混合可信与不可信内容也是危险的。代理通过将自然语言解释为指令，重新引发了这一风险，允许第三方和用户生成的内容通过提示注入劫持代理。核心挑战在于，推导任务特定的安全策略需要对与攻击者内容纠缠在一起的页面结构进行推理。我们提出Prismata，一种为Web代理实施上下文最小权限的防御机制，同时限制代理所能看到的内容和能执行的操作。Prismata的动态信任推导为页面内容生成权限标签，并提供结构隔离保证，其灵感源自经典完整性模型，能约束任何标签错误，使得标签只能降低权限且错误标签被限制在一定范围内。Prismata的机械隔离通过编辑内容和限制代理能力来执行这些标签。重要的是，这些机制无需开发者注解，因此Prismata支持大量长尾网站。在最近已发布的Web代理攻击（包括自适应变体）中，Prismata显著降低了攻击成功率，同时保留了良性任务效用。

## Abstract
Autonomous web agents promise to automate everyday browsing tasks, but inherit one of the web's oldest attack surfaces. Cross-Site Scripting proved that mixing trusted and untrusted content is dangerous, even on benign pages. Agents resurface this risk by interpreting natural language as instructions, allowing third-party and user-generated content to hijack the agent via prompt injection. The core challenge is that deriving a task-specific security policy requires reasoning over page structure that is entangled with the attacker's content.   We present Prismata, a defense enforcing contextual least privilege for web agents, constraining both what the agent sees and what it can do. Prismata's dynamic trust derivation produces permission labels for page content, with structural confinement guarantees, inspired by classical integrity models, that bound any labeling errors so that labels can only decrease in privilege and mislabelings are bounded. Prismata's mechanical confinement enforces these labels by redacting content and restricting agent capabilities. Importantly, these mechanisms require no developer annotations, so Prismata supports the long tail of websites. Across recent published web agent attacks, including adaptive variants, Prismata substantially reduces attack success while preserving benign task utility.

### 1. 论文的核心问题与整体含义

- **研究背景**：自主 web 代理（如浏览器自动化工具）在提升工作效率的同时，继承了 web 最早的攻击面——跨站脚本（XSS）式的信任混淆。即使是在良性网站中，可信的开发者内容与不可信的用户生成内容、第三方广告等也混杂在一起。
- **核心问题**：web 代理将自然语言视为指令，导致攻击者可以通过在页面中植入自然语言或图像的“提示注入”（prompt injection）来劫持代理行为。作者将这类攻击定义为**跨站提示注入（Cross-Site Prompting, XSP）**。
- **关键挑战**：为 web 代理制定细粒度的安全策略，必需理解页面的结构语义（如某个按钮的功能），而该结构本身又与被攻击者控制的内容纠缠在一起（即“web 纠缠问题”）。已有的模型级防御易被自适应攻击绕过，而系统级防御要么需要预规划且不适应数据依赖的 web 任务，要么依赖开发者手动标注站点地图，难以扩展。
- **整体含义**：迫切需要一种**无需开发者注解**、可普遍适用于任何网站的上下文最小权限防御机制，既能防止代理执行越权操作，又能保持正常任务完成能力。

### 2. 论文提出的方法论

Prismata 作为一个透明的防御中间层，介于 web 代理与浏览器之间，其核心包括两个互补的技术环节：

- **动态信任推导（Dynamic Trust Derivation）**
  - **行动门（Action Gate）**：对于页面上每一个可交互元素（按钮、链接、输入框等），从 DOM 根节点到该元素的完整祖先路径（critical path）被独立提取出来。仅将用户任务、元素自身文本及该路径上的结构上下文交给语言模型，判断该元素是否在任务权限范围内。由于路径外的任何内容（包括恶意注入）均不可见，因此当注入不在该路径上时，判断天然不受污染（Case 1）。
  - **Biba 解析（Biba Parsing）**：灵感源于 1977 年的 Biba 完整性模型（no-read-down, no-write-up）。对每个元素的祖先路径进行递归评估：每层只看到自身及祖先，子女和兄弟被屏蔽；一旦某层确定内容来源（开发者/用户/托管方/外部），即锁定标签，并沿子树向下级联，不再查询更深节点。若在路径中先遇到结构性线索（如标题、ARIA 属性、类名），则可提前识别不可信区域。对于任务不需要的子树直接剪除，对于任务需要但不可信的内容赋予只读权限。这样即使注入在路径上，只要有先行线索，标签推导仍不受污染（Case 2）。
- **机械隔离（Mechanical Confinement）**
  - **策略模型**：仅基于任务描述（无 DOM 内容）决定需要保留哪些来源的内容。
  - **强制实施**：将上述推导出的能力标签映射到可访问性树（accessibility tree）上，删除不应出现的节点，或降级可交互元素（移除/禁用标识符）。每次代理发出动作时，都根据能力图检查目标元素是否被允许执行该动作，否则拒绝。由于过滤的是每步捕获的页面快照，不修改真实 DOM，不会引入额外状态漂移。

- **安全保证**：上述机制组合使得只有注入恰好位于决定性路径且无结构性前置线索的极端情况下（Case 3），标签推导才可能暴露给攻击者。这种情形在真实页面中极为罕见（约占采样路径的 0.1%）。同时，任何标签错误只会发生在包含注入的路径上，不会影响页面其他元素。

### 3. 实验设计

- **数据集/环境**：使用 WebArena 环境（模拟真实的 GitLab, Shopping, Reddit, CMS 等网站）。另外利用 Common Crawl（前 1 万个域名的 28.3 万页面）和 Mind2Web（任务轨迹页面）作为实证基础。
- **基准**：未开启任何防御的 gpt-5.4-mini web 代理作为基准（无 Prismata）。基础代理基于 BrowserGym + Playwright + Chromium 框架。
- **攻击方法**：实现了三种弹出式提示注入攻击模板（改编自 Pop-up 攻击）：
  - Shortcut（捷径攻击）：弹出明显“捷径”诱导点击。
  - Fake Completion（假完成攻击）：欺骗代理任务已结束。
  - Ignore Instruction（忽略指令攻击）：试图覆盖代理原有指令。
  此外还评估了 WASP（另一公开的 web 代理攻击基准）和自适应攻击（WA20，进行多轮迭代攻击）。

- **评估指标**：
  - 攻击有效性：攻击成功率（ASR）、任务成功率（TSR）、防御成功率（DSR）。

### 6. 论文的主要结论与发现

- Prismata 能强力遏制跨站提示注入：在主要的攻击设定中，平均攻击成功率从 **85.5%** 降至 **0.7%**，防御成功率平均达 **93.1%**。
- 即使受到攻击，任务成功率也从 **4.5%** 提升至 **23.0%**，同时无攻击下的基线任务成功率仅从 29.9% 轻微降至 26.6%，证明实用性损失较小。
- WASP 攻击和自适应 WA20 攻击在 Prismata 下均被有效阻断（ASR 从 100% / 65% / 85% 降至 0%）。
- 实证分析验证了安全假设：在 9 万多个不可信路径中，仅 1.2% 包含可交互后代，其中绝大多数（92%）有前置结构线索，使得 Biba 解析能在接触注入前完成标记。最终残留风险路径比例极低（0.1%），若站点遵循 web 最佳实践则更低（0.017%）。
- 标签推导的模型精度足够：所有被测模型在允许元素集上的精确率均 >93%，其中 GPT-5.4-nano 精确率最高达 98.7%，但召回率偏低（82.7%）；整个系统在召回与安全之间保持了可调的平衡。
- 缓存机制对降低推理开销至关重要，能大幅减少成本和延迟。

### 7. 优点

- **无需开发者介入**：完全依赖 DOM 结构和通用模型，支持长尾网站，解决了以往需要手动标注站点地图的痛点。
- **安全基础坚实**：借鉴经典的 Biba 完整性模型，从原理上限制了标签错误的传播方向（只能降级，不会升级），提供了可解释的安全界限。
- **细粒度的最小权限**：不是简单的域名级别控制，而是针对每个页面元素、根据路径上下文和任务动态赋予能力，符合“刚好够用”原则。
- **体系结构清晰**：行动门和 Biba 解析互补，覆盖了注入在路径内外的不同情况，并用闭式动作空间确保执行不可绕过。
- **实验评估全面**：既有横向的不同攻击模板，也有纵向的消融验证；同时从安全、效用、成本、标签质量四个维度进行衡量；引入了自适应攻击和 WASP 验证了泛化性。
- **实用设计**：通过缓存重用 DOM 标签，降低了增量推理开销；只过滤观察而不修改真实页面，不改变原有代理的执行逻辑。

### 8. 不足与局限

- **仅限文本模态**：当前 Prismata 仅处理基于可访问性树的文本观测，无法防御视觉或听觉等非文本攻击（如对抗性图像扰动、语音注入）。虽然框架原理可扩展，但尚未实现。
- **模型依赖**：信任推导的准确性受底层语言模型能力影响，可能出现漏标（过宽松，降低安全性）或过标（过严，降低可用性）。论文以精确率为优先，但召回率偏低意味着可能过度删减有用元素，影响某些任务。
- **Case 3 残余风险**：在极为罕见的情况下（无结构线索且注入在交互路径上），标签推导会接触到注入内容，此时系统安全性依赖于模型在这些特殊情况下的抵抗力，而非纯粹的机械隔离。
- **高权限任务**：如果用户任务本身需要不可信内容具有写权限（如让代理代替用户回复评论），则最小权限范围自然包含这些能力，无法防御在此范围内的恶意影响（如虚假评论影响购买决策）。这属于平台风控问题，而非 Prismata 的明确目标。
- **环境局限**：评估在 WebArena 模拟环境中进行，虽然页面基于真实代码，但与真实互联网的动态性和对抗性仍有差距。在更复杂或对抗构造的 DOM 上，标签推导的可靠性可能变化。
- **标注的主观性**：人类专家标注本身可能带有主观偏差，且样本量（20 个观察）较小，可能高估或低估标签质量，但论文已明确将此作为参考而非绝对真值。
- **性能开销**：虽然成本可控（利用缓存后），但仍比无防御代理增加了额外推理步骤和延迟，在实时性要求极高的场景下可能需要进一步优化。
