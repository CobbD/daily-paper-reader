## 摘要
训练工具使用智能体从自身经验中改进仍然具有挑战性，因为监督微调依赖于固定的教师蒸馏轨迹，而稀疏奖励强化学习为长周期交互提供弱的监督信号。我们提出了DeepSearch-Evolve，一个基于DeepSearch-World构建的面向网页智能体的自蒸馏框架，DeepSearch-World是一个具有可重现搜索和页面阅读工具的确定性可验证环境。DeepSearch-World包含420K个由实体级随机游走构建的多跳问答任务，并支持对自我进化有用的关键智能体认知行为，包括进度验证、有依据的反思和失败恢复。DeepSearch-Evolve迭代地进行轨迹生成、过滤、数据混合和微调，以训练更强的智能体。在无需从更强大模型蒸馏的情况下，DeepSearch-World-9B与开源智能体相比取得了具有竞争力的表现，在BrowseComp上达到31.2%，在GAIA上达到61.5%，在HotpotQA上达到93.4%，表明可验证环境能够实现长周期网页智能体的可扩展自我进化。我们将发布环境、420K训练池、验证集、模型和代码，以促进未来对自我改进深度搜索智能体的研究。

## Abstract
Training tool-use agents to improve from their own experience remains challenging, as supervised fine-tuning relies on fixed teacher-distilled trajectories, while sparse-reward reinforcement learning provides weak supervision for long-horizon interactions. We present DeepSearch-Evolve, a self-distillation framework for web agents built on DeepSearch-World, a deterministic and verifiable environment with reproducible search and page-reading tools. DeepSearch-World contains 420K multi-hop QA tasks constructed from entity-level random walks and supports key agentic cognitive behaviors useful for self-evolving, including progress verification, grounded reflection, and failure recovery. DeepSearch-Evolve iteratively performs trajectory generation, filtering, data mixing, and fine-tuning to train stronger agents. Without distillation from more capable models, DeepSearch-World-9B achieves competitive performance compared with open-source agents, reaching 31.2% on BrowseComp, 61.5% on GAIA, and 93.4% on HotpotQA, showing that verifiable environments enable scalable self-evolution for long-horizon web agents. We will release the environment, 420K training pool, validation set, model, and code to facilitate future research on self-improving deep search agents.

### 1. 论文的核心问题与整体含义

*   **核心问题**：训练能够使用搜索、阅读等工具的网页智能体时，如何让智能体**从自身经验中持续改进**是一个关键难题。现有方法存在明显短板：
    *   **监督微调**依赖固定的教师轨迹，容易饱和，且受限于教师模型的能力。
    *   **强化学习**（如 GRPO）通常只提供轨迹级别的稀疏奖励，难以定位失败来源（查询构造、工具选择、信息提取等）。
    *   **在策略自蒸馏**虽然提供更密集的监督，但在长程工具使用中，教师分布容易因不可靠的环境而变得噪声，限制了其有效性。
*   **整体含义**：本文提出了一种**可验证的离线环境 + 自蒸馏框架**，使智能体无需依赖更强的外部教师模型，即可通过自我产生的经验实现可扩展的自我进化。

### 2. 方法论

*   **可验证工具环境（DeepSearch-World）**
    *   基于 Wikipedia 超链接图进行实体级随机游走，构建 **420K 多跳 QA 任务**；问题通过模糊化实体名称迫使智能体真正进行搜索和推理。
    *   提供**确定性工具**：
        *   `web_search_wiki`：基于 BM25（Pyserini）的搜索，返回标题、摘要和 URL，模拟搜索引擎。
        *   `visit_wiki`：通过 SQLite 偏移索引读取完整文章，模拟浏览器访问。
    *   **过程级验证**：环境维护一个隐藏的实体集，每次工具调用后检测是否命中未解决实体，并给出阶段性的“成功/失败”信号和**有依据的反思**（如失败时提示规范名称或模糊描述），鼓励智能体进行查询改写和恢复。

*   **脚手架教师智能体（Scaffold Teacher Agent）**
    *   在生成训练轨迹时，引入 **Plan–Act–End** 三个阶段：
        *   **Plan**：初始化结构化进度状态（包含已完成列表、待办列表、经验教训、已提取信息）。
        *   **Act**：执行工具调用，根据环境反馈更新状态。
        *   **End**：基于已验证的工作记忆生成最终答案。
    *   这种显式的规划、记忆和错误恢复过程，生成了高质量的“搜索–失败–反思–重试”轨迹。

*   **自我进化训练循环（DeepSearch-Evolve）**
    *   **迭代流程**：
        1.  使用当前模型作为教师，在 DeepSearch-World 中生成轨迹。
        2.  **拒绝采样与质量过滤**：仅保留答案正确且轨迹质量高（无冗余、对齐目标）的轨迹。
        3.  **支架到 ReAct 转换**：将教师轨迹中的结构化状态和环境反思重写为标准 ReAct 格式的 `<think>` 块，使学生（标准 ReAct 策略）习得规划、记忆跟踪和自纠错能力，同时避免对训练时支架的依赖。
        4.  **微调更新**：基于转换后的 ReAct 轨迹进行监督微调（本质上是硬标签的令牌级策略蒸馏）。
    *   **异步执行与数据混合**：轨迹生成与训练异步进行，并采用重要性采样（以衰减因子 0.5）混合不同轮次的数据，防止灾难性遗忘。

*   **核心公式**：训练目标为监督微调损失 \( \mathcal{L}_{\text{SFT}}(\theta) = \mathbb{E}_{\tilde{\tau} \sim \tilde{\mathcal{D}}^{(R)}} \sum_{t=1}^T \text{KL}(\delta_{y_t} \| \pi_\theta(\cdot | x, y_{<t})) \)，同时对比了在策略自蒸馏的 KL 散度形式，强调 SFT 使用已验证的离线前缀和硬目标，更适合长程稳定的自进化。

### 3. 实验设计

*   **数据集与基准测试**：评估覆盖 7 个搜索密集和推理基准：
    *   **BrowseComp**（英文复杂浏览）、**BrowseComp-ZH**（中文浏览）、**HLE**（专家级学术推理）、**GAIA**（文本多步推理）、**xBench**（中文深度搜索）、**HotpotQA**、**SearchQA**（含多跳和单跳 QA）。
    *   同时构建了内部验证集 **DeepSearch-Val**（377个实例）用于训练验证和行为分析。

*   **对比方法**：
    *   **闭源专有模型**：OpenAI Deep Research、OpenAI-o3。
    *   **开源智能体**：与近期的搜索/深度研究智能体对比，包括 R1-Searcher、Search-R1、ZeroSearch、ASearcher、DeepResearcher、PokeeResearch、WebSailor、Marco-DR、MiroThinker、DeepDive 等（多数为 7B-8B 规模）。
    *   **自身基座**：**Qwen3.5-9B-Instruct** 作为未经过环境进化的基线。

### 4. 资源与算力

*   **硬件**：训练使用 **2 节点 H20 集群**（配备 8 GPU），模型基于 **Qwen3.5-9B**。
*   **异步训练效率**：生成使用 vLLM 异步进行（支持 TP=4 或 8），与训练过程重叠。一个完整的 **5 轮自进化循环**大约需要 **2-3 天**。主实验执行了 **11 轮**，每轮生成约 1 万条轨迹，触发训练需累积 4 千条合格轨迹。训练部分使用 LlamaFactory，DeepSpeed ZeRO-2，BF16 精度，最大序列长度 32768。

### 5. 实验数量与充分性

*   **实验组合丰富**：
    *   **主实验**：在 7 个公开基准上与多个强基线进行全面对比。
    *   **消融实验**：系统分析了**数据池规模**（420K vs. 100K）、**轨迹质量过滤**（拒绝采样与质量过滤的组合）、**状态内部化与反思重写**等关键组件的影响。
    *   **行为分析**：统计了模型在验证集上的平均交互轮次、搜索/访问工具调用次数，并用 LLM 评委评估了规划、记忆等高级能力。
*   **充分性与公平性**：对比的基线既有闭源最强模型，也有同期同规模的开源智能体，评估维度多样。消融实验清晰展示了各模块的贡献，说服力强。由于环境是可验证和确定的，实验结果具备良好的可复现性。

### 6. 主要结论与发现

*   **显著性能提升**：**DeepSearch-World-9B** 在未使用任何更强模型蒸馏的情况下，相对基座模型 **Qwen3.5-9B-Instruct** 在所有基准上均大幅提升（BrowseComp: +23.8，GAIA: +37.6，HotpotQA: +48.1），并取得与一流开源智能体竞争的性能。
*   **行为质变**：进化后的智能体表现出更长的持续交互（平均 18 轮 vs 4.7 轮）和更充分的深度阅读（访问 5.4 次 vs 0.9 次），其规划、自纠错等高级能力显著增强。
*   **环境与数据的关键作用**：可验证的过程监督和多样化的大规模任务池是支撑智能体稳定自我进化的核心。数据池的多样性远比重复暴露更重要，可防止过拟合。

### 7. 优点

*   **创新的训练范式**：将“确定性可验证环境”、“过程级监督”与“自蒸馏”结合，提供了一条不依赖人类标注或强教师模型来实现智能体自我进化的可行路径。
*   **巧妙的环境设计**：离线 Wikipedia 环境既保证了可复现性和验证能力，又保留了真实搜索-阅读工具的模式（搜索->摘要->访问->全文），减小了仿真与真实应用的差距。
*   **有效的知识迁移**：通过“支架到 ReAct”的轨迹转换，将复杂的规划、记忆和错误恢复能力内化到标准的 ReAct 策略中，使得部署时不需要额外的结构化提示，实用性强。
*   **全面开源与复现**：承诺开源环境、数据、模型和代码，并详细阐述了异步训练配置，有助于后续社区研究。
*   **实验扎实**：对比全面，消融分析深入，对各个设计选择（如数据规模、过滤策略、格式转换）的作用有清晰的实证。

### 8. 不足与局限

*   **知识覆盖有限**：当前环境完全基于 Wikipedia，领域和知识类型单一，可能影响在更开放、多样化真实网络任务上的泛化能力。
*   **训练算法途径单一**：更新规则目前只采用了基于 SFT 的自蒸馏，而 RL 或更纯粹的在策略自蒸馏可能会进一步提升智能体的探索和泛化上限。如何将规划、纠错等高级行为更直接地植入 RL 奖励信号仍有待研究。
*   **跨语言能力**：中文基准（BrowseComp-ZH）的提升幅度（+22.9）低于英文对应基准（+23.8），且训练数据为纯英文，说明跨语言迁移仍有局限。
*   **潜在的偏差风险**：论文也指出，Wikipedia 本身可能包含社会偏见，以此为唯一知识源训练出的模型可能继承这些偏见，需要更广泛的语料和更强的安全对齐。
