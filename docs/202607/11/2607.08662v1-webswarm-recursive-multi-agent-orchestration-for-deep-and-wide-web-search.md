## 摘要
基于大语言模型（LLM）的网络搜索智能体正将信息查找从简单的事实性问答转变为复杂、深度与广度兼具的搜索和研究型任务。单一的 ReAct 风格智能体受限于单条长轨迹和有限上下文，难以同时处理深度和覆盖广度。现有多智能体系统通过并行执行和聚合来提升搜索覆盖面，但在递归深度、协作自适应性和基于证据的扩展方面仍存在明显局限。我们提出 WebSwarm，一种递进式递归委托框架，在推理过程中联合构建任务分解、递归扩展和智能体协作。WebSwarm 动态实例化智能搜索节点，每个节点将局部目标与指定该节点如何组织搜索和协作的搜索模式耦合在一起。每个节点既可以自行解决其目标，也可以进一步委托子节点；在解决后，它向上返回证据和结果，使父节点能够进一步扩展、修正或聚合搜索过程。为引导这一过程，WebSwarm 首先探查网络上任务相关信息是如何组织的，以此作为后续节点扩展的依据，并在同质兄弟节点之间复用过程级经验。在 BrowseComp-Plus、WideSearch、DeepWideSearch 和 GISA 上的实验表明，WebSwarm 在深度、广度以及深度广度交错的任务上始终优于单智能体和多智能体基线。消融实验、任务难度、网络工具效率和模型泛化性的进一步分析解释了 WebSwarm 的有效性，并为多智能体搜索系统提供了洞见。

## Abstract
Large language model (LLM)-based web search agents are transforming information seeking from simple factoid question answering into complex, deep-and-wide search and research-oriented tasks. A single ReAct-style agent is constrained by one long trajectory and limited context, making it difficult to handle depth and coverage simultaneously. Existing multi-agent systems improve search coverage through parallel execution and aggregation, but still exhibit clear limitations in recursive depth, collaboration adaptability, and evidence-grounded expansion. We propose WebSwarm, a progressive recursive delegation framework that jointly constructs task decomposition, recursive expansion, and agent collaboration during inference. WebSwarm dynamically instantiates agentic search nodes, each coupling a local objective with a search mode that specifies how the node should organize search and collaboration. Each node can either solve its objective itself or further delegate child nodes; after solving, it returns evidence and results upward, enabling parent nodes to further expand, revise, or aggregate the search process. To guide this process, WebSwarm first probes how task-relevant information is organized on the web to ground subsequent node expansion, and reuses process-level experience across homogeneous sibling nodes. Experiments on BrowseComp-Plus, WideSearch, DeepWideSearch, and GISA show that WebSwarm consistently outperforms single-agent and multi-agent baselines on deep, wide, and interleaved deep-and-wide tasks. Further analyses of ablation, task difficulty, web tool efficiency, and model generalization explain WebSwarm's effectiveness and provide insights for multi-agent search systems.

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

### 2.1 核心思想：渐进式递归委托
WebSwarm 将原始任务构建为一棵递归委托树 \( T = (V, E) \)。根节点接收原始查询，每个非根节点 \( v \) 都是一个搜索智能体，拥有：
- **局部目标 \( q_v \)**：当前节点要解决的子任务；
- **搜索模式 \( m_v \)**：指定如何组织搜索与协作。

节点既可以**直接执行搜索**（原子节点），也可以**生成并委托子节点**（形成 deeper 的树），完成后将结果与证据**向上返回**给父节点，父节点据此决定继续扩展、修正或聚合。

### 2.2 关键技术细节

**（1）目标—模式耦合的委托（Objective–Mode Delegation）**
生成子委托时，不仅给出子任务 \( q_i \)，同时赋予与之匹配的搜索模式 \( m_i \)。WebSwarm 定义了四种搜索模式，分别应对不同的网络搜索瓶颈：
- **`atom`**：原子事实查找。直接调用网络工具（搜索+页面抓取）快速、可靠地获取证据。
- **`deep`**：迭代搜索与验证。适用于目标实体未知、需通过间接线索推理的场景。内部采用串行的“搜索者–验证者”结构：搜索者从不同线索提出候选，验证者独立核验约束，最终锁定答案。
- **`wide`**：并行分治。适用于对一组已知实体/维度收集同质信息（如构建表格）。同时派发一批独立的子节点，子节点本身可以是任何模式（嵌套实现深度+广度），最后合并结果。
- **`entity_collect`**：开放式实体集合枚举。适用于集合边界未知、需高召回与高精度的情形。并行多个 `atom` 节点从不同来源召回候选实体，再合并、去重、低置信度验证，输出完整集合。

节点依据自身目标选择合适的模式，实现了在同一递归流程中动态切换协作结构。

**（2）证据更新与任务继续**
子节点完成返回结果 \( r_i \) 后，父节点更新自身证据状态 \( R_v \leftarrow R_v \cup \{r_i\} \)，并判断局部目标是否已解决。若未解决，可生成新的子委托、修正搜索方向或请求更细粒度的证据。若已解决，聚合结果并向上返回：
\[
r_v = \text{Aggregate}(q_v, R_v)
\]
这种自上而下的委托与自下而上的证据反馈交替进行，使搜索结构随执行渐进展开。

### 2.3 引导信号：网络结构与经验复用

**（1）网络信息结构探查（Web-Structure-Guided Expansion）**
对于需要“宽扩展”的节点，引入 **Web-Probing Agent** 进行轻量级前置探查：通过少量搜索和页面读取，判断任务相关信息是在少数聚合性中心页面集中，还是按特定维度（品牌、时间、系列等）分散。输出网络结构提示 \( h_v \)（集中/有缺页/分散），指导后续子节点是按照抽取导向还是维度导向进行扩展，从而避免过度分解或方向失误。

**（2）同质节点经验复用（Experience-Guided Node Solving）**
在 `wide` 节点派发大量同质子节点（如为不同实体填充属性）时，先执行少量“侦察节点”（scout），从它们的完整搜索轨迹中提取可复用的过程级经验 \( k_v \)（有效查询模式、可靠来源、应避免的死胡同、页面结构特点等）。然后将这些经验注入剩余子节点的求解上下文，提高一次成功率与结果一致性。

### 2.4 算法流程（伪代码摘要）
论文提供了完整的伪代码（Algorithm 1），核心逻辑为：
- 根节点调用 `SolveNode(v0)`，内部循环判断是否需要继续扩展；
- 如需扩展，先通过 `WebProbing` 获取结构提示，再调用 `SolveOrDelegate` 决定自身求解或生成子委托；
- 若生成子委托，通过 `ScoutExperience` 选择侦察子集并提取经验，然后用 `RunChildren` 并行/串行执行剩余子节点；
- 子节点递归调用 `SolveNode`，返回结果后父节点更新证据，最终聚合输出。

## 6. 论文的主要结论与发现

- **一致且显著的性能提升**：在全部四个 benchmark 上，WebSwarm 均优于单智能体 ReAct 和各类多智能体基线，尤其在深度搜索（BrowseComp-Plus 准确率 +17.5 点）、宽搜索（WideSearch Item-F1 +9.76 点）和深度宽结合任务（DeepWideSearch Item-F1 +11.77 点）上表现突出。
- **递归委托与模式匹配至关重要**：去除递归或强制统一搜索模式会显著降低性能，说明因地制宜地分配 `atom/deep/wide/entity_collect` 模式是框架发挥优势的关键。
- **网络结构探查主要提升效率**：虽对最终 F1 影响较小，但大幅减少无效/冗余的工具调用（如 WideSearch 从 239.9 降至 137.0），使搜索过程更聚焦。
- **经验复用提升同质子节点质量**：侦察节点提取的过程经验能有效避免兄弟节点重复失败的策略，提高表格填充的正确率。
- **困难样本上的优势尤为明显**：难度分层显示，WebSwarm 在 Easy 样本上与基线持平，但在 Mid 和 Hard 样本上拉开显著差距，突破了单智能体和传统多智能体的性能天花板。
- **模型泛化性良好**：无论使用较弱还是较强的 LLM 主干，WebSwarm 均能带来性能增益，说明其框架价值独立于基础模型的能力水平。

## 7. 优点：方法或实验设计上的亮点

- **创新性的递归委托范式**：将搜索过程统一为“目标—模式—委托—反馈”的循环，实现任务分解、扩展和协作在推理过程中协同演进，区别于静态计划或单层派发。
- **搜索模式的细粒度设计**：四个模式精准对应不同的网络搜索瓶颈，且支持嵌套组合（如 `wide` 中包含 `deep` 或 `entity_collect`），灵活应对复杂交织任务。
- **证据驱动的扩展方式**：利用网络信息结构探查来接地气地决定扩展维度，避免盲目分解；经验复用则从内部优化同质节点的执行可靠性，二者互补且轻量。
- **实验评估系统且透明**：覆盖多个专用 benchmark，基线丰富，消融有针对性，难度分析与工具效率分析提供了深入洞察，案例分析生动展示了工作原理。所有方法使用相同工具接口，对比公平。
- **推理阶段框架，即插即用**：无需额外训练或数据构建，可直接应用于不同 LLM，工程可行性高。

## 8. 不足与局限

- **推理时计算开销较大**：相比单智能体，WebSwarm 需要更多 LLM 调用和网络工具请求，导致更高的推理成本和延迟，文中未提供详尽的开销对比数据。
- **仅限文本模态**：当前仅针对基于文本的网络工具（搜索、页面抓取），未涉及多模态信息（图片、视频等），在真实富媒体检索场景中适用性受限。
- **依赖基模型能力**：节点如何委托、模式如何分配、经验如何提取，均依赖底层 LLM 的理解与推理能力，若基础模型较弱，递归链路的可靠性仍可能下降；方法未涉及通过训练内化这些协作策略。
- **超参数敏感性未充分探讨**：例如侦察节点数量（默认 2）、并行路径数（默认 3）等对性能的影响未见详细敏感性分析。
- **跨语言泛化未知**：实验集中在英文数据集，中文或其它语言环境下的有效性有待验证。
- **评估指标的局限性**：部分基准（如 DeepWideSearch 的中间指标）被作者指出不够可靠而未采用，但最终表格指标仍可能无法全面反映搜索过程的中间推理质量。
