## 摘要
微调大语言模型以注入新知识面临一个关键挑战：LLM 能快速记忆新事实，却无法将其用于下游推理任务。我们将这种失败形式化为“知道-使用差距”，其特征是记忆与泛化之间存在准确度差距和时间滞后。为理解这一现象，我们使用未见过的知识微调 LLM，并通过一种新颖的干预技术——自修补（self-patching），监控知识在内部的空间渗透动态。自修补识别激活位置，在这些位置重定位表征能显著改善失败的泛化案例。这些结果与知识回路错位假说一致：记忆的表征可能在内部存在，但未必被路由到计算有效的层。为证明这一诊断发现的实用性，我们设计了一种简单的启发式策略，能在泛化失败中恢复 58% 至 75% 的理论性能余量。实验在跨领域进行，以验证该发现的鲁棒性。

## Abstract
Fine-tuning LLMs to inject new knowledge faces a critical challenge: LLMs can quickly memorize new facts, yet fail to use them for downstream reasoning tasks. We formalize this failure as the \textit{\textbf{Knowing--Using Gap}}, characterized by an accuracy gap and a temporal lag between memorization and generalization. To understand this phenomenon, we fine-tune LLMs with unseen knowledge and monitor the spatial permeation dynamics of the knowledge internally using a novel intervention technique called self-patching. Self-patching identifies activation locations where relocating representations substantially improves failed generalization cases. These results are consistent with a knowledge-circuit misalignment hypothesis: memorized representations can exist internally but may not be routed to computation-effective layers. To demonstrate the practicality of this diagnostic finding, we design a simple heuristic strategy which recovers 58--75\% of the oracle headroom in generalization failure. Experiments are done cross-domain for the robustness of this finding.

## 1. 论文的核心问题与整体含义

- **问题定义**：大语言模型（LLM）微调注入新知识时，存在“知道–使用差距”（Knowing–Using Gap）：模型能快速**记忆**新事实（单跳查询准确率近100%），但在需要多步推理的**泛化**任务中无法可靠使用这些事实，表现为显著准确率差距和到达可靠泛化的时间滞后。
- **研究动机**：现有的知识更新方法（如RAG、知识编辑）存在局限性；微调虽直接操作参数记忆，但“记住但不使用”的现象普遍存在，阻碍了 LLM 作为端到端推理引擎的能力。以往研究多关注行为层面，**缺乏对内部机制的因果理解**。
- **整体含义**：该差距并非模型容量不足，而是**路由问题**——记忆的知识编码在易于存储的早期或晚期层，但未被路由到执行推理所需的中层计算回路中。通过手动重定位表征可大幅恢复泛化能力，表明该失败是可逆的且可被诊断干预修复。

## 2. 论文提出的方法论

### 2.1 核心思想：知识回路错位假说
- **知识回路错位**：微调首先将新事实编码在“易拟合”的存储状态（往往位于早期或极晚期层），支持直接回忆但不支持复杂推理；推理所需的有效计算发生在中层的特定回路中。若记忆表征无法在微调过程中自然渗透至该有效位置，泛化即失败。
- **干预策略**：通过因果干预手动将记忆表征“搬迁”至中层的计算有效层，即可立即激活下游推理能力。

### 2.2 关键技术：自修补（Self-Patching）

- **定位**：一种基于激活修补的因果干预扫描方法，无需“干净”的正确轨迹即可应用于泛化失败案例。
- **操作流程**（见论文算法1）：
  1. 给定源提示 \(P_s\)（通常为记忆任务）和目标提示 \(P_t\)（为泛化任务），指定一个实体锚点 \(E\)（如头实体）。
  2. 在源运行的源层 \(l_s\) 处缓存锚点位置的隐藏状态 \(z = h_{l_s}^{\text{token}(P_s, E)}\)。
  3. 在目标运行的目标层 \(l_t\) 处，将锚点位置的隐藏状态替换为 \(z\)，然后继续前向传播。
  4. 测量正确答案概率的变化 \(\Delta I = I(\tilde{M}(P_t), y^*) - I(M(P_t), y^*)\)。
  5. 扫描所有层对 \((l_s, l_t)\) 得到一张因果效应热力图；正值单元格表示从 \(l_s\) 移动表征到 \(l_t\) 能改善泛化准确率。
- **关键洞察**：若某层对热力图为明显的红色区域（正效应）并扩展到对角线（\(l_s = l_t\)），则表明知识已自然对齐到推理回路；若红色区域停留在非对角线位置，则表明记忆存在但未对齐。

### 2.3 形式化定义

- **记忆准确度** \(A_{\text{mem}}(t)\)：在训练步 \(t\) 时，直接回忆注入事实的准确率。
- **泛化准确度** \(A_{\text{gen}}(t; T)\)：在推理任务 \(T\) 上的准确率。
- **准确率差距**：\(\Delta A = A_{\text{mem}}(T_{\text{max}}) - A_{\text{gen}}(T_{\text{max}}; T)\)。
- **时间滞后**：定义饱合时间 \(T_{\text{gen}}\) 为泛化准确率首次稳定为 1 的最早时机（排除失败案例），滞后 \(\Delta T = T_{\text{gen}} - T_{\text{mem}}\)。
- **对比方法**：自修补与因果追踪、PatchScope 对比（图 3）：因果追踪需干净轨迹；PatchScope 解码语义；自修补直接评估“搬到正确位置”后的下游任务准确性。

## 3. 实验设计

- **知识来源**：基于 STaRK 真实世界知识图谱，选取 **生物医学（STaRK-Prime）** 和 **学术出版（STaRK-MAG）** 两个领域，确保事实在预训练中极少出现（零样本准确率 ≤ 6%，且事后过滤掉已可回答的样本）。
- **任务构建**：自动流水线生成记忆问答对和泛化问答对。

    - **交叉推理**：从多个事实中找出共享实体并排除噪声（如“同时与疾病 A 和疾病 B 相关的暴露是什么？”）。

## 6. 论文的主要结论与发现

1. **知道–使用差距普遍存在**：在链式推理中，记忆可极快饱和（数 epoch 内达近 100%），但泛化准确率长期极低（<20%），且存在显著时间滞后；交叉推理因内在对齐程度不同，差距较小。
2. **知识渗透动态**：自修补揭示记忆在训练早期就已出现在某些层的表征中，但直到红色区域扩展到对角（即知识进入中层计算位置），泛化才出现；失败案例中扩展在到达对角前停止。
3. **知识回路错位假说获因果支持**：Oracle 自修补可立即、大幅度提升泛化准确率（链式提升 1.5~6 倍，交叉差距几乎消失），表明知识已存储，问题在于路由。
4. **空间定位**：有效的源层聚簇在早期层和晚期层，目标层聚簇在中层；移动早期或晚期表征到中层均可激活推理，说明瓶颈在中层回路。
5. **实用性**：固定启发式策略（两个预定层对）能恢复 58–75% 的 oracle 余量，说明该诊断可转化为无需逐例搜索的实用方法。
6. **跨域鲁棒**：在生物医学和学术两个截然不同的领域均复现了相同现象和恢复效果。

## 7. 优点

- **机制化归因**：跳出了行为描述，通过因果干预直接指向内部路由问题，假设清晰且有实验支撑。
- **自修补技术**：新颖且具可解释性，无需干净轨迹即可评估失败案例，填补了因果追踪类方法的空白。
- **多维度验证**：跨模型尺寸、架构、领域、任务类型进行了系统性评估，且重视统计显著性。
- **诊断到药方的转化**：不仅找到原因，还设计了即使简单的启发式也能大幅恢复性能，证实发现的实际价值。
- **数据集与代码开放**：发布了专门评测注入知识多跳推理的数据集，促进可复现研究。

## 8. 不足与局限

- **干预粒度**：自修补在固定锚点位置和层层面操作，未深入注意力头或 MLP 子层，可能低估完整可恢复余量。
- **启发式非自适应**：固定层对虽有效，但仍与 oracle 有差距，需要未来开发实例自适应对齐方法。
- **仅诊断、少预测**：未提供训练早期的预测信号以提前发现哪些事实会泛化失败，干预仍为事后操作。
- **场景限制**：依赖结构化知识图谱生成的任务，未在完全无结构或纯生成式推理任务上验证，结论的一般性有待扩展。
- **知识表征假设**：假设锚点位置（实体 token）捕获了全部事实信息，但知识可能分布于多位置，修补单一 token 可能不完整。
- **依赖微调固定协议**：启发式层对可能受微调超参数影响，鲁棒性需更多协议下测试。
