## 摘要
方面情感三元组抽取（ASTE）要求从给定的评论句子中联合识别（方面，观点，情感）三元组。虽然大型语言模型（LLMs）在许多NLP基准上实现了强大的零样本性能，但它们在ASTE上的有效性仍然有限，因为单次生成迫使模型在单个解码步骤中确定跨度边界、观点分组和情感极性。常见的补救措施，如少样本上下文学习和思维链提示，仅带来有限的改进，并且严重依赖于从标注训练数据中采样的域内演示或精心设计的推理提示，这两者在零样本部署中都无法广泛获得。受经典智能体范式的启发，我们提出了MASTE，一种用于零样本方面情感三元组抽取的多智能体流水线。MASTE将ASTE分解为四个顺序阶段，其中专门的智能体处理不同的组合子任务，并明确依赖于先前的输出。这种设计实现了完全无需训练的零样本ASTE，并可在不同的骨架模型和数据集上泛化。在四个ASTE基准上的大量实验表明，在相同骨架模型下，MASTE大幅优于零样本和思维链LLM基线，缩小了与完全监督方法之间的差距，而无需使用任何标注三元组。代码可在 https://github.com/Hankerlove/MASTE 获取。

## Abstract
Aspect Sentiment Triplet Extraction (ASTE) requires jointly identifying (aspect, opinion, sentiment) triples from a given review sentence. While large language models (LLMs) achieve strong zero-shot performance on many NLP benchmarks, their effectiveness on ASTE remains limited, as single-pass generation forces the model to determine span boundaries, opinion grouping, and sentiment polarity in a single decoding step. Common remedies, such as few-shot in-context learning and chain-of-thought prompting, offer only marginal improvements and rely heavily on either in-domain demonstrations sampled from labeled training data or carefully engineered reasoning prompts, neither of which is broadly available in zero-shot deployment. Inspired by the classical agent paradigm, we propose MASTE, a multi-agent pipeline for zero-shot Aspect Sentiment Triplet Extraction. MASTE decomposes ASTE into four sequential stages, where specialized agents handle different compositional subtasks with explicit conditioning on prior outputs. This design enables entirely training-free zero-shot ASTE and generalizes across different backbones and datasets. Extensive experiments on four ASTE benchmarks show that MASTE substantially outperforms zero-shot and chain-of-thought LLM baselines under the same backbone, narrowing the gap to fully supervised methods without using any labeled triplets. Code is available at https://github.com/Hankerlove/MASTE.

- **现有方法局限**：少样本学习、思维链提示等补救措施改进有限，且严重依赖领域内标注示例或精心设计的推理提示，无法满足严格意义上的零样本部署需求。
- **研究动机**：提出一种**无需任何训练、无需标注三元组**的零样本ASTE方法，通过多智能体流水线将复杂任务分解为可管理的子任务，从而克服单次生成的固有缺陷。

## 2. 论文提出的方法论
- **整体框架**：MASTE（Multi-Agent pipeline for zero-shot Aspect Sentiment Triplet Extraction）是一个**四阶段顺序多智能体流水线**。各智能体共享同一个冻结的LLM骨架，上游输出作为结构化条件显式传递给下游。
- **核心思想**：将ASTE解耦为**方面抽取 → 观点抽取 → 情感推理 → 一致性校验**四个有序子任务，每个子任务由专用智能体完成，并将中间结构化结果对外暴露，便于验证和复用。

- **详细流程**：
    1. **方面抽取智能体（Aspect Extraction Agent）**：  
       - 从原句中识别方面词集合 `A`。  
       - 约束为仅提取原文中显式出现的片段，不做改写或边界扩展。  
       - 输出后经过**确定性串验证**：任何未在句子中出现的方面直接丢弃，避免错误传播。
    2. **观点抽取智能体（Opinion Extraction Agent）**：  
       - 基于句子和已验证的方面集 `A`，为每个方面定位关联的观点表达式。  
       - **跨度感知四步抽取**：  
         ① 方面条件化的观点定位；  
         ② 最小边界校准（去除仅表程度的副词，保留改变极性的部分）；  
         ③ **一对多配对**（一个方面可对应多个独立观点）；  
         ④ 原文片段过滤与去重。  
       - 输出方面‑观点对集合 `P`。
    3. **情感推理智能体（Sentiment Reasoning Agent）**：  
       - 基于原句和 `P`，为每一对分配情感极性 `{POS, NEG, NEU}`。  
       - 依托完整句子上下文，处理否定、强化、对比等极性线索。
    4. **一致性校验智能体（Consistency Check Agent）**：  
       - 在全局三元组集合层面进行**跨三元组验证与修正**：  
         ① **片段扎根**：移除方面或观点未在原文中逐字出现的三元组；  
         ② **极性校准**：依据上下文修改冲突的极性标签；  
         ③ **重复合并**：合并冗余或语义相近的三元组；  
         ④ **输出规范化**：转换为标准三元组格式以适应严格匹配评估。

- **关键设计亮点**：  
  - **训练无关**：全程无需微调任何模型参数，仅通过提示工程使LLM扮演不同角色。  
  - **骨架无关**：同一框架可在GPT-4o、GPT-3.5、Claude、Gemini等不同LLM上直接运行。  
  - **显式条件化**：每个智能体同时接收原句和上游结构化输出，缓解错误传播并增强可解释性。

## 8. 不足与局限
- **错误传播风险**：方面抽取阶段的遗漏（尤其对谓词型、事件型方面）会导致下游全链失效，方面召回仍是主要瓶颈。
- **推理成本较高**：每句4次LLM调用增加了延迟和经济成本，限制了对大规模或实时场景的适用性。
- **数据集与语言局限**：仅在英文餐厅/笔记本评论域上评测，尚未验证多语言、多领域（如社交媒体、医疗、法律）以及更长/更嘈杂文本上的表现。
- **标注约定敏感性**：当数据集标注标准与常识性情感判断不一致时（如“no”被标为中性），模型表现下降，缺乏自适应校准能力。
- **共享观点处理不足**：对于同一观点同时修饰多个方面的情形（如“pleased”对应多个方面），模型倾向于提取局部属性词而遗漏全局情感词。
- **表面形式依赖**：严格精确匹配的评估方式导致拼写错误等小偏差被判定为错误，尽管语义正确，可能低估模型实际能力。
