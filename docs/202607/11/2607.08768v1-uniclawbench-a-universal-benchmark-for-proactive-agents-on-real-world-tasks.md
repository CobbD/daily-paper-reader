## 摘要
大型语言模型和多模态大型语言模型的快速发展加速了主动式智能体的出现，这些智能体能够操作日常工具并在真实世界环境中协助用户。然而，现有的基准测试难以有效评估此类智能体，因为它们通常依赖沙盒环境和单轮评估范式。此外，它们基于场景的任务分类法将多种模型能力混在同一任务类别中，导致难以识别智能体失败的根本原因。为了解决这些局限性，我们提出了 UniClawBench，这是首个以能力为驱动的基准测试，旨在动态、真实的设定中评估主动式智能体。UniClawBench 围绕五项基础模型能力构建：技能使用、探索、长上下文推理、多模态理解和跨平台协调。基于这些能力，我们设计了 400 个双语真实世界任务。与依赖静态、预先录制答案的以往基准不同，我们的基准在实时 Docker 容器中使用细粒度、逐步的完成检查点来评估智能体。此外，我们设计了一种闭环评估策略，包含一个执行者智能体、一个隐藏的监督者智能体和一个用户智能体，以模拟真实的多轮人类反馈，同时不泄露评分标准。为了将基础模型能力与框架层面的设计选择解耦，我们在多个智能体框架下评估了最先进的模型。通过模型和框架之间的全面比较，我们展示了基础模型能力和智能体框架设计如何共同塑造真实世界环境中的性能。为了促进未来的研究，我们将基准和代码公开在 https://github.com/HKU-MMLab/UniClawBench。

## Abstract
The rapid development of large language models and multimodal large language models has accelerated the emergence of proactive agents capable of operating everyday tools and assisting users in real-world environments. However, existing benchmarks struggle to evaluate such agents effectively, as they often rely on sandboxed environments and single-turn evaluation paradigms. Moreover, their scenario-based task taxonomies mix multiple model capabilities within the same task category, making it difficult to identify the root causes of agent failures. To address these limitations, we introduce UniClawBench, the first capability-driven benchmark designed to evaluate proactive agents in dynamic, real-world settings. UniClawBench is built around five foundational model capabilities: Skill Usage, Exploration, Long-Context Reasoning, Multimodal Understanding, and Cross-Platform Coordination. Based on these capabilities, we design 400 bilingual real-world tasks. Unlike previous benchmarks that rely on static, pre-recorded answers, our benchmark evaluates agents in live Docker containers using fine-grained, step-by-step completion checkpoints. Furthermore, we design a closed-loop evaluation strategy comprising an executor agent, a hidden supervisor agent, and a user agent to simulate realistic multi-turn human feedback without leaking grading criteria. To disentangle base model capabilities from framework-level design choices, we evaluate state-of-the-art models under multiple agent frameworks. Through comprehensive comparisons across both models and frameworks, we show how base model capabilities and agent framework designs jointly shape performance in real-world environments. To facilitate future research, we make our benchmark and code publicly available at https://github.com/HKU-MMLab/UniClawBench.

### 2. 方法论：核心思想与技术细节
- **能力导向的任务分类（Capability-Oriented Taxonomy）**：
  将智能体的失败瓶颈解耦为五项基础能力，所有任务均围绕这五者设计：
  - **技能使用（Skill Usage）**：检查、选择并正确执行指定工具或API。
  - **探索（Exploration）**：在信息不完全、存在误导时进行开放式调查，主动发现解决路径。
  - **长上下文推理（Long-Context Reasoning）**：整合、追踪跨多源长文档的证据，保持全局一致性。
  - **多模态理解（Multimodal Understanding）**：从图像、视频、音频中提取并生成准确信息。
  - **跨平台协调（Cross-Platform Coordination）**：在浏览器、桌面应用、文件系统、日历等异质应用间同步状态和转移信息。
- **基准构成**：
  - 包含 **400 个双语任务**（中、英文各半），每个能力维度下各有 80 个任务。
  - 所有任务均为手工构建，模拟真实日常场景，覆盖 11 个应用领域和多种输入/输出模态（表格、日志、视频、PDF、日历文件等）。
  - 每个任务以“任务包”（task package）形式提供，包含公开指令、源文件、可用工具/技能、服务，以及仅监督者可见的隐藏参考答案与评分细则（`eval_rule.md`）。
- **闭环三角色评估策略（Three-role Closed-loop Evaluation）**：
  为模拟真实的多轮用户交互且不泄露答案，设计了一个信息防火墙（Information Firewall）：
  - **执行者（Executor）**：被评测的智能体，仅在 Docker 容器中获得公开任务包，在真实浏览器/终端中执行操作。
  - **隐藏监督者（Hidden Supervisor）**：独立的 LLM 代理，有权访问任务隐藏参考和评分规则。它根据执行者产生的轨迹和中间产物，按照细粒度的**逐步检查点（checkpoint）**进行评分，并给出三种状态：`pass`（通过）、`continue`（可恢复，需继续）或 `fail`（不可恢复失败）。
  - **用户模拟器（User Simulator）**：另一个 LLM 代理，仅接收执行者的可见轨迹和从监督者传来的**粗粒度进度信号**（不含任何具体扣分点或答案）。它据此生成自然语言反馈给执行者，推动其修正错误。
  - **防泄漏机制**：监督者的具体评分理由、隐藏参考答案绝不流向执行者或用户模拟器，用户模拟器的反馈还经过一次确定性改写（sanitize），确保评测安全。
- **运行与评分流程**：
  - 执行者在 Docker 环境中运行，经多轮“执行-反馈”循环（最多 2 次用户跟进），直至通过、失败或超时。
  - 采用两个指标：**Pass Rate (PR)**（最终通过率）和**Average Score (AS)**（基于检查点的加权平均分）。
  - 评分规则在隐藏的 `eval_rule.md` 中统一定义，包含权重、计分上限（caps）和恢复/失败判定。

### 3. 实验设计
- **测试平台与数据**：直接在 UniClawBench 的 400 个任务上进行评估，所有任务均在配备真实浏览器、桌面 GUI 和本地文件系统的 Docker 容器内运行。
- **对比维度与方法**：
  - **跨模型对比（Cross-Model）**：选取 10 个先进模型（含 Claude Opus-4.8, GPT-5.4, Kimi-2.6, Qwen-3.5-Plus 等），统一在 **OpenClaw** 框架下测试，以剥离基础模型能力差异。
  - **跨框架对比（Cross-Framework）**：选取 3 个代表性模型（GPT-5.4, Claude Opus-4.8, Kimi-2.6），分别在 **OpenClaw**（集中式单代理）、**EDICT**（多代理编排）和 **Nanobot**（超轻量级核心）三种框架上测试，分析框架设计对性能的影响。
  - **评估可靠性验证**：随机抽取 50 条完成轨迹，由 3 位人类专家独立评判，验证自动监督者评分与人类判断的一致性（通过率一致度 92%，AS 的皮尔逊相关系数 r=0.71）。

### 6. 主要结论与发现
- **整体任务仍极具挑战**：表现最好的模型（Claude Opus-4.8）整体通过率仅 **47.5%**，所有模型均低于 50%，暴露出沙盒能力与真实世界执行间的巨大鸿沟。
- **“半途而废”现象普遍**：大部分模型的中间步骤得分（AS）较高，但最终通过率（PR）骤降，说明智能体在长链条执行中频繁犯下不可恢复的错误。
- **能力瓶颈明确**：技能使用和探索类任务相对简单；**长上下文、多模态和跨平台任务**是共性的主要瓶颈，要求智能体兼具记忆、跨模态对齐和多应用状态协调能力。
- **框架设计影响可能超越模型能力**：在某些对比中，框架选择对性能的影响比更换基础模型更大。集中式框架 OpenClaw 凭借统一的轨迹管理获得最高通过率；多代理框架 EDICT 因子代理间协调不畅，出现“高平均分、低通过率”；极度轻量的 Nanobot 令牌效率极高，但难以生成足够完整的证据链以满足严格评分。
- **多轮交互的必要性**：引入用户模拟器反馈后，智能体性能在交互周期中持续提升，印证了闭环评估与真实人类交互循环的重要性。

### 7. 优点
- **首创能力驱动的诊断基准**：不再按场景“黑盒”评估，能直接映射失败到具体能力维度。
- **高度拟真与动态**：在实时 Docker 中操作真实网站和桌面应用，解决了沙盒评估 “环境过时” 的痛点。
- **精妙的闭环评估设计**：通过三角色与信息防火墙，在模拟多轮用户纠错的同时，严格杜绝评分泄露，兼顾了真实性与内部有效性。
- **联合分析模型与框架**：同时评估模型基座和系统架构两个层面，揭示了框架设计对智能体上限的放大或瓶颈效应。
- **良好的可复现性与验证**：公开了详尽的任务定义格式（YAML）、评分细则模板及人工对齐实验，增强了评测的可信度。

### 8. 不足与局限
- **任务规模与覆盖偏差**：400 个手工构建的任务虽精心设计，但可能无法完全覆盖真实世界的长尾情况；且目前仅支持中英双语。
