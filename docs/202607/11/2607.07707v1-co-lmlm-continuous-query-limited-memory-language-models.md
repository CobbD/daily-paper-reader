## 摘要
有限记忆语言模型（LMLM）在预训练期间将事实知识外部化到知识库（KB）中，而不是将其记忆在模型权重中。在生成过程中，模型根据需要从知识库中获取知识。这一新近引入的范式提供了多种优势，包括传统LLM仍然无法实现的知识控制能力。我们提出了连续查询LMLM（CO-LMLM），其中知识库将连续键与文本知识值配对，这与先前依赖关系知识库和查询的方法有显著不同。CO-LMLM以最小的代价生成灵活的向量查询，同时仍然将人类可读且可归因的检索知识集成到其生成中。我们将这一设计与一个标注流水线配对，该流水线在任意文本中标记自由形式的事实性片段，消除了先前工作对维基百科的限制。在维基百科和FineWeb-Edu上的预训练以及多个模型规模下，CO-LMLM在困惑度和事实精确度方面均优于先前的LMLM和普通LLM。在360M参数量规模下，这包括比使用40倍以上数据预训练的模型更低的困惑度，以及经过SimpleQA验证的性能，该性能与gpt-4o-mini相当，且高于Claude Sonnet 4.5。

## Abstract
Limited memory language models (LMLMs) externalize factual knowledge during pretraining to a knowledge base (KB), rather than memorizing it in their weights. During generation, the model then fetches knowledge from the KB as needed. This recently introduced paradigm provides multiple advantages, including knowledge control capabilities that remain beyond conventional LLMs. We propose continuous-query LMLM (CO-LMLM), where the KB pairs continuous keys with textual knowledge values, a significant departure from prior reliance on relational KB and queries. CO-LMLM generates flexible vector queries at minimal cost, while still integrating human-readable and attributable retrieved knowledge into its generation. We pair this design with an annotation pipeline that tags free-form factual spans in arbitrary text, removing prior work's restriction to Wikipedia. Across pretraining on Wikipedia and FineWeb-Edu and at multiple model scales, CO-LMLM outperforms prior LMLMs and vanilla LLMs in both perplexity and factual precision. At 360M scale, this includes lower perplexity than models pretrained on 40x more data, and SimpleQA-verified performance that is in line with gpt-4o-mini and higher than Claude Sonnet 4.5.

# Co-LMLM 论文详细总结

## 1. 核心问题与整体含义

- **研究背景**：大语言模型（LLM）通常将事实知识隐式存储在模型参数中，导致知识难以编辑、归因和遗忘，且长尾知识回忆不佳。有限记忆语言模型（LMLM）通过在预训练时将事实知识外部化到可读知识库（KB）来解决该问题，但先前的 **REL‑LMLM** 依赖关系型 KB 和显式的自然语言查询，存在如下核心局限：
  - 必须基于维基百科的结构化实体‑关系事实，无法轻松扩展到任意文本；
  - 查询必须解码为自然语言元组，带来额外的生成开销；
  - 查询表示固定且由预处理决定，限制了检索的灵活性和表达力。
- **整体含义**：**Co‑LMLM** 用连续向量查询替代关系查询，用**连续键‑文本值**的 KB 替代关系元组，同时保留知识的外部化、可读性、可编辑性等优点，为**可归因、可控的知识语言生成**提供了更通用、可扩展的新范式。

## 2. 方法论

### 2.1 核心思想
- 将外部 KB 设计为**向量键 → 文本事实片段**的键值对。
- 模型在生成过程中通过一个特殊的 `<FACT>` token 触发检索：该位置的 **Transformer 最后一层隐藏状态** 直接作为 **连续查询向量**，从 KB 中检索最相似的文本知识值，插入到生成上下文中继续解码。

### 2.2 模型训练
- **训练目标**：联合优化 **下一 token 预测（NTP）** 和 **双向 InfoNCE 对比损失**。
- **数据处理**：训练文档中被标注出若干个 `<FACT>…</FACT>` 包裹的事实片段，且每个片段配有一个自然语言问句（其答案即为该片段）。
- **NTP 损失**：**不优化**事实片段内部及结束 token 的预测概率（即强制模型遗忘这些知识），仅优化非事实 token 和触发 token `<FACT>` 的预测。
- **对比损失**：
  - 以文档中 `<FACT>` 位置的隐藏状态作为检索向量 \(f\)；
  - 以对应问句中 `<FACT-q>` 位置的隐藏状态作为查询向量 \(q\)；
  - 使用当前 batch 内的 \( (f, q) \) 正对和所有其他问句作为负对，计算双向 InfoNCE 损失，温度 \(\tau=0.07\)，权重 \(\lambda=0.25\)。
- **索引构建**：训练完成后，用模型在预训练语料上提取所有 `<FACT>` 位置的隐藏状态并 L2 归一化，与对应的事实文本一同存入 FAISS 密集向量索引。

### 2.3 数据预处理流水线
- **种子标注**：使用 Gemini‑3.1‑Pro 在小规模文档上以 `<FACT q="... " a="...">...</FACT>` 格式标注事实片段。
- **轻量标注器训练**：
  - **事实片段标注器**：基于 ModernBERT 编码器，BIO 序列标注方式识别文本中的事实片段。
  - **问句生成器**：基于 Qwen2.5‑1.5B‑Instruct，输入已标记事实的文档，输出每个片段的问句‑答案对。
- **大规模应用**：将两个标注器应用于全量预训练语料（维基百科约 3B tokens，FineWeb‑Edu 90B tokens），生成最终训练数据。

### 2.4 推理流程
- 模型生成到 `<FACT>` token 时，提取该步隐藏状态作为向量查询，执行 FAISS 查找，将 Top‑1 文本值插入上下文，随后添加 `</FACT>` 并继续生成。

## 6. 主要结论与发现

- **性能优越**：Co‑LMLM 在所有规模和基准上 **困惑度显著低于 Standard LM 和 REL‑LMLM**；在 360M 规模下，动态困惑度（10.5）甚至低于用 **40 倍数据** 训练的 SmolLM2‑360M（14.0）。
- **事实精确度大幅提升**：相比 Standard 基线，**TriviaQA 提升 20+ 点，PopQA 提升 30+ 点，SimpleQA 提升 18‑21 点**；360M‑FW 变体的 SimpleQA 得分（21.7）与 gpt‑4o‑mini 持平，高于 Claude Sonnet 4.5。
- **NLU 保持**：在 CSQA、HellaSwag 等 5 个 NLU 任务上，Co‑LMLM 性能与 Standard LM 基本持平，并未因知识外部化而下降。
- **连续查询优势**：对比 LMLM‑ASKER（文本查询），Co‑LMLM 不仅事实得分更高（差距达 7‑21 点），且 **查询生成开销降低约 13 倍**（2.2ms vs 28ms）。
- **知识真正外部化**：禁用 KB 后，Co‑LMLM 的事实指标急剧下降（如 TriviaQA 下降 23‑28 点），证实知识主要存储在外部而非参数记忆中。
- **可编辑与可遗忘**：在 TOFU 基准上，仅删除 KB 中的相关条目即可实现有效遗忘（p 值 > 0.05），且模型通用能力无损；而传统 NPO 方法则导致通用能力退化。
- **数据规模收益**：使用 FineWeb‑Edu 进行预训练并扩大索引，可进一步提升事实精度，表明方法具有随数据扩展的潜力。

## 7. 优点

- **方法创新**：首次将 LMLM 中的显式关系查询替换为 **连续向量查询**，既保留了 KB 的可读性，又极大提升了检索灵活性和效率。
- **标注流水线**：自动化的 Gemini‑蒸馏 **two‑stage 标注框架** 可以低成本地将任意文本转化为带事实标注的训练数据，突破了维基百科的限制。
- **训练设计巧妙**：通过 **不优化事实 token 的 NTP 损失 + 双向对比损失** 实现了端到端的外部化训练，不依赖额外查询编码器。
- **实验扎实**：多维度、多基线的对比，消融实验全面，且针对困惑度评估的局限性提供了悲观估计，保证了结论的可靠性。
- **实际价值**：实现训练无关的知识遗忘，提供可解释、可归因的生成，对负责任 AI 部署有直接意义。

## 8. 不足与局限

- **规模有限**：模型参数仅到 360M，预训练数据最多 90B tokens，尚不明确能否线性扩展到数十亿参数、万亿 token 的大规模训练。

- **索引更新问题**：一旦模型微调，其隐藏空间改变，原构建的索引将失效，论文未讨论如何高效增量更新索引。
- **查询触发时机**：强制查询可比自发查询带来进一步增益，表明模型在决定何时检索上仍存在不足，影响整体性能。
