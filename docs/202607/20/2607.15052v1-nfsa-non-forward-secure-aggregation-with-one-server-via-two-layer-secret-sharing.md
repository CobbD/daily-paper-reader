## 摘要
联邦学习（FL）在保持数据本地化的同时实现了协作模型训练，从而保护隐私。然而，通过模型更新泄露敏感数据的风险要求使用安全聚合协议。现有的基于服务器的安全聚合协议通常要求服务器转发用户之间共享的敏感数据，这增加了通信开销并引入了潜在的安全风险。在这项工作中，我们提出了一种基于两层秘密共享的新型安全聚合协议来解决这些问题。通过将 Shamir 秘密共享与使用伪随机函数（PRF）的 2取2 加法秘密共享相结合，我们的协议消除了用户之间的直接通信，从而无需服务器转发数据。我们进一步使用密钥同态 PRF（KhPRF）扩展协议以支持高维数据聚合，并将其应用于联邦学习，从而实现了单服务器且无中间数据转发的一次性安全聚合。为了降低用户开销，我们基于中国剩余定理为几乎 KhPRF 的掩码设计了一种新的编码方法，减少了 KhPRF 的调用次数，并减轻了掩码后模型更新的膨胀问题。实验结果表明，我们的方案在辅助节点开销方面显著优于现有方法。例如，当用户数量为 100 时，我们的方案将通信效率提高了近 100 倍，并将计算开销降低了约 17%。此外，当输入长度为 $2^{18}$ 时，用户计算时间可减少 51% 至 75%。

## Abstract
Federated Learning (FL) enables collaborative model training while preserving privacy by keeping data local. However, the risk of sensitive data leakage through model updates necessitates the use of secure aggregation protocols. Existing server-based secure aggregation protocols typically require the server to forward sensitive data shared between users, which increases communication overhead and introduces potential security risks. In this work, we propose a novel secure aggregation protocol based on two-layer secret sharing to address these issues. By combining Shamir's Secret Sharing with 2-out-of-2 additive secret sharing using a Pseudo-Random Function (PRF), our protocol eliminates direct communication between users, thereby removing the need for the server to forward data. We further extend the protocol with Key-homomorphic PRF (KhPRF) to support high-dimensional data aggregation and apply it to FL, enabling one-shot secure aggregation with a single server and no intermediary data forwarding. To reduce user overhead, we design a new encoding method based on the Chinese Remainder Theorem for the almost KhPRF-based mask, reducing the number of KhPRF calls and mitigating the model update expansion issue after masking. Experimental results show that our scheme significantly outperforms existing methods in terms of auxiliary node overhead. For instance, when the number of users is 100, our scheme improves communication efficiency by nearly 100 times and reduces computational overhead by approximately 17\%. Moreover, user computation time can be reduced by 51\% to 75\% when the input length is $2^{18}$.

- **核心问题**：现有基于单一服务器的安全聚合方案（如利用Shamir秘密共享）通常要求服务器**转发用户间共享的敏感数据**（秘密份额）。这带来两个关键问题：
  - **通信开销巨大**：服务器需要将份额转发给持有者（解密者），复杂度为 $O(N M R)$（$N$为用户数，$M$为持有者数，$R$为轮数），成为训练瓶颈。
  - **安全隐患**：不可信服务器需额外使用认证加密（AE）保护转发内容，且服务器仍可能通过与部分持有者合谋获得信息。
- **整体含义**：本文提出一种**无需服务器转发**的安全聚合协议 NFSA，通过两层秘密共享和基于中国剩余定理的编码，在单服务器场景下实现高效、一次性的安全聚合，显著降低通信和计算开销。

## 2. 方法论

### 2.1 两层秘密共享（TLSS）
- **第一层**：**Shamir秘密共享**（阈值 $T$）——用户将私密数据（如KhPRF密钥）分成 $M$ 份，容忍最多 $T-1$ 个持有者离线或合谋。
- **第二层**：**2-out-of-2加法秘密共享**结合**伪随机函数（PRF）**——对每一份Shamir份额 $s_m$，用户利用与持有者 $P_m$ 预先协商的密钥 $\kappa_{d,m}$（通过密钥协商协议KA获得），生成伪随机掩码 $s^{A1}_m = F_{\kappa_{d,m}}(v)$，然后计算 $s^{A2}_m = s_m - s^{A1}_m \mod p$ 并仅将 $s^{A2}_m$ 发送给服务器。
- **消除转发**：持有者 $P_m$ 可在需要时自己用 $\kappa_{m,d}$ 生成 $s^{A1}_m$ 并返回给服务器。服务器通过 $s'_m = s^{A1}_m + s^{A2}_m$ 恢复Shamir份额，整个过程**服务器不转发任何秘密信息**，且持有者在分享阶段无需在线。
- **性质**：TLSS具有**计算安全性**、**加法同态性**以及**批处理能力**（支持使用打包Shamir秘密共享和一次PRF调用生成多个伪随机数）。
- **开销优势**：与经典Shamir SS相比，分享阶段服务器通信量约减半，持有者无需接收数据，总通信和计算开销显著降低。

### 2.2 面向几乎密钥同态PRF的CRT编码
- **问题背景**：为支持一次性聚合（用户只与服务器交互一次），现有工作采用**几乎密钥同态PRF（KhPRF）**对输入进行掩码。但KhPRF的加法同态会引入线性累积误差 $e$（当聚合 $n$ 个输入时，$|e| \le n$）。传统做法通过放大因子 $\Delta$（需 $\Delta > n$）抑制误差，但这使得每个输入元素的比特数增加 $\log_2 (n+1)$，当用户数和模型维度大时，通信开销急剧膨胀。
- **CRT编码核心思想**：
  - 将输入向量 $\mathbf{x} \in \mathbb{Z}_{p_m}^{d_c}$ 的每一维视为不同模数 $p_i$ 下的余数（$p_i > p_m$ 且两两互质）。
  - 利用 **表示扩展**（处理负数范围映射）和**中国剩余定理（CRT）**将 $d_c$ 个元素打包为一个整数 $x \in \mathbb{Z}_{p_c}$ （$p_c = \prod p_i$）。
  - 由于CRT和表示扩展均保持加法同态性，聚合可在打包后的单个数上进行，解码时再解包。
- **收益**：掩码维度降为原来的 $1/d_c$，从而**减少了KhPRF调用次数**（计算开销降低），同时避免了每个元素单独乘以 $\Delta$，仅打包后的结果需要 $\Delta$，**大幅缩减通信量**。

### 2.3 NFSA协议流程（高维模型聚合）
- **离线阶段**：所有用户与解密者（持有者）之间执行密钥协商，产生共享的对称密钥 $\kappa_{i,m}$。
- **在线阶段（每一轮）**：
  - **掩码上传**：用户 $U_i$ 采样KhPRF密钥 $\mathbf{k}_i$，用TLSS将其共享给解密者（份额上传服务器），并对本地模型更新 $\mathbf{x}_i$ 做CRT编码后计算掩码 $\mathbf{c}_i = \Delta \cdot \text{CRT}(\mathbf{x}_i) - F_H(\mathbf{k}_i, r) \mod \Delta p_c$，上传 $\mathbf{c}_i$ 和TLSS份额。
  - **解密者响应**：在线解密者 $P_m$ 利用TLSS的加法同态性，直接计算所有在线用户的聚合密钥份额 $s^{A1,r}_m = \sum_i F_{\kappa_{m,i}}(r, n_\lambda)$，并发送给服务器。
  - **服务器解掩码**：服务器聚合用户上传的 $s^{A2}_m$ 份额，用TLSS重构出聚合密钥 $\mathbf{k} = \sum_i \mathbf{k}_i$，然后计算 $y = F_H(\mathbf{k}, r) + \sum_i \mathbf{c}_i$，最后通过除以 $\Delta$ 和 CRT 解码获得聚合结果。
- **安全模型**：半诚实模型，服务器至多与 $T-1$ 个解密者合谋。
- **安全性**：基于KA、哈希函数、TLSS和KhPRF的安全性，证明用户输入除最终聚合结果外不会泄露。
