## 摘要
越来越多的一类智能体系统通过记忆文件、行为偏好和知识库在会话之间维持持久状态。虽然这使智能体更加有用并能自我改进，但它也为提示注入创造了新的攻击面，恶意指令可被嵌入持久文件中并影响未来的行为。在这项工作中，我们使用沙盒化的合成工作区研究基于记忆的智能体系统中的提示注入攻击。我们评估了两个智能体系统，Anthropic Claude Code 和 OpenAI Codex，在四个模型上：Claude Haiku 4.5、Claude Opus 4.7、GPT-5.2 和 GPT-5.5。我们的结果表明，尽管难以利用不可信的外部内容让智能体覆盖其自身的记忆文件，但已植入这些文件中的载荷能够成功攻击当前和未来的会话。攻击成功率和载荷持久性在不同系统、模型、对抗目标和多会话攻击序列之间差异很大。这些发现表明，持久记忆改变了提示注入的威胁模型，并促使我们构建能够保护记忆更新同时不消除有用智能体适应性的防御机制。

## Abstract
A growing class of agentic systems maintain persistent state across sessions through memory files, behavioral preferences, and knowledge bases. While this makes agents more useful and self-improving, it also creates a new attack surface for prompt injections in which malicious instructions can be embedded within persistent files and influence future behavior. In this work, we study prompt injection attacks in memory-based agentic systems using a sandboxed synthetic workspace. We evaluate two agentic systems, Anthropic Claude Code and OpenAI Codex, across four models: Claude Haiku 4.5, Claude Opus 4.7, GPT-5.2, and GPT-5.5. Our results show that although it is difficult to make an agent overwrite its own memory files using untrusted external content, payloads already planted in those files can successfully attack current and future sessions. Attack success and payload persistence vary substantially across systems, models, adversarial goals, and multi-session attack sequences. These findings show that persistent memory changes the threat model for prompt injection and motivate defenses that protect memory updates without removing useful agent adaptation.

### 6. 主要结论与发现
- **载荷植入即可生效**：一旦恶意内容进入持久文件，无论是自动加载还是引用文件，均可影响智能体行为，成功执行凭证窃取、安装漏洞包或品牌推荐。
- **模型强弱影响攻击成功率，但规律不绝对**：
  - 强模型（Opus、GPT‑5.5）对明显的恶意目标（凭证窃取）抵抗性更强（ASR 0%）。
  - 但对于更隐蔽的攻击（如品牌定向），强模型可能更顺从（GPT‑5.5 100% 推荐指定品牌），而弱模型有时会主动移除规则。
  - 同一模型族内强弱模型表现可相反（Haiku 高 ASR vs. Opus 低 ASR；GPT‑5.5 高 ASR vs. GPT‑5.2 低 ASR）。
- **攻击可跨会话持久与叠加**：
  - 相同攻击载荷在经历稳定化会话后仍能保持较高持久率（Claude 模型普遍接近 100%）。
  - 不同攻击可同时驻留记忆，且攻击顺序影响成功率（例如先进行凭证窃取可能使模型“警觉”，降低后续攻击成功率）。
- **ASR 与持久性的脱节**：Opus 攻击成功率最低（18.3%），但负载持久率最高（93.3%）；它倾向于**警告而不删除**载荷，给未来会话或更弱模型留下风险。
- **智能体会主动修改记忆以应对攻击**：包括删除恶意规则、添加安全策略，或错误地将危险指令移至更受信任的文件（如 GPT‑5.2 把漏洞依赖移至 `AGENTS.md`）。
- **整体结论**：持久记忆使提示注入从瞬时攻击变为持久性威胁，需要更细粒度的信任分级、更新审核和会话加载时验证。

### 7. 优点
- **问题新颖**：系统性地首次定量研究文件型持久记忆在**多会话提示注入**中的作用，将攻击持久性、叠加效应与模型特性关联。
- **威胁模型覆盖全面**：区分自动加载、条件引用和通用行为文件三种记忆层级，搭配不同恶意目标。
- **评估协议细致**：通过探测–稳定化–探测序列解耦攻击即时效应与持久/叠加效应，同时记录攻击成功率和载荷持久率两个维度。
- **案例研究深入**：结合定量表格与定性案例（如 Opus 识别漏洞但保留攻击、GPT‑5.2 迁移攻击至更受信文件），展示了模型行为的微妙差异。
- **现实意义强**：指出记忆共享、社区配置复用的真实风险，为防御设计提供有据可查的动机。

### 8. 不足与局限
- **环境与任务单调**：仅使用一个合成工作区和少量固定良性任务，未覆盖真实项目中多样的文件结构和交互模式。
- **攻击注入方式限定**：假设攻击载荷已存在，未成功实现“由不可信外部内容诱导智能体自动覆写记忆”的完整攻击链，可能低估真实攻击发起的难度/概率。
- **实验规模限制**：
  - 每种条件仅 10 次试验，对高方差行为（尤其智能体随机性）统计效力不足。
  - 仅测试 4 个商业模型，缺少开源模型及更多中间规模模型的对照。
- **缺少防御评估**：未测试任何缓解策略（如提示隔离、文件签名、人工审批），降低了直接指导实践的价值。
- **模型版本依赖**：评分结果可能随时因 API 更新或模型微调而改变，可复现性受限。
- **ASR 不可直接跨目标比较**：因目标、植入位置、探测任务均不同，横向 ASR 对比需谨慎解读。
