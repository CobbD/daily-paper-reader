## 摘要
多模态AI代理越来越依赖持久长期记忆来基于过去的视觉和文本片段进行生成。我们表明，对视觉数据的无条件信任会造成严重漏洞。我们提出了Lucid，一个黑盒对抗框架，在严格限于图像的威胁模型下破坏多模态记忆管道，无需访问目标多模态大语言模型、目标检索编码器或文本通道。Lucid精心构造不可察觉的扰动，根据历史上下文的可用性实现两种不同的失效模式：（1）记忆中毒，一种上下文内攻击，其中对抗性图像替换了内容由先前文本上下文强化的良性图像，从而可靠地破坏视觉回忆并将代理引向攻击者选择的叙述；（2）记忆注入，一种上下文外攻击，其中对抗性图像替换了没有先前文本基础的对话轮次中的良性图像，导致代理生成受攻击者影响的响应，且无法从记忆中纠正。我们跨多种对话领域和五种黑盒记忆架构（包括图结构、LLM总结和商业部署系统）评估了Lucid。Lucid在中毒攻击上取得了61.6%的ASR，在注入攻击上取得了58.4%的ASR，暴露了多模态记忆管道中的结构性漏洞。

## Abstract
Multimodal AI agents increasingly rely on persistent long-term memory to ground generation in past visual and textual episodes. We show that unconditional trust in visual data creates a critical vulnerability. We propose Lucid, a black-box adversarial framework that compromises multimodal memory pipelines under a strictly image-bounded threat model, requiring no access to the target MLLM, target retrieval encoder, or the text channel. Lucid crafts imperceptible perturbations to enable two distinct failure modes based on the availability of historical context: (1) Memory poisoning, an in-context attack where the adversarial image replaces a benign one whose content is reinforced by prior textual context, reliably corrupting visual recall and steering the agent toward attacker-chosen narratives; (2) Memory injection, an out-of-context attack where the adversarial image replaces a benign one in a conversation turn devoid of prior textual grounding, causing the agent to generate attacker-influenced responses with no corrective signal from memory. We evaluate Lucid across various conversation domains and five black-box memory architectures, including graph-structured, LLM-summarized, and commercially deployed systems. Lucid achieves 61.6% ASR on poisoning and 58.4% ASR on injection, exposing a structural vulnerability in multimodal memory pipelines.

## 1. 核心问题与整体含义（研究动机和背景）

多模态AI代理（如虚拟助手、具身智能体）正越来越多地依赖**长期记忆**来跨越会话存储并利用用户共享的文本和图像，以提供个性化回复。它们默认假设记忆中的图像是真实可信的，未对视觉内容的语义完整性进行任何验证。这种**对视觉数据的无条件信任**构成了一个严重的安全漏洞。

本文旨在回答一个关键问题：**攻击者能否在仅对用户上传的图像施加肉眼不可察觉的扰动、且无法访问文本通道、视觉编码器、检索编码器或模型权重（严格黑盒）的条件下，系统性地污染多模态代理的长期记忆？**论文的研究揭示了这一威胁的可行性，并提出了攻击框架**LUCID**，以此警示社区多模态记忆管道存在结构性弱点。

## 2. 论文提出的方法论

LUCID 是一个**全黑盒对抗框架**，以**图像扰动为唯一自由度**，无需接触目标系统的任何内部组件。攻击分为三个关键阶段：

- **阶段1：对抗目标设计**
  - 从大规模图像–描述语料库（ShareGPT4V-100K）构建候选池，利用 CLIP 检索位移分数、语义槽矛盾分数及与真值答案的重叠惩罚，为每一个受害者问答对自动选出语义上矛盾、能引发错误信念的目标图像及其描述。
- **阶段2：对抗载荷构造**
  根据可用上下文分为两种攻击模式：
  - **上下文内记忆中毒**：替换已有对话轮次中的图像，但保留原对话文本，使代理在正确文本锚定下错误回忆该事件。
  - **上下文外记忆注入**：在全新主题的伪对话轮次中植入被动过的图像，使代理在无任何先验文本纠正信号的情况下生成受攻击者影响的响应。
- **阶段3：对抗扰动生成**
  基于**迁移攻击**思想，在代理模型（CLIP ViT-B/16、CLIP ViT-B/32、LAION-CLIP 的集合）上优化损失函数：
  \[
  \mathcal{L}(\tilde{x}, x^*, c^*) = \mathcal{L}_{\text{OT}}(\tilde{x}, x^*) + \lambda_{\text{txt}} \mathcal{L}_{\text{txt}}(\tilde{x}, c^*)
  \]
  其中 \(\mathcal{L}_{\text{OT}}\) 是特征最优对齐损失（结合全局余弦相似度与最优传输聚类匹配），\(\mathcal{L}_{\text{txt}}\) 是本文扩展的**文本对齐损失**，用于强制对抗样本的视觉嵌入与目标描述的文本嵌入对齐，提升下游多模态模型描述的误导效果。
  采用投影 I-FGSM 在 \(\epsilon = 16/255\) 约束下迭代 1000 步得到最终对抗图像。

攻击完全不需要对抗性文本、触发器或对记忆存储的直接写入权限，突破了现有视觉记忆攻击的假定。

## 3. 实验设计

- **数据集与基准**  
  使用 **Mem-Gallery** 基准，包含 5 个多会话、多模态对话数据集（涵盖科技伦理、育儿、房产等主题），总计 68 个会话、1071 轮对话、346 个人工标注问答对。问题类型涵盖视觉搜索、记忆回忆、时间推理、缺席识别、知识修订等 9 类，能全面测度长期记忆的完整性。

- **记忆后端**  
  覆盖 5 种架构迥异的记忆系统：
  - MuRAG（密集多模态检索）
  - NGMemory（图结构情景记忆）
  - AUGUSTUS（概念标记图＋协同检索）
  - UniversalRAG（模态路由 RAG）
  - Mem0Memory（基于 LLM 提炼事实的商用记忆系统）

- **多模态模型**  
  作为产生回复的骨干模型，实验涉及 GPT-4o-mini、GPT-4o、GPT-4.1、Claude-Haiku-4.5、Gemini-2.5-Flash 等五种 MLLM。

- **对比条件**  
  每个攻击家族（投毒 / 注入）均设置三种条件：
  - **干净基线**：无攻击
  - **Oracle**：直接用真实目标图像替换原图（攻击效果理论上界）
  - **对抗性**：使用黑盒迁移扰动图像（实际攻击）

- **评估指标**  
  攻击成功率（ASR，含视觉相似度 ASR 和条件 ASR）、检索命中率、回答正确性损失（Corr Loss）、F1、Hit Rate、LLM 裁判评分等。

- **防御实验**  
  测试了 5 种防御：图像预处理（高斯模糊、JPEG 压缩）、写入端过滤（LLM 裁判、语义漂移）、检索端过滤（异常打分）。

## 4. 资源与算力

- 对抗扰动生成在**单个 NVIDIA RTX 3090（24 GB 显存）**上运行。
- 单张图像的最优化扰动大约需要 **10~15 分钟**。
- 所有检索编码器和 MLLM 的推理通过 API 调用，未消耗额外本地训练算力。
- 文中未提及大规模分布式训练或大量 GPU 集群的需求。

## 5. 实验数量与充分性

实验设计**非常全面，覆盖了攻击、防御、多种因素消融**，具体包括：

- 跨 5 个 MLLM 的迁移实验（投毒和注入）。
- 针对注入攻击的 4 种语义类别（身份翻转、过敏安全、联系凭证、活动限制）。

## 6. 论文的主要结论与发现

1. **攻击高度有效且跨架构通用**  
   在严格的图像扰动威胁模型下，LUCID 在记忆中毒和注入上平均达到 61.6% 和 58.4% 的攻击成功率，逼近甚至在某些情况下超越 Oracle 理论上界。攻击效果在五种结构完全不同的记忆后端上均能复现。

2. **记忆注入尤其隐蔽**  
   注入模式不制造文本矛盾，且无历史上下文可对照，故难以通过文本一致性检查发现。

3. **模型间的生成级抵抗力存在差异**  
   某些模型（如 Claude-Haiku-4.5）检索率高但攻击成功率低，表明“检索稳健性”与“生成稳健性”是两个独立的安全维度。

4. **传统防御的局限性**  
   图像预处理（高斯模糊、JPEG 压缩）能一定程度降低 ASR，但写入端和检索端的文本过滤几乎无效。根本原因在于攻击仅依赖图像，文本侧无异常信号。

5. **结构性脆弱性被证实**  
   多模态长期记忆无条件相信视觉数据，缺少语义验证机制，此为当前广泛采用的设计范式下的内生缺陷。

## 7. 优点

- **威胁模型严格**：纯粹黑盒、仅靠图像扰动、无触发器、无文本访问，贴近现实攻击场景。
- **覆盖全面**：五大记忆架构、五种 MLLM、多领域对话，实验链深且广。
- **双模式设计**：区分上下文内中毒和上下文外注入，更全面地刻画视觉记忆攻击面。
- **迁移攻击创新**：融合最优传输对齐和文本对齐损失，提升黑盒迁移效果，且方法通用。
- **诊断性实验**：通过区分检索成功与生成误导，精准定位了安全性的两个独立环节。
- **对防御的深入分析**：系统评估了写时、图像级、检索时的防御，为未来对策提供清晰方向。

## 8. 不足与局限

- **基准局限性**：仅在 Mem-Gallery 上测试，未能涵盖更多样化的现实世界对话记录或不同图像扰动约束。
- **资源开销**：单张图像扰动需 10~15 分钟（RTX 3090），对实时或大规模攻击有性能障碍。
- **扰动可感知性**：虽 ℓ∞ 限幅下肉眼难察觉，但在强分析（如频域检测）下可能被识别。
- **威胁模型前提假设**：攻击需有能力篡改用户发送前的图像（如 CDN 劫持、恶意客户端），在实际中虽可能但依赖具体部署环境。
- **防御评估不够纵深**：未测试对抗训练、输入重建等更强防御，仅尝试了简单预处理和过滤；且图像预处理可能被后续更强扰动（如 PSI）绕过。
- **缺乏真实用户研究**：评估仅基于自动指标，没有衡量人类用户对攻击后果的感知难度。
